# 개발결과보고서 v3 — D2C 통합 CDP / ROAS 어트리뷰션 SaaS (시리즈A 데모)

> 본 보고서는 [`5_과업지시서_v3.md`](./5_과업지시서_v3.md) §5 성과품 중 **자체 산출물(v3 CDP SaaS 본체)** 의 검수 증거다.
> v3는 v2(5억 베타)의 부족함을 메워 **시리즈A 투자 데모** 수준으로 격상하는 사이클이다.

## v2 한계 및 v3 개선 매핑

| # | v2 한계 | v3 해결 방식 | 증거 캡처 |
|:---:|:---|:---|:---|
| 1 | 매체 OAuth **1종(네이버)** 만 | **5종(네이버·카카오·메타·구글·쿠팡)** OAuth 2.0 mock — provider별 동의·token·만료·동기화 시각 기록 | `v3_02`, `v3_03`, `v3_04` |
| 2 | 자사몰 주문 이벤트 인입 부재(CSV 수동) | **카페24/Shopify webhook 수신** — 주문 이벤트 인입 로그 → 대시보드 실시간 활동·인입 매출 반영 | `v3_05`, `v3_06` |
| 3 | 알고리즘 2종(Markov·RFM) | + **예산 재배분 최적화(한계 ROAS 균등화)** + **이상탐지(z-score)** 도입 → 총 4종 | `v3_07`, `v3_08`, `v3_09` |
| 4 | RFM은 분류까지만 (액션 없음) | **세그먼트별 추천 액션·채널·CTA** + 캠페인 발송으로 연결 | `v3_10`, `v3_11` |
| 5 | 자동 리포트 = 발송 로그 수준 | **예약 → 발송 실행 → 발송 이력(ROAS 기록) + 리포트 미리보기** 완결 워크플로 | `v3_11` |
| 6 | 단일 사용자(역할 개념 없음) | **오너/마케터/뷰어 3역할** 권한 매트릭스 + 액션 단계 권한 차단 | `v3_12`, `v3_14` |
| 7 | 변경 거버넌스·감사 추적 부재 | **감사로그/활동 이력**(행위자·역할·브랜드·액션·상세·시각) + CSV 내보내기 | `v3_13` |
| 8 | 뷰 10종 | **14종**(예산·이상탐지·webhook·권한·감사로그 신규) + 모바일 Bottom Nav 유지 | `v3_01`, `v3_16`, `v3_17` |

## v3 신규/심화 산출물

과업지시서 §5 **"v3 자체 산출물 핵심 8가지"** 를 모두 실제 구현했다. 아래 8가지를 캡처와 함께 입증한다.
구동 환경: macOS(Darwin 24.6.0) · Chromium(file://) · 데스크톱 1440×900 / 모바일 390×844 · 페이지 에러 0건.

### ① 매체 OAuth 5종 mock 연동 관리
![매체 연동 5종 개요](./captures/v3/v3_02_oauth_5media.png)
- **무엇을**: '매체 연동(OAuth 5종)' 뷰 — 네이버검색·카카오모먼트·메타IG·구글PMax·쿠팡애즈 5개 카드. 각 provider별 'OAuth 연결' 버튼·미연결 상태.
- **의도**: 제안서 §8 v3 로드맵의 "매체 OAuth 5종"을 실 화면으로 구현(외부 통합 #1).
- **검토 결과**: 5종 모두 노출, 사이드바 '연결 매체' 점등 동기화.

![OAuth 동의 화면 — 카카오](./captures/v3/v3_03_oauth_consent_kakao.png)
- **무엇을**: 카카오모먼트 'OAuth 연결' → **OAuth 2.0 동의 화면**. 요청 scope(광고 캠페인·비용 읽기 / 전환·성과 리포트 읽기 / 결제 요청 안 함) + `provider=kakao · redirect_uri=.../oauth/kakao/callback · scope=ads.read report.read · response_type=code`.
- **검토 결과**: provider 값이 카드별로 동적 치환됨(naver/kakao/meta/google/coupang).

![5종 중 4종 연결 완료](./captures/v3/v3_04_oauth_connected.png)
- **무엇을**: 네이버·카카오·메타·구글 4종 연결 완료 — 각 카드에 `access_token: mock_<provider>_xxxx`, 만료 `3600s`, 최근 동기화 시각이 기록. 쿠팡은 미연결(대비). 하단 '네이버검색 OAuth 연결 완료' 토스트.
- **검토 결과**: 발급 토큰이 `state.brands[].spend[].token`에 저장, 연결/해제 토글 정상.

### ② 자사몰 webhook 수신 시뮬레이션
![webhook 인입 로그](./captures/v3/v3_05_webhook_ingest.png)

- **의도**: 외부 시스템 통합 #2 — 자사몰 주문 이벤트 실시간 인입.
- **검토 결과**: 최신 인입 행 강조(blink), localStorage 영속.

![대시보드 실시간 반영](./captures/v3/v3_06_dashboard_webhook_reflect.png)
- **무엇을**: webhook 인입 후 대시보드 우측 '실시간 활동' 패널에 주문 카페24/Shopify ₩146,103·₩214,496·₩324,898 노출, 매출 KPI(₩351,910,375)에 합산 반영.
- **검토 결과**: 인입 1건당 전환 1건이 당일(day 29)에 주입되어 KPI·실시간 활동에 즉시 반영.

### ③ 슬랙·이메일 자동 리포트 스케줄러
![자동 리포트 — 예약·발송 이력·미리보기](./captures/v3/v3_11_reports_schedule.png)
- **무엇을**: 채널(이메일/Slack/카카오)·주기(매주 월요일)·시간(08:00) 예약 → '스케줄 저장'(활성화 배너 '이메일 · monday 08:00') / '지금 발송 실행' → **발송 이력**(Slack #마케팅-리포트 · ROAS 3.46x · 발송 성공) + **리포트 미리보기**(어제 ROAS 요약 실 데이터: 매출 ₩13,817,065·ROAS 3.46x·네이버 기여·연동 4/5).
- **의도**: 외부 통합 #3(발송) + 다단계 워크플로(예약→발송→이력→미리보기).
- **검토 결과**: 발송 이력에 ROAS 기록, localStorage 영속.

### ④ RFM 자동 세그먼트 + 액션 추천
![RFM + 액션 추천](./captures/v3/v3_10_rfm_action.png)
- **무엇을**: 전환 고객 R/F/M 5분위 채점 → 5세그(VIP 76 / 충성 77 / 잠재 111 / 위험 100 / 이탈 32명) KPI + **세그먼트별 추천 액션 표**(VIP→전용 혜택·카카오 알림톡+이메일·'VIP 캠페인 발송' / 이탈→고할인 컴백 오퍼 등) + 점수 분포 바차트 + 상위 30명 R/F/M 표.
- **의도**: 실 알고리즘(RFM 5분위)을 액션으로 연결. 'CTA 발송'은 발송 이력에 기록.
- **검토 결과**: KPI 인원과 분포 바차트가 동일 `computeRFM()` 산출로 정합.

### ⑤ 예산 재배분 추천 알고리즘 (실 계산)
![예산 재배분 추천](./captures/v3/v3_07_budget_recommend.png)
- **무엇을**: 어트리뷰션(Markov) 기여도 기반 — 현재 ROAS 2.82x → **예상 ROAS 2.89x(+2.4%)**. 현재 vs 추천 예산 바차트 + 매체별 표(한계 ROAS·현재 예산·추천 예산·증감액·증감률). 예: 네이버 ▲₩460,000(+1.2%), 카카오 ▼₩2,340,000(-10.9%), 쿠팡 ▲₩1,910,000(+21.6%).
- **의도**: 실 알고리즘 — 한계 ROAS(기여매출/광고비)를 평균 대비 √비율로 가중, 총 예산 고정 하 재정규화(체감수익 sqrt 완화).

![추천 예산 적용](./captures/v3/v3_08_budget_applied.png)
- **무엇을**: '추천 예산 적용' → 매체별 광고비가 추천값으로 갱신, 표·차트·증감률 재계산. '추천 예산 적용됨 — 대시보드 반영' 토스트.
- **검토 결과**: 적용 후 ROAS 개선폭이 재산정(+0.6%로 수렴 — 고효율 매체 비중 이동 완료), 대시보드 광고비에 반영.

### ⑥ 이상탐지 알림 (z-score)
![이상탐지](./captures/v3/v3_09_anomaly.png)
- **무엇을**: 매체별 일자 ROAS 시계열 z-score 스캔 — KPI(스캔 5종·경고 0·심각 0·정상 5) + 매체별 평균 vs 최근 ROAS·z값·30일 스파크라인. 임계 초과 시 급락/급등·경고/심각 플래그.
- **의도**: 실 알고리즘 — 평균·표준편차 산출 후 최근일 z, |z|≥1.8 경고·≥2.4 심각. 대시보드 상단 배너로도 노출(브랜드별, `v3_15` Beanly에서 ROAS 급락 1건 배너 확인).
- **검토 결과**: '재스캔' 시 재계산, 시계열·평균·z 정합.

## 2. 구현/제작 범위

- 단일 파일 SPA(`v3.html`, HTML + Tailwind CDN + Chart.js 4.4.1), 빌드 불필요·오프라인 자체완결.
- **뷰 14종**: 대시보드·어트리뷰션·예산 최적화·이상탐지·깔때기·세그먼트·RFM 액션·코호트·매체 연동·자사몰 Webhook·자동 리포트·권한·멤버·감사 로그·데이터.
- **워크플로 4개+**: ① OAuth 동의→코드교환→토큰발급→연동완료 ② webhook 인입→로그적재→대시보드 반영 ③ 예산 추천→적용→재계산 ④ 리포트 예약→발송 실행→이력 기록 (추가: 역할 전환→권한 차단, 세그먼트 빌더).
- **실 알고리즘 4종**: Markov removal-effect · RFM 5분위 · 예산 재배분(한계 ROAS 균등화) · 이상탐지 z-score.
- **다중 테넌트/다중 사용자**: 브랜드 3개 독립 데이터 + 역할 3종 권한 분기.
- **외부 시스템 통합 3건+**: OAuth 5종 mock · 자사몰 webhook 인입(카페24·Shopify) · 발송(슬랙·이메일·알림톡) + CSV 입출력.
- 상태 영속: `localStorage[cdp_v3_state_1]` (브랜드·역할·모델·세그먼트·스케줄·발송이력·webhook이벤트·감사로그·OAuth 토큰).
