# 과업지시서 v3 — D2C 통합 CDP / ROAS SaaS (시리즈A 데모)

> v3 위탁 분: **매체 OAuth 연동 SDK 패키징 + 자사몰 webhook 어댑터 + 디자인 QA** 가정 시나리오. CDP SaaS 본체 확장은 자체.
> 사이클 격상 목표: v2(5억 베타) → **v3 시리즈A 데모** (제안서 §8 로드맵 v3: 매체 OAuth 5종 + Shopify·카페24 webhook + 슬랙·이메일 자동 리포트 + RFM 자동 세그먼트).

## 1. 과업의 개요

### 과업의 목적

1) v2 5억 베타 검수 통과 후, **시리즈A 투자 데모** 단계로 격상한다.
2) 외주에 맡기는 직접 목적: 매체 5종(네이버·카카오·메타·구글·쿠팡) OAuth 클라이언트 등록·토큰 교환 SDK 모듈과, 자사몰(카페24·Shopify) 주문 webhook 수신 어댑터를 패키징하여 자체 SaaS 본체에 연결 가능한 형태로 인도받는다.
3) 그 목적이 발주자 측 사업([`1_제안서.md`](./1_제안서.md))과 연결되는 지점: 제안서 §4 블루오션(한국 매체 1-click 연동), §8 로드맵 v3(OAuth 5종·자사몰 webhook), §9 리스크(매체 API 정책 변경 대비 다중 매체 분산)의 실현 근거가 된다.
4) v2 한계 — 매체 OAuth 1종(네이버)만 / 자사몰 이벤트 인입 부재 / 단일 사용자 권한 / 알고리즘 2종 — 을 해소한다.

### 과업의 범위

1) 외주 수행 업무 (가정)
   - 매체 5종 OAuth 2.0 클라이언트 등록 가이드 + 토큰 교환 모듈 (provider별 scope·redirect_uri 규격)
   - 자사몰 webhook 어댑터 (카페24 `order.created`, Shopify `orders/create`) — HMAC 서명 검증·멱등키 dedup 규격
   - 발송 채널(슬랙 Incoming Webhook·이메일 SMTP·카카오 알림톡) 송신 모듈 규격서
   - 시리즈A 데모용 디자인 QA (반응형·접근성 점검 12편)
2) 범위 외 (Out of Scope)
   - CDP SaaS 본체(대시보드·어트리뷰션·예산 최적화·이상탐지·RFM·권한·감사로그) 구현 — **자체**
   - 예산 재배분/이상탐지 알고리즘 — **자체**
   - 실제 매체 API 운영 계정 발급·정산 — 발주자 책임

## 2. 과업의 세부 내용

- **OAuth 5종**: provider(`naver`/`kakao`/`meta`/`google`/`coupang`)별 동의 화면 → `authorization_code` → `access_token`(Bearer, expires_in 3600) 교환. 키 부재 시 mock 토큰 fallback. 연결 상태·토큰·만료·최근 동기화 시각 기록.
- **자사몰 webhook**: `POST /webhooks/ingest`. 소스별 이벤트 스키마(주문번호·금액·시각) 정규화 → 인입 로그 적재 → 대시보드 실시간 반영. HMAC 서명 검증·멱등키 중복 제거.
- **자동 리포트**: 채널(이메일/Slack/카카오)·주기(매일/매주/매월)·시간 예약 → 발송 실행 → 발송 이력(시각·채널·대상·ROAS·상태) 적재 → 리포트 본문은 실 데이터 동적 생성.
- **RFM + 액션**: R/F/M 5분위 채점 → 5세그(VIP/충성/잠재/위험/이탈) → 세그먼트별 추천 액션(채널·CTA).
- **예산 최적화**: 어트리뷰션 기여도 기반 한계 ROAS 균등화 — 정량 재배분 제안(증감액·증감률·예상 ROAS 개선폭).
- **이상탐지**: 매체별 일자 ROAS 시계열 z-score, |z|≥1.8 경고·≥2.4 심각 자동 플래그.
- **권한**: 오너/마케터/뷰어 3역할 권한 매트릭스, 액션 단계 권한 분기.
- **감사로그**: 모든 변경 액션의 행위자·역할·브랜드·액션·상세·시각 기록 + CSV 내보내기.
- 품질·성능 기준 (정량): 뷰 10종+ · 워크플로 4개+ · 실 알고리즘 2종+ · 외부 통합 3건+ · 신규 캡처 12장+ · 페이지 에러 0건.

> **v3 자체 산출물 핵심 8가지** (검수 측정값은 [`5_1_개발결과보고서_v3.md`](./5_1_개발결과보고서_v3.md) §6 참조):
> 1. **매체 OAuth 5종** mock 연동 관리 — 네이버·카카오·메타·구글·쿠팡 각 동의→토큰(Bearer 3600s)→연결상태·토큰·만료·동기화 시각 기록.
> 2. **자사몰 webhook 수신 시뮬레이션** — 카페24/Shopify 주문 이벤트 인입 로그 → 대시보드 실시간 반영(인입 매출·실시간 활동 패널).
> 3. **슬랙·이메일 자동 리포트 스케줄러** — 예약(채널·주기·시간) → 발송 실행 → 발송 이력 + 리포트 미리보기.
> 4. **RFM 자동 세그먼트 + 액션 추천** — R/F/M 5분위 5세그 + 세그먼트별 추천 액션·채널·CTA.
> 5. **예산 재배분 추천 알고리즘** — 어트리뷰션 기여도 기반 한계 ROAS 균등화, 매체별 증감·예상 ROAS 개선폭 실 계산 + 적용.
> 6. **이상탐지 알림** — 매체별 일자 ROAS z-score, |z|≥1.8/2.4 급락·급등 자동 플래그 + 대시보드 배너.
> 7. **다중 사용자 권한** — 오너/마케터/뷰어 역할 분기 + 권한 매트릭스 + 액션 단계 차단.
> 8. **감사로그/활동 이력** — 모든 변경 액션 누가·언제·무엇을 기록 + CSV 내보내기.
