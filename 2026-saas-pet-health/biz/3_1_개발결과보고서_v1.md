# 3_1_개발결과보고서 v1 — 반려동물 토탈 헬스케어 SaaS 「펫헬스북」

> 본 보고서는 [`3_과업지시서_v1.md`](./3_과업지시서_v1.md) §5 성과품 목록의 **검수 증거** 문서다. 자체 개발 분(클라이언트 데모 A)의 실 구동 결과를 PC·모바일 캡처와 함께 검증한다. 모든 스크린샷은 [`projects/pet-health/v1.html`](../projects/pet-health/v1.html)을 Playwright로 **실제 구동**해 얻은 것이다(목업·합성 없음).

## 2. 구현 범위

- **뷰 5종**: ①홈 대시보드 ②반려동물 프로필 ③건강기록 타임라인 ④접종·투약·구충 스케줄 ⑤예약·건강수첩.
- **다단계 워크플로(5단계)**: 반려동물 등록 → 건강기록 추가 → 스케줄 생성 → D-day 알림 → 병원예약/건강수첩 발행.
- **실동작 액션**: 사진 실 업로드·미리보기, 기록 추가·localStorage 영속, 주기 기반 다음일·D-day 자동 계산, 완료 처리 시 재계산 + 타임라인 자동 기록, 병원 예약 폼, jsPDF 건강수첩/진료요약 PDF 실 생성·다운로드(한글 깨짐 0).
- **데이터 시각화**: 체중 추이 라인차트(Chart.js).
- **반응형**: 모바일(≤767px) 바텀 탭바·바텀시트, PC(≥768px) 사이드바 256px. 인증 없이 즉시 사용(데모 데이터 시드: 강아지 콜라·고양이 나비).

## 3. 환경

| 항목 | 값 |
|:---|:---|
| OS | macOS (Darwin 24.6.0) |
| 런타임 | Node.js + Playwright(Chromium) |
| 구동 방식 | `file://` 단일 HTML (외부 서버 없음) |
| 라이브러리 | Chart.js 4.4.1(CDN), jsPDF 2.5.1(CDN) |
| 캡처 뷰포트 | PC 1280×800 / 모바일 390×844(isMobile, hasTouch) |
| 저장 | localStorage |

## 4. 실행/구동 방법

```bash
# 1) 데모 실행 (브라우저에서 직접 열기)
open 2026-saas-pet-health/projects/pet-health/v1.html

# 2) 캡처 재현 (Playwright)
cd 2026-saas-pet-health/projects/pet-health
npm i -D playwright && npx playwright install chromium
node capture.mjs        # PC → biz/captures/v1, 모바일 → biz/captures/mobile/v1
rm -rf node_modules package.json package-lock.json   # 캡처 후 정리
```
