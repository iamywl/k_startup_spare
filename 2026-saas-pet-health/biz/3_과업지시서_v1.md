# 3_과업지시서 v1 — 반려동물 토탈 헬스케어 SaaS 「펫헬스북」

> 외주 용역 기업에 시제품 개발의 일부를 위탁하기 위해 **발주자(동아리) 측이 작성**한 지시서. [`2_개발계획서.md`](./2_개발계획서.md) §3 WBS의 외주 구분 행을 본 문서 §1 과업의 범위로 옮겨 확정했다.

## 1. 과업의 개요

### 1.1 과업의 목적

1) **직접 목적** — 반려동물 헬스케어 SaaS 「펫헬스북」 시제품(v1)의 **클라이언트 데모와, 정식화에 필요한 백엔드·외부 연동 모듈**을 개발한다. 본 v1 위탁의 핵심은 ① 보호자용 5개 뷰 클라이언트 데모(자체 개발과 병행 검수), ② 클라우드 동기화 백엔드, ③ 푸시·알림톡 D-day 발송 모듈, ④ 동물병원 예약·펫보험 청구 연동 mock이다.
2) **사업 연결** — 본 과업의 산출물은 [`1_제안서.md`](./1_제안서.md)가 정의한 3대 페인포인트(깜빡함·흩어짐·못 씀)를 해결하는 제품의 기술 토대다. 특히 스케줄 자동 알림(외주 ②③)과 보험청구 연동(외주 ④)은 제안서 §2.4·§차별성의 핵심 가치를 구현한다.

### 1.2 과업의 범위

**In Scope (외주사 수행)**

| # | 범위 | 설명 |
|:---:|:---|:---|
| A | 클라이언트 데모(자체분 병행 검수) | 보호자용 5개 뷰(홈·반려동물·건강기록·스케줄·예약/건강수첩) 단일 HTML 데모. 사진 업로드·타임라인·스케줄 자동계산·PDF 발행 실동작 |
| B | 클라우드 동기화 백엔드 | 계정·다기기 동기화 API + DB 스키마. 키 부재 시 mock 모드 |
| C | 푸시·알림톡 D-day 발송 | 접종/투약/구충 D-day 도래 시 발송 로직. 실 발송 키 없으면 발송 로그 mock |
| D | 동물병원 예약 연계 mock | 병원 캘린더/슬롯 연동 인터페이스 + 예약 요청·확정 흐름 mock |
| E | 펫보험 청구 연동 mock | 건강기록→청구서 포맷 매핑 + 보험사 제출 인터페이스 mock |

**Out of Scope (외주사 미수행)**

- 실제 보험사·동물병원과의 상용 API 계약·인증 취득 (발주자/제휴 단계 별도 진행)
- 앱스토어·구글플레이 정식 배포 및 심사 대응
- 마케팅·콘텐츠 제작, 동물의학적 진단 알고리즘
- 개인정보 처리방침·약관의 법률 자문 (발주자 책임)

## 2. 과업의 세부 내용

### 2.1 클라이언트 데모(A) 기능 사양

| 뷰 | 사양 |
|:---|:---|
| 홈 대시보드 | 전 반려동물 통합 '다가오는 일정'(D-day 정렬), 지난 일정 카운트, 최근 체중, 체중 추이 라인차트 |
| 반려동물 프로필 | 사진 **실 업로드·미리보기**(FileReader→dataURL), 종·품종·성별·생일, 여러 마리 전환, 종별 기본 스케줄 자동 시드 |
| 건강기록 타임라인 | 체중·증상·진료·투약 4종 기록 추가, 시간순 타임라인, **localStorage 영속**(새로고침 유지) |
| 접종·투약·구충 스케줄 | 항목별 주기 입력 → `다음일=마지막+주기` **자동 계산**, D-day·알림 배지, 완료 처리 시 재계산 + 타임라인 자동 기록 |
| 예약·건강수첩 | 병원 예약 폼·내역, 건강수첩 PDF + 보험청구용 진료요약 PDF **실 생성·다운로드**(jsPDF, 한글 깨짐 0) |

### 2.2 UI/UX 제작 사양

- 디자인 톤: Apple × Toss(명료함·다정한 정확함). 브랜드=신뢰 인디고 + 건강 틸 포인트, WCAG AA(4.5:1) 이상.
- **반응형 필수**: `width=device-width` viewport. 모바일(≤767px)은 **바텀 탭바**, PC(≥768px)는 **사이드바 256px**. 좁은 화면 가로 overflow 0, 차트 고정높이, 모달은 모바일 바텀시트.
- 화면 흐름: 펫 등록 → 기록 추가 → 스케줄 생성 → D-day 알림 → 예약/PDF(5단계 워크플로).
- 인증: 시제품은 **로그인 없이** 즉시 사용(기본 데모 데이터 시드).

### 2.3 백엔드·연동 사양(B~E)

- **B 동기화**: REST/JSON API, DB 스키마(반려동물·기록·스케줄·예약). 키 부재 시 로컬 fallback 보장.
- **C 알림**: D-day(≤예: 3일) 도래 항목 추출 → 발송 큐. 카카오 알림톡 키 없으면 **발송 로그 파일/콘솔 mock**.
- **D 병원 예약**: 슬롯 조회·예약요청·확정 콜백 인터페이스. 실제 병원 연동 전까지 mock 응답.
- **E 보험 청구**: 진료·투약 기록 → 표준 청구 항목 매핑 → 제출 페이로드 생성. 보험사 키 없으면 mock 접수번호 반환.

### 2.4 사용 라이브러리·의존성

- 프런트: Vanilla JS(시제품), Chart.js(CDN), jsPDF(CDN). 외부 키·서버 의존 0(오프라인 시연 보장).
- 백엔드(정식 모듈): Node.js(ESM). API 키는 `.env` + `.gitignore`, 하드코딩 금지, 부재 시 mock.
- 검증 안 된 라이브러리 무분별 추가 금지, 미사용 의존성 제거.

### 2.5 품질·성능 기준 (정량)

| 항목 | 기준 |
|:---|:---|
| 스케줄 계산 정확도 | 주기 기반 다음일·D-day 계산 오차 0(완료 처리 후 재계산 포함) |
| 상태 지속성 | 새로고침 후 등록 데이터·기록 100% 유지 |
| PDF 한글 | 건강수첩·진료요약 PDF에서 한글 깨짐 0 |
| 반응형 | 360~430px 가로 overflow 0, 사이드바 하이재킹·잘림·빈 차트 0 |
| 오프라인 | 외부 서버 없이 단일 HTML 구동(CDN 외 의존 0) |
