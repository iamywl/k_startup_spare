## v2 신규/심화 산출물

### A. 실 알고리즘 4종 (입력 → 계산 → 권고)

1. **체중 추세 + 이상치 탐지** — 체중 시계열에 이동평균(3점)과 표준편차(±2σ) 밴드를 적용, 직전 윈도 기준 밴드를 벗어난 측정값을 이상치로 탐지하고 최소제곱 기울기로 추세를 라벨링(증가/감소/안정). (PC 08)
2. **BCS 체형 평가 + 권장 사료량** — 현재/이상 체중비로 BCS 1~9 구간화, `RER=70×kg^0.75`·`DER=RER×활동계수`로 일일 권장 칼로리 산출. (PC 07)
3. **건강 위험 스코어** — 연령·BCS·최근 증상(90일)·예방일정 지연을 가중 합산해 0~100 스코어·등급(양호/주의/경계/위험)·자동 권고를 생성. 멀티펫 비교. (PC 01·07)
4. **견종·묘종 프로토콜 엔진** — 종·품종별 기본 접종/구충 + 품종 위험(웰시코기 척추 IVDD, 푸들·포메 슬개골, 페르시안 신장 PKD 등) 맞춤 검진을 펫 등록 시 자동 시드·추천. (PC 02·03)

### B. 멀티 역할 RBAC (보호자·수의사·보험사)

- 역할 스위처로 즉시 전환, 역할별 내비게이션·뷰·포인트 컬러(인디고/틸/퍼플) 분기.
- **수의사**: 위험 환자 랭킹 → 환자 차트(보호자 공유 데이터 열람) → 진료기록 작성 → **보호자 타임라인·알림 자동 반영** → 예약 확정.
- **보험사**: 손해율 대시보드 → 청구 검토(승인/반려·보험금 산정) → 품종별 리스크 분석(언더라이팅 근거).

### C. 외부 통합 mock 4종

- **카카오 알림톡 로그** — D-day·예약 확정·진료 등록·청구 결과 시 발송 페이로드를 타임스탬프·수신자·메시지로 로그 기록(키 없이 mock). (PC 11)
- **보험청구 상태추적** — 진료 선택 → 청구서 자동 구성(보장률) → PDF → 제출(접수번호) → 심사중 자동 전이 → 보험사 승인/반려 → 보험금 산정. (PC 09·10·19)
- **CSV/JSON 입출력** — 전체 상태 JSON 백업·복원(가져오기), 건강기록 CSV(BOM·한글 정상) 내보내기. (PC 11)
- **웨어러블 활동량** — 7일치 걸음·활동시간 동기화 mock → 막대 차트. (PC 11)

## 2. 구현/제작 범위

- **뷰 16종**: 보호자 9(홈·반려동물·건강기록·스케줄·건강리포트·병원예약·보험청구·데이터연동·알림센터) + 수의사 4(진료대시보드·환자차트·진료기록작성·예약관리) + 보험사 3(청구대시보드·청구검토·리스크분석).
- **워크플로 3개**: ① 펫 등록 → 견종 프로토콜 스케줄 자동 시드 → D-day 알림톡, ② 수의사 진료기록 작성 → 보호자 타임라인 반영 → 알림, ③ 진료 선택 → 청구서 PDF → 제출 → 심사 → 승인/반려 → 보험금.
- **실 알고리즘 4종** + **외부 통합 mock 4종** (상단 신규/심화 산출물).
- **실동작 액션**: 사진 업로드·미리보기, localStorage 영속(역할·로그·청구 포함), 주기 자동 계산·완료 재계산, jsPDF PDF 3종, CSV/JSON 실 다운로드·업로드, 차트 4종(추세·활동량·리스크 도넛·체중).
- **반응형**: 모바일(≤767px) 오프캔버스 드로어(역할 전환) + 역할별 바텀탭, PC(≥768px) 사이드바 256px. 인증 없이 즉시 사용(데모 시드: 보호자A 콜라·나비, 보호자B 보리).

## 4. 실행/구동 방법

```bash
# 1) 데모 실행 (브라우저에서 직접 열기)
open 2026-saas-pet-health/projects/pet-health/v2.html

# 2) 캡처 재현 (Playwright)
cd 2026-saas-pet-health/projects/pet-health
npm i -D playwright && npx playwright install chromium
node capture.mjs        # PC → biz/captures/v2, 모바일 → biz/captures/mobile/v2
rm -rf node_modules package.json package-lock.json   # 캡처 후 정리
```
