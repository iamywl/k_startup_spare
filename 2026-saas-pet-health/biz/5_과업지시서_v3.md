# 5_과업지시서 v3 — 반려동물 토탈 헬스케어 SaaS 「펫헬스북」 (시리즈A 데모)

> 외주 용역 기업에 시제품 고도화(v3)의 일부를 위탁하기 위해 **발주자(동아리) 측이 작성**한 지시서.
> v2([`4_과업지시서_v2.md`](./4_과업지시서_v2.md)) 검수 후 추가/심화된 과업을 정의한다. **v1·v2를 수정하지 않고 신규 파일로 분리**하며, 앱은 `v1.html`·`v2.html`을 보존하고 `v3.html`을 신규 생성한다([§3.5](../../CLAUDE.md)).
> 작성 근거: [`2_개발계획서.md`](./2_개발계획서.md) §3 WBS의 v3 행을 본 문서 §1 과업의 범위로 옮겨 확정.

## 1. 과업의 개요

### 1.1 과업의 목적

1) **직접 목적** — v2의 3역할(보호자·수의사·보험사)·실 알고리즘 4종·외부통합 mock 4종 플랫폼을, **시리즈A 데모 수준의 헬스케어 인텔리전스 플랫폼**으로 도약시킨다. 본 v3 위탁의 핵심은 ① **고급 알고리즘 3종**(질병 위험 예측 — 로지스틱 회귀, 성장곡선 percentile, 약물 상호작용 체크), ② **외부 통합 확장 4종 mock**(병원 EMR 양방향 연동, 다보험사 청구 API, 웨어러블 OAuth 동기화, 외부 캘린더 동기화), ③ **병원 admin 역할 신설**(예약·진료 운영 대시보드·캘린더)로 4역할 RBAC 완성, ④ **KO/EN 다국어 토글**(글로벌 투자 데모 대응)이다.

2) **사업 연결** — 본 과업의 산출물은 [`1_제안서.md`](./1_제안서.md)가 정의한 ⓐ 차별성(데이터 출구 확장: 건강기록 → 위험 예측 → 보험·병원 연계), ⓑ 양면 네트워크(B2B2C: 보호자 ↔ 병원 ↔ 보험사)를 **실제 동작으로 입증**하며, 특히 제안서의 `## 차별화 기술의 구매동인 논증`에서 정의한 **must-have 구매동인(접종 누락 방지·청구 통합·세그먼트별 질병 조기발견)**을 데모에서 직접 시연한다. 병원 EMR·웨어러블 OAuth 연동은 제안서 §차별성 "입력 마찰 제거"(이탈 위협 ④의 극복 훅)를 구현한다.

## 2. 과업의 세부 내용

### 2.1 고급 알고리즘(A·B·C) 사양

| 알고리즘 | 입력 | 처리 | 출력 |
|:---|:---|:---|:---|
| 질병 위험 예측(로지스틱) | 연령(정규화)·BCS 이탈·최근증상(90일)·일정지연·견종 플래그 | 질환별 가중 `z=w·x+b` → `sigmoid(z)` 확률, 내림차순 정렬 | 질환별 발병 확률(%)·최우선 위험·검진 권고 |
| 성장곡선 percentile | 체중 시계열·생년월일·품종 표준(평균/표준편차) | 월령 성숙 시그모이드로 기대치 산출, z-score→정규분포 CDF | 측정별 percentile·실측/기대 곡선·표준 판정 |
| 약물 상호작용 | 현재 투약 약물 목록 | 약물→클래스 매핑, 룰 기반 페어 충돌 탐지 | 충돌 페어·심각도(major/moderate/minor)·설명 |

### 2.2 역할 분기(D) 사양 — 병원 admin 신설

- **병원 admin(테마: 주황)**: 운영 대시보드(미확정/확정 예약·환자 수·기록 매출·EMR 연동 상태·위험 상위 환자) · 예약·진료 캘린더(예약 확정·캘린더 동기화) · 환자 차트(수의사 화면 공유). 3뷰.
- 4역할 RBAC: 보호자(인디고)·수의사(틸)·보험사(퍼플)·병원 admin(주황). 역할 전환 시 내비게이션·뷰·테마·바텀탭 즉시 분기.

### 2.3 외부 통합(E·F) 사양

- **EMR 양방향**: 병원 admin이 EMR 연결 → 진료기록 1건 pull → 보호자 타임라인·수의사 기록에 동시 반영, 연결 상태·최근 동기화·가져온 기록 수 표시, 발송 로그 기록.
- **다보험사 청구 API**: 청구 작성 시 3개 보험사(`state.insurers`) 중 선택 → 보험사 API(mock) 라우팅·접수번호 발급, 보험사 워크스페이스에서 심사.
- **웨어러블 OAuth**: OAuth 2.0 인가 흐름 mock(access_token 발급 로그) → 인가 후에만 활동량 동기화 허용(미인가 시 안내).
- **캘린더 동기화**: 병원 admin이 외부 캘린더(구글) 연결 → 예약·접종 일정 push 동기화, 동기화 일정 수·최근 시각 표시.

### 2.4 i18n(G) 사양

- 언어 토글(한국어/EN)을 사이드바·드로어·모바일 상단바에 노출. UI 셸·내비게이션·역할·섹션·핵심 라벨을 사전(`I18N`) 기반 전환.
- 언어 상태는 localStorage 지속(새로고침 후 유지). 전환 시 즉시 리렌더.

### 2.5 사용 라이브러리·의존성

- 프런트: Vanilla JS(시제품), Chart.js(CDN), jsPDF(CDN). 외부 키·서버 의존 0(오프라인 시연 보장).
- 모든 외부 통합(EMR·보험사·웨어러블·캘린더·알림톡)은 **키 부재 시 mock**으로 동작. 하드코딩 금지, 미사용 의존성 제거.

### 2.6 품질·성능 기준 (정량)

| 항목 | 기준 |
|:---|:---|
| 로지스틱 예측 | 질환별 `sigmoid(w·x+b)` 재현성 100%, 확률 내림차순 정렬·최우선 위험 표기 |
| 성장곡선 | z-score→percentile(정규 CDF) 산출, 실측/기대 곡선 렌더, 표준 미만/초과 판정 정확 |

| 역할 분기 | 4역할 전환 시 내비·뷰·테마·바텀탭 즉시 분기, 데이터 일관성 유지 |
| EMR 양방향 | 연동 시 진료기록이 보호자·수의사 양쪽에 동시 반영, 로그 기록 |
| OAuth/캘린더 | 인가 토큰 발급 로그·미인가 시 동기화 차단, 캘린더 일정 수 정합 |
| i18n | KO/EN 토글 시 셸·내비·라벨 전환, 새로고침 후 언어 유지 |
| 상태 지속성 | 새로고침 후 펫·기록·청구·알림·로그·역할·언어·연동상태 100% 유지(localStorage) |
| 반응형 | 360~430px 가로 overflow 0, 콘텐츠 클리핑·사이드바 하이재킹·빈 차트 0, 표 가로 스크롤 정상 |
| 오프라인 | 외부 서버·키 없이 단일 HTML 구동(CDN 외 의존 0) |
