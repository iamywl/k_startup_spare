## 1. 과업의 개요

### 과업의 목적

1) **직접 목적** — v2(멀티테넌트·낙상 로지스틱·인력배치 최적화·공단 EDI) 베타 수준을 **시리즈A 투자 데모 가능 수준**으로 도약시킨다. 구체적으로 ① **고급 분석 알고리즘 4종 신규**(입소·수요 시계열 예측 / 케어 질 지표 다기관 벤치마킹 / 약물 상호작용(DDI) 안전 점검 / 재원일수·이탈 예측), ② **외부 시스템 통합 3건 신규(Mock, 키 0)**(민간보험사 정산 API + webhook 콜백 / IoT·웨어러블 낙상센서 실시간 스트림 / 전자처방·EMR 연계), ③ **전 화면 KO/EN i18n**, ④ 역할·뷰 확장(v2 23뷰 → v3 25뷰)을 v2 단일 HTML 위에 추가 구현한다.
2) **사업 연결** — 본 도약 산출물은 [`1_제안서.md`](./1_제안서.md) §Scale-up(다기관·글로벌 확장)·§수익모델(데이터 기반 부가서비스)·§차별화 기술의 구매동인 논증(예측·자동화가 must-have 동인)에서 약속한 로드맵을 실증한다. v1·v2와 동일하게 인터넷·서버 없이 단독 구동되어야 한다(오프라인 시연 보장).

### 과업의 범위

1) **수행 업무(In Scope)** — v2 범위 위에 **추가**되는 과업만 기술한다.
   - **입소·수요 예측(Holt 이중 지수평활)**: 과거 12개월 재원 시계열에 level+trend 평활을 적용해 향후 3·6·9개월 재원 인원·정원 가동률·예상 월매출을 예측. 95% 예측구간(잔차 표준편차 기반)·SVG 라인차트·CSV 내보내기.
   - **케어 질 지표 벤치마킹(다기관 백분위)**: 법인 내 전 시설의 KPI(방문 완수율·고위험 비율·평균 위험·인력 부하 편차·1인 청구)를 동점 보정 백분위(midrank)로 비교하고, 외부 업계 벤치마크 대비 갭과 우수/보통/개선필요를 자동 판정.
   - **약물 상호작용(DDI) 안전 점검**: 어르신 질환에서 대표 처방 약물군을 추정하고 알려진 상호작용 규칙쌍(8쌍)과 매칭해 중대/주의 등급·임상 메모를 제시. 어르신별 상세 모달·약물점검 PDF.
   - **재원일수·이탈 예측(생존분석 근사)**: 질환·등급·낙상/입원 이력·체중감소를 위험인자로 한 월별 이탈 위험률(hazard)을 누적해 6개월 잔존확률·기대 재원개월(기대값)을 산출. 위험 밴드·잔존 곡선·권고.
   - **민간보험사 정산 연계(외부통합 ①)**: 간병·요양 민간보험 청구분을 보험사 API로 일괄 전송(Mock)하고, 정산 webhook 콜백을 수신해 접수번호·정산액·입금 예정을 확정.
   - **IoT·웨어러블 낙상센서(외부통합 ②)**: 거치형 레이더·웨어러블 밴드·출입센서·SOS 펜던트의 낙상·심박·활동 이벤트를 실시간 스트림(Mock)으로 수신. 낙상 이벤트는 알림·감사 로그로 자동 연계.
   - **전자처방·EMR 연계(외부통합 ③)**: 협력 의원/병원 EMR에서 처방·진단(ICD) 코드를 가져오고(Mock), 처방을 약물안전 점검·케어플랜에 반영.
   - **전 화면 KO/EN i18n**: 메뉴 라벨·뷰 제목·신규 7뷰 본문을 언어 토글로 동적 전환.
   - v2의 23뷰를 **25뷰**로 확장(신규 7뷰), 워크플로 유지, 신규 캡처(PC·모바일 분리) 및 검수 협조.
2) **Out of Scope(범위 외)** — v1·v2와 동일하게 다음은 본 사이클 대상이 아니다.
   - 실 백엔드 서버·DB·실 시계열 학습 파이프라인 구축 (예측은 클라이언트 결정적 시뮬레이션)
   - 보험사·EMR·IoT **실** 연동·실 OAuth/SSO·실 처방전 전송 (Mock 스트림·Mock 콜백·Mock 토큰으로 대체)
   - 약물 상호작용 규칙셋의 **임상 검증**(데모 규칙셋 — 약사·의사 검토 전)
   - 회원가입·로그인 정식 인증(역할 전환으로 우회, `CLAUDE.md` §3.4)
   - 모바일 네이티브 앱 패키징, ISMS·의료기기 인증, 개인정보 실 운영 처리·법적 자문

## 2. 과업의 세부 내용

### 2.1 시제품 사양
- **구동 형태**: 단일 HTML 파일(`v3.html`), 빌드리스·오프라인 단독 구동. `v1.html`·`v2.html`은 보존(덮어쓰기 금지, `CLAUDE.md` §3.5). `<meta name="viewport" content="width=device-width, initial-scale=1">` 필수.
- **상태/멀티테넌트**: v2의 `_tenants`(3 시설)·`_activeTenant` 컨텍스트·테넌트 프록시(`installTenantProxies`)를 계승. localStorage 키 `doltbom-v3`(v2와 분리).
- **알고리즘 정확성**:
  - 수요 예측: Holt(α=0.5, β=0.3) level+trend. 향후 h개월 `ŷ = level + h·trend`, 95% 구간 `±1.96·σ·√h`(σ=잔차 std). 시계열은 테넌트 id·현 재원수 기반 **결정적 합성**(랜덤 없음 → 재현 가능), 최신값은 실제 현 재원수에 고정.
  - 벤치마킹: 백분위 = `(below + (equal−1)/2)/(n−1)×100`(midrank, 동점 보정). higherBetter=false 지표는 `100−pct`.
  - DDI: 질환→약물 추정(`DISEASE_DRUGS`) + 다약제(meds≥8) 시 항응고제 가정 → 규칙쌍 매칭. 중대/주의 분류.
  - 이탈: 월 hazard = 기본 + 등급·입원·낙상·체중감소·중증질환 가산(상한 0.6). 6개월 잔존 = Π(1−hazard), 기대 재원개월 = Σ 월별 잔존확률.
- **외부통합 Mock**: 키 없이 동작. 보험사 정산 접수번호 `INS-YYYYMM-XXXXX`, IoT 스트림은 setInterval 기반(뷰 이탈 시 자동 정지), EMR 처방은 샘플 ICD/처방 반환.

### 2.2 사용자 UI/UX 제작 사양
- 디자인 톤: [`design-guide/디자인_애플_토스.md`](./design-guide/디자인_애플_토스.md) 준수. 모바일=[`디자인_모바일.md`], 데스크톱=[`디자인_PC.md`].
- 신규 7뷰(수요예측·벤치마킹·약물안전·이탈예측·IoT·EMR·보험사정산) 모두 좁은 화면(390px)에서 가로 overflow 0. 예측 라인차트는 SVG `viewBox`로 폭 100% 반응. 표(벤치마킹·정산)는 `.tablewrap` 가로 스크롤로 가두고 라벨 셀은 `white-space:nowrap`로 글자 세로 붕괴 방지.
- IoT 스트림 로그·정산 webhook 로그는 monospace 코드 블록 내 `overflow-y:auto`로 가둔다(페이지 overflow 금지). 사이드바는 모바일 바텀 탭바 + 더보기 드로어(25뷰 스크롤). 출고 전 [`반응형_검수_필수.md`] 통과(390/1280 눈검수).

### 2.3 사용 부품·라이브러리·외부 의존성
- jsPDF 2.5.1 (CDN). 부재 시 mock 폴백. 약물점검 등 PDF 한글 깨짐 방지(캔버스 렌더).
- 예측·벤치마킹·DDI·이탈·SVG 차트 모두 외부 라이브러리 없이 자체 JS/SVG/CSS 구현.
- 보험사·IoT·EMR·공단·카카오는 Mock(키 없이 로그/토큰/스트림). API 키 하드코딩 금지(`CLAUDE.md` §3.1).
- 캡처: Playwright Chromium(설치 후 캡처 종료 시 `node_modules`·`package*` 삭제).

### 2.4 품질·성능 기준 (정량)
- 구현 기능 수: **180개 이상**(코드 하단 `FEATURES[]`·`window.__FEATURES.length` 검증, 실측 186).
- 화면/뷰: **25종 이상**(admin 25 / social 11 / care 7 / guardian 4, 합집합 25). 다단계 워크플로 3개 이상 유지.
- **신규 고급 알고리즘 3종 이상**: 수요 예측(Holt) + 케어 질 벤치마킹(백분위) + DDI 점검 + 이탈 예측(생존분석) = **4종**.
- **외부 시스템 통합 3건 이상 신규**: 보험사 정산(+webhook) + IoT·웨어러블 스트림 + 전자처방·EMR.
- **KO/EN i18n**: 메뉴·뷰 제목·신규 7뷰 본문 전환.
- 상태 지속성: 새로고침 후 입력·테넌트·역할·뷰 컨텍스트 100% 유지.
- 반응형: 390px·1280px 가로 overflow 0(전 25뷰), 레이아웃 깨짐 0. 콘솔 pageerror 0.
