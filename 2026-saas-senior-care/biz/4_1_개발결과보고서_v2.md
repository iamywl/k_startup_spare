## v2 신규/심화 산출물

### 신규-A. 법인 본부 콘솔 (멀티테넌트)
3개 시설(행복재가복지센터·늘봄요양원·한울주야간보호센터)을 하나의 법인으로 묶어 KPI 합산(운영 3시설·재원 12명·월 공단청구 합계 ₩11,824,350·고위험 7명)과 시설별 비교 막대·운영 현황표를 제공한다. 표의 시설 행을 클릭하면 해당 테넌트로 컨텍스트가 전환되어 이후 모든 화면이 그 시설 데이터로 분기한다.

### 신규-B. 낙상 위험 예측 — 로지스틱 회귀 (설명가능 AI)
규칙 가중치를 넘어, 9개 위험요인(고령·등급·낙상이력·입원이력·다약제·인지저하·보행장애·체중감소·방문주의신호)을 `p = sigmoid(intercept + Σ wᵢ·featureᵢ)`로 결합해 3개월 내 낙상 확률을 추정한다. 어르신별로 **확률에 가장 크게 기여한 상위 요인**을 함께 노출(예: 이말순 98% · ▲최근 낙상 이력 ▲최근 입원 이력 ▲인지저하 동반)하여 임상 판단 근거를 제공한다.

### 신규-C. 인력배치 최적화 (휴리스틱)
직원-어르신 배정을 비용함수 = 스킬 패널티(간호 필요 케이스↔간호사) + 이동거리비용(haversine 근사) + 부하비용으로 정의하고, 무거운 케이스 우선(LPT) 그리디로 최소비용 직원에 배정한다. 적용 전/후 직원 부하 표준편차 개선폭(예: 6.3→5.7)을 제시한다.

### 신규-D. 수가·매출 시뮬레이션
정원 가동률·수가 인상률·주야간보호 비중 슬라이더를 조정하면 월·연 매출이 실시간 투영되고(공격 시나리오 가동 100%·+4% → 연 ₩625,305,296·기준 대비 +940%), 보수/기본/공격 3시나리오를 막대로 비교한다.

### 신규-E. 공단 EDI 전자청구 (외부통합 ①)
국민건강보험공단 장기요양 포털 OAuth 2.0 인증(Mock 토큰)을 거쳐, 급여비용을 EDI 전문(XML envelope)으로 변환·미리보기 후 전송하고 접수번호(`LTC-202606-A611T` 형식)·접수증 PDF를 수신한다. 키 없이 Mock으로 동작하며 미인증 시 전송 버튼이 비활성된다.

### 신규-F. CSV 외부 입출력 (외부통합 ②) · 감사 로그
타 시스템 청구 CSV(성명·등급·서비스코드·횟수)를 파일 업로드 또는 샘플로 가져와 파싱·미리보기한다. 모든 주요 액션(EDI 전송·CSV 가져오기·OAuth 인증·인력배치 적용)은 감사 로그에 역할·테넌트와 함께 기록되며 CSV로 내보낼 수 있다.

## 3. 환경
- OS: macOS (Darwin 24.6.0)
- 구동: 브라우저에서 `v2.html` 직접 열기(file://), 빌드·서버 불필요
- 언어/런타임: 단일 HTML + Vanilla JS(ES2020), 외부 차트/알고리즘 라이브러리 없음
- 외부 의존성: jsPDF 2.5.1(CDN, 부재 시 mock 폴백)
- 캡처: Playwright Chromium, PC 뷰포트 1280×800 / 모바일 390×844(deviceScaleFactor 2, isMobile)

## 4. 실행/구동 방법
1. `projects/senior-care/v2.html` 을 브라우저로 연다(더블클릭 또는 `file://`).
2. 최초 진입 시 데모 데이터(3 테넌트·어르신·직원·방문·청구) 자동 시드.
3. 상단/드로어의 역할 전환으로 관리자·사회복지사·요양보호사·보호자 화면 확인.
4. 관리자 메뉴에서 법인 본부 콘솔·위험 예측·인력배치·수가 시뮬·EDI 청구·감사 로그 진입.
5. 새로고침 후에도 입력·테넌트 컨텍스트 유지(localStorage).
