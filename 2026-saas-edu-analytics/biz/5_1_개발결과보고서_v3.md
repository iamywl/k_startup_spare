## v3 신규/심화 산출물 (핵심 8가지 캡처 입증)

### 1. 예측 모델 (학기말 성취 예측·중도이탈/위험 예측)
- **학기말 성취 예측**: 각 학생의 회차 점수를 `[bias, 회차index, 출석률, 과제완수율, 취약개념수]` 특성으로 두고 **능선회귀(Ridge, `w=(XᵀX+λI)⁻¹Xᵀy`, λ=1, 가우스 소거 닫힌해)** 로 적합해 다음 회차 점수를 예측한다. 잔차 표준편차로 **95% 신뢰구간**(±1.96σ)을 함께 산출한다(02 — "학기말 예측"·"95% 신뢰구간" 열).
- **중도이탈/위험 예측**: 동일 특성(추세 하락·저평균·취약개념·낮은 출석률·낮은 과제완수율)을 **로지스틱 함수**로 0~100% 위험확률로 변환·등급화(고위험≥60%·주의≥35%·안정). 각 학생의 **특성 기여도**(예: `취약개념 +1.12 · 저평균 +0.35`)를 100% 노출한다(02 — "위험확률 상위 — 특성 기여도"). 블랙박스가 아닌 설명 가능한 예측이다.

### 2. 학교/교육청 다계층 통합 대시보드 (다계층 테넌트)
교육지원청(`한빛교육지원청`, 상위 테넌트) > 학교 2개(`한빛초`·`샛별초`) > 학급 5개 3계층. 교육청 관리자는 **관할 학교 롤업**(학교별 평균·예측 위험군 비중·형평성 지니)을 보고, 학교 관리자는 학급 롤업을 본다. 상위 테넌트는 하위 **집계·가명만** 조회하며 개별 학생 실명·식별정보는 노출되지 않는다(01 — "개별 학생 식별정보 비노출 — 집계만"). 다계층 격리 검증에 "교육청은 집계·가명만 — 개별 식별정보 0건", "타 교육지원청 district_id 격리 0건"이 표시된다.

### 3. 개입 추천 엔진 (취약개념→맞춤 학습경로)
개념 선수관계 그래프(`약분→통분→분수의덧셈/뺄셈→대분수`)에서 학생의 취약개념을 시작점으로 **선수 결손까지 역추적**(`prereqClosure`)해, 결손 선수개념이 있으면 보충 단계를 끼워 **진단→선수보충→본개념→심화** 단계별 학습경로를 자동 생성한다. 학생별 경로·예상 소요시간·자료(TASK_BANK)가 매핑되고 "이 경로 배포"로 학생에게 배포된다(04). 위험군 일괄 경로 생성·배포도 워크플로 A에서 입증된다(11).

### 4. 다중 소스 LMS/AIDT 학습로그 연동 mock (멀티 커넥터)
**AIDT·LMS·CSV 3개 커넥터**가 각자 다른 원시 스키마(AIDT `student_ext_id/concept/correct`, LMS `userId/skill/pass`, CSV `외부ID,개념,정답(O/X)`)를 **표준 학습로그로 정규화**해 인입한다. 외부ID↔내부 학생 매핑, 미등록 외부ID(AIDT-9999) 매핑 실패(격리), 소스 태깅이 파이프라인 로그에 기록된다(05). 정규화 결과는 CSV로 내보낼 수 있다(06, BOM 포함 한글 깨짐 방지). 키 부재 시 mock 응답으로 동작한다.

### 5. 학부모 앱 (알림톡 캠페인·상담 예약)
**세그먼트**(전체·예측 위험군·취약개념 보유)를 대상으로 알림톡 캠페인을 일괄 발송하고 발송 통계(발송 성공·미동의 제외·SMS 폴백)를 집계한다. 미동의/철회 학부모는 자동 제외된다(08). **상담 예약 슬롯**(교사 가용 시간)을 학부모가 선택·확정하며 확정 알림톡(mock)이 발송된다. 학부모 모바일 앱에서도 예약이 가능하다(17).

### 6. 교사 협업 (공동 루브릭·평가 공유)
교사가 **공동 루브릭**(평가 기준표)을 작성해 학교 교사 라이브러리에 공유하면, 다른 교사가 **채택(복제)** 할 때마다 확산 카운트가 올라간다. 학급 간 평가 결과(평균)도 공유되어 평가 기준을 정렬한다(07).

### 7. 개인정보 거버넌스 고도화 (접근감사·보존정책·동의 철회)
모든 민감정보 접근(주체·역할·행위·대상·시각)이 **감사 로그**에 자동 기록된다. **보존정책**은 보존기간(예: 졸업생 3년·감사로그 1년) 만료 도래 항목을 식별해 "만료 데이터 파기 처리"로 파기한다. **동의 일괄 철회**는 세그먼트(현재 학급·동의자)를 골라 일괄 철회하면 분석/발송에서 즉시 제외되고 감사 로그에 기록된다(09→10).

### 8. 성취 격차·형평성 분석 리포트
성취 분포의 **지니계수**(정렬·누적 기반 산출), **사분위(Q1~Q4) 학생 수**, **로렌츠 곡선**(완전 평등선 대비 누적 성취), 상하위 25% 격차, **학급 간 격차지수**(최고−최저 학급 평균)를 산출하고 지니계수 기반 권고를 제시한다(03). 교육청 관점의 학교 간 형평성 진단·권고는 워크플로 D에서 입증된다(12).

## 2. 구현/제작 범위

- **단일 자체완결 HTML 앱**(`v3.html`): 빌드·서버·번들러 없이 단독 동작. Tailwind/Chart.js는 CDN, 핵심 로직(다계층 테넌트·예측 알고리즘·선수개념 그래프·멀티 커넥터·캠페인·협업·거버넌스·형평성)은 내장 JS로 오프라인 동작.
- **뷰 10종(데스크톱) + 4탭(모바일)**, 모든 뷰가 인터랙션 가능(예측 실행·경로 생성/배포·멀티소스 인입·캠페인 발송·예약·루브릭 채택·동의 철회·파기·역할/스코프 전환).
- **실 알고리즘 2종+**: ① 능선회귀(L2 정규화 닫힌해, 가우스 소거) 학기말 성취 예측 + 신뢰구간, ② 로지스틱 위험확률(특성 기여도 분해). 추가로 지니계수·로렌츠 누적·선수개념 그래프 역추적.
- **다계층 테넌트·4역할 RBAC**: 교육청>학교>학급 롤업, 상위는 집계·가명만, 역할(교육청·학교·교사·학부모)별 화면·데이터 분기, 권한 경계 위반 시 차단.
- **외부 통합 3건+(mock 실 흐름)**: AIDT·LMS·CSV 3소스 정규화 인입(스키마별 매핑) + 알림톡 캠페인 발송 로그 — 추가로 CSV 파일 출력.
- **개인정보 거버넌스**: 모든 민감정보 접근 감사 로그 자동 기록, 보존정책 만료 파기, 동의 일괄 철회·발송/분석 제외.
- **상태 지속성**: 전체 상태(`classlens_v3_state`)를 localStorage에 직렬화, 누락 필드 마이그레이션. 새로고침 후 인입분·예측·가명처리 포함 복원(14).
- **인증**: [`../../CLAUDE.md`](../../CLAUDE.md) §3.4에 따라 로그인 불요, 역할 셀렉터로 자동 통과.

## 4. 실행/구동 방법

1. `projects/classlens-poc/v3.html` 을 브라우저로 연다(더블클릭 또는 `file://`).
2. 최초 진입 시 교육청 1·학교 2·학급 5·학생 34명·상담 슬롯·보존정책 시드가 자동 주입된다.

## 데이터 정직성 선언

본 보고서의 모든 화면·수치는 실 구동된 `v3.html`을 Playwright(Chromium, 데스크톱 1440×900·모바일 390×844)로 캡처한 결과이며, 목업·합성 이미지는 사용하지 않았다. 교육청·학교·학급·학생·정답률·출석률·과제완수율·동의·알림·예약 로그는 데모용 시드/시뮬레이션 값으로 실제 개인정보가 아니다. AIDT·LMS 연동·카카오 알림톡은 외부 키 없이 동작하는 mock이며, mock임을 화면·로그에 명시했다. 예측(능선회귀·로지스틱)·형평성(지니)·선수개념 그래프는 앱 내장 결정적 실 계산 결과다. 외부 통계 인용은 본 결과보고서에 포함되지 않는다(제안서 §참고문헌 참조).
