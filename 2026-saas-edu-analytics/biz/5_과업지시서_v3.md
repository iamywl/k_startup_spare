# 과업지시서 v3 — 클래스렌즈 학습분석 SaaS 상용·심화(Series A 데모)

> v2([`4_과업지시서_v2.md`](./4_과업지시서_v2.md)) 검수 후 추가·심화된 과업. v1·v2를 수정하지 않고 신규 파일로 분리한다.
> 범위는 [`2_개발계획서.md`](./2_개발계획서.md) §WBS 의 v3 행(16~23)에서 옮겨 확정한다.

## 1. 과업의 개요

### 1.1 과업의 목적

1) **직접 목적**: v2 베타를 **지역(교육청) 도입·Series A 데모 가능한 상용·심화 제품**으로 끌어올린다. 실 예측 알고리즘(학기말 성취 예측·중도이탈/위험 확률), 학교/교육청 다계층 통합 대시보드(다계층 테넌트), 개입 추천 엔진(취약개념→맞춤 학습경로), 다중 소스 LMS/AIDT 학습로그 연동(멀티 커넥터), 학부모 앱(알림톡 캠페인·상담 예약), 교사 협업(공동 루브릭·평가 공유), 개인정보 거버넌스 고도화(접근감사·보존정책·동의 철회), 성취 격차·형평성 분석 리포트를 구현한다.
2) **사업 연결**: [`1_제안서.md`](./1_제안서.md) §8 로드맵의 상용/심화(v3) 단계 산출물(지역 도입·Series A 데모)에 해당한다. §10.1 멀티테넌트 아키텍처를 **교육청>학교>학급 다계층 롤업**으로 확장 입증하고, §10.2 개인정보 보호 설계를 **접근감사·보존정책·동의 철회**의 거버넌스 실 구현으로 입증한다. §5.3 데이터 락인 해자를 멀티소스 인입·교사 협업으로 강화한다.

## 2. 과업의 세부 내용

### 2.1 기능 사양

| 기능 | 세부 사양 |
|:---|:---|
| 예측 모델 | (a) **학기말 성취 예측** — 회차 점수에 시간 특성 + 출석률·과제완수율·취약개념수 특성을 능선회귀(L2 정규화, 닫힌해)로 적합해 다음 회차 점수 예측 + 신뢰구간. (b) **중도이탈/위험 예측** — 동일 특성을 로지스틱 가중합으로 0~100% 위험확률 변환·등급화. 두 모델 모두 특성 기여도(근거) 노출(블랙박스 금지) |
| 다계층 통합 대시보드 | 교육청(상위 테넌트) > 학교(테넌트) > 학급 3계층. 교육청 관리자는 관할 학교 롤업(학교별 평균·경보·예측 위험), 학교 관리자는 학급 롤업. 상위는 하위 가명·집계만, 개별 식별정보 차단 |
| 개입 추천 엔진 | 개념 선수관계 그래프(예: 약분→통분→분수의덧셈)에서 취약개념의 **선수 결손까지 역추적**해 단계별 학습경로(진단→선수보충→본개념→심화) 자동 생성. 학생별 경로·예상 소요·자료 매핑 |
| 멀티 소스 연동 mock | AIDT·LMS(무들/구글클래스룸 류)·CSV 3개 커넥터. 소스별 스키마 매핑(필드 정규화), 외부ID↔내부 학생 매핑, 인입 이력·소스 태깅. 키 부재 시 mock 응답 |
| 학부모 앱(캠페인·예약) | 세그먼트(위험군/미동의 제외/특정 학급) 대상 알림톡 캠페인 일괄 발송·발송 통계. 상담 예약 슬롯(교사 가용시간) 선택·확정, 예약 현황 |
| 교사 협업 | 공동 루브릭(평가 기준표) 작성·공유 라이브러리, 다른 교사가 채택(복제)·확산 카운트, 평가 결과 학급 간 공유 |
| 개인정보 거버넌스 | 모든 민감 데이터 접근의 **감사 로그**(주체·대상·시각·행위), 데이터 **보존정책**(보존기간·만료 도래 항목·파기 처리), 동의 **일괄 철회**(세그먼트 철회 시 분석/발송 즉시 제외·감사기록) |
| 형평성 분석 | 성취 분포의 **지니계수**·사분위(Q1~Q4)·상하위 격차, 학급/학교 간 **격차지수**, 격차 완화 권고. 형평성 리포트 화면 |

### 2.2 UI/UX 사양

- 화면/뷰 **10종 이상**: (데스크톱) 교육청 통합 대시보드·학교 대시보드·예측 모델·개입 추천·멀티소스 연동·교사 협업·형평성 분석·개인정보 거버넌스·워크플로 + (모바일 학부모) 홈·예약·캠페인 수신.
- 워크플로 **4개 이상**: ① 예측→개입추천→경로배포 ② 멀티소스 인입→정규화→예측 갱신 ③ 동의철회/보존만료→감사기록→발송제외 ④ 교육청 롤업→형평성 진단→학교 권고.
- 반응형: 교사/관리자 데스크톱 + 학부모 모바일.
- 역할 **4종**: 교육청 관리자·학교 관리자·교사·학부모.

### 2.3 사용 부품·라이브러리·의존성

- v2 스택 유지 + 예측 알고리즘(능선회귀 닫힌해·로지스틱)·선수개념 그래프·형평성 통계(지니) 구현. 멀티 커넥터·감사로그·보존정책·캠페인·예약·루브릭 라이브러리.
- AIDT/LMS/알림톡은 mock 인터페이스(키 부재 시 로그·시드). API 키 하드코딩 금지([`../../CLAUDE.md`](../../CLAUDE.md) §3.1).
- 단일 자체완결 HTML(`projects/classlens-poc/v3.html`), 빌드 불요, 키 없이 구동.
