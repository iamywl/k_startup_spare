# 과업지시서 v1 — 클래스렌즈 학습분석 SaaS 시제품(PoC)

> 발주자(클래스렌즈 사업팀)가 외주 용역사에 v1 시제품 제작을 위탁할 때 작성하는 지시서.
> 범위는 [`2_개발계획서.md`](./2_개발계획서.md) §WBS 의 외주 구분 행에서 옮겨 확정한다.

---

## 1. 과업의 개요

### 1.1 과업의 목적

1) **직접 목적**: 초·중등 교사·학교용 학습분석 SaaS 「클래스렌즈」의 v1 시제품(PoC) 1식을 제작한다. 교사가 평가·과제 결과를 입력하면 **취약개념을 진단하고, 맞춤 과제를 추천하며, 학부모 리포트를 생성**하는 **평가 → 분석 → 피드백** 단일 워크플로가 실제 동작하는 웹 애플리케이션을 납품한다.
2) **사업 연결**: 본 과업은 [`1_제안서.md`](./1_제안서.md) §2 솔루션의 핵심 가치명제("AIDT가 못 흘려보내는 학습데이터를 교사 워크플로로 끌어내는 분석 레이어")를 동작 PoC로 입증하기 위한 것이다. 제안서 §8 로드맵의 MVP(v1) 단계 산출물에 해당한다.

### 1.2 과업의 범위

**In Scope — 외주사가 수행할 업무**

| # | 업무 항목 |
|:---:|:---|
| 1 | 평가·과제 결과 입력 화면(수기 입력 + CSV 업로드 인입) 구현 |
| 2 | 학생·학급별 **성취 대시보드**(점수·추세 차트) 구현 |
| 3 | 단원·개념 단위 **취약개념 히트맵**(정오답 패턴 시각화) 구현 |
| 4 | 취약개념 기반 **맞춤 과제 추천** 생성·배포 화면 구현 |
| 5 | 성취추세·취약개념을 담은 **학부모 리포트** 생성(화면 + PDF/이미지 저장) |
| 6 | **평가 → 분석 → 피드백** 다단계 워크플로(입력→진단→추천→리포트) 연결 |
| 7 | 상태 지속성(localStorage/IndexedDB 등으로 새로고침 후 유지) |
| 8 | 데모 시드 데이터(가상 학급·학생·평가) 주입 및 키 없이 동작하는 mock fallback |

**Out of Scope — 외주사가 수행하지 않는 업무 (v2 이후 또는 발주자 자체)**

| # | 제외 항목 | 사유 |
|:---:|:---|:---|
| 1 | 다중 학급/다중 교사 멀티테넌트 권한 분기 | v2 범위([`4_과업지시서_v2.md`](./4_과업지시서_v2.md)) |
| 2 | 학습분석 고급 알고리즘(군집·위험학생 조기경보) | v2 범위 |
| 3 | AIDT/LMS 실 API 연동 | v3(실 연동은 외부 승인 리드타임) |
| 4 | 카카오 알림톡 실 발송(비즈채널 심사) | v2 mock → v3 실발송 |
| 5 | 개인정보 보호 인증(ISMS-P 등) 취득 | v3, 발주자 주관 |
| 6 | 팀·서명·계약 등 행정 양식 | 발주자 영역 |

## 2. 과업의 세부 내용

### 2.1 기능 사양

| 기능 | 세부 사양 |
|:---|:---|
| 평가/과제 입력 | 학생명·문항·정오답·점수 수기 입력 + CSV 업로드(컬럼 자동 매핑). 입력 즉시 대시보드 반영 |
| 성취 대시보드 | 학생별·학급 평균 점수, 시간순 성취 추세 라인 차트, 취약 학생 하이라이트 |
| 취약개념 히트맵 | 행=학생/열=개념(또는 단원), 셀=정답률. 학급 공통 약점 색강도 시각화 |
| 맞춤 과제 추천 | 취약개념 정답률 임계값 이하 시 해당 개념 연습 과제 자동 추천, 학생별 배포 목록 생성 |
| 학부모 리포트 | 학생 성취추세 + 취약개념 + 추천 과제를 1페이지 리포트로 생성, PDF/이미지 저장(예: jsPDF) |
| 워크플로 | 4단계(평가 입력 → 취약개념 분석 → 과제 추천 → 학부모 리포트) 화면 전환·진행 상태 표시 |

### 2.2 UI/UX 사양

- 교사 중심 단일 워크스페이스. 좌측 내비(학급·학생·평가·리포트), 본문 카드/차트.
- 화면/뷰 **4종 이상**: ① 평가 입력 ② 성취 대시보드 ③ 취약개념 히트맵 ④ 맞춤 과제 추천 ⑤ 학부모 리포트.
- 각 뷰는 단순 표시가 아니라 인터랙션 가능(입력·필터·생성·저장).
- 한글 깨짐 없는 PDF 폰트 임베드.

### 2.3 사용 부품·라이브러리·의존성

- 프론트: Next.js + TypeScript, 차트 라이브러리(예: Recharts/Chart.js), PDF(jsPDF 등).
- 상태/저장: localStorage 또는 IndexedDB(데모 단독 구동 보장, 외부 인프라 불요).
- API 키 하드코딩 금지, `.env` + mock fallback(키 없이 시연 가능).

### 2.4 품질·성능 기준(정량)

| 항목 | 기준 |
|:---|:---|
| 워크플로 완주 | 평가 입력→리포트 저장까지 끊김 없이 1회 완주 |
| 상태 지속성 | 새로고침 후 입력·분석 결과 유지 100% |
| PDF 한글 | 리포트 PDF 한글 깨짐 0건 |
| 핵심 액션 실동작 | 토스트 mock 금지 — 입력·분석·추천·PDF 생성 모두 실 동작 |
| 단독 구동 | 외부 서버·키 없이 로컬에서 빌드·구동 성공 |
