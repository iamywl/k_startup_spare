## 1. 과업의 개요

**과업의 목적**

1) 소상공인 카드매출 선정산·현금흐름·임베디드 대출 시제품(PoC) 웹앱 **「캐시브릿지 v1」 1식**을 제작·검수한다. 사업주·심사역·관리자 3역할이 단일 화면에서 카드매출 집계 → 현금흐름 예측 → 선정산 4단계 워크플로 → 상환·정산 자동차감까지 **실 동작(토스트 mock 아님)** 으로 시연 가능한 수준을 산출물로 한다.
2) 본 과업은 발주자 측 사업 제안([`1_제안서.md`](./1_제안서.md))의 **§2 Solution 핵심 5기능과 §S-2-1 5기능↔3차별기술 매핑**을 시제품으로 입증한다. 특히 §차별화 기술 구매동인에서 must-have로 논증한 **선정산 시뮬레이터(③)·현금흐름 대시보드(①)** 의 실 계산 엔진을 v1 시점에 작동시켜, 논증과 산출물의 정합(§B-5)을 증거로 만든다.

**과업의 범위**

1) 외주사(또는 자체 개발팀)가 수행할 업무
   - 정보구조·3역할 RBAC·13뷰 라우팅 및 **반응형 셸**(데스크톱 사이드바 / 모바일 바텀탭·오프캔버스 드로어) 구현.
   - 핵심 5기능 영역 구현: ① 현금흐름 대시보드 ② 정산 캘린더/스케줄 ③ 선정산 4단계 워크플로 ④ 임베디드 대출(한도→약정→상환 시뮬) ⑤ 상환·정산 자동차감 추적.
   - 심사역·관리자 운영 화면: 심사 큐·포트폴리오·익스포저·연동/Webhook·알림·설정.
   - **실 알고리즘**: 로지스틱 회귀 부도확률(PD)·신용점수/등급·최소제곱 매출추세 회귀·현금흐름 시뮬레이션 엔진.
   - **한글 PDF 3종**(약정서·정산서·월간리포트) 캔버스 렌더, CSV 입출력, mock webhook/알림톡/오픈뱅킹.
   - 상태 지속성(localStorage)·새로고침 복원·데모 초기화.
   - **반응형 검수**: 390px·1280px 실 구동 캡처 눈검수(가로 overflow 0, 표·모달·차트 깨짐 0).
   - 산출물 인도 및 v1 결과보고서용 실 구동 캡처 협조.
2) 범위 외 (Out of Scope) — v1에서 외주사가 수행하지 않음
   - 실 카드사·PG·VAN·오픈뱅킹 **운영 연동**(v1은 mock). 실 자금이동·정산 처리.
   - 여신·대부 인허가, 법정 최고금리·표시의무 등 **규제 준수 실집행**(전문가 자문 영역, [`1_제안서.md`](./1_제안서.md) §핀테크 규제).
   - 회원가입·로그인 등 인증(시제품은 무로그인 데모 — CLAUDE.md §3.4).
   - 위험 스코어링 운영화·동적 한도/금리·다점포 정산(v2 이후).
   - 서버 백엔드·DB 구축(v1은 단일 HTML 자체완결).

## 2. 과업의 세부 내용

**시제품 사양**

| 구분 | 사양 |
|:---|:---|
| 형태 | 단일 HTML 자체완결 웹앱(`v1.html`), 별도 빌드·서버 없이 `file://` 또는 정적 호스팅으로 구동 |
| 역할 | 사업주(owner)·심사역(analyst)·관리자(admin) 3종, 역할별 메뉴 가시성 분기(RBAC) |
| 뷰 | 13뷰(대시보드·카드매출·선정산·정산스케줄·상환·현금흐름·계산기·리포트·심사큐·포트폴리오·연동·알림·설정) |
| 기능 수 | **누적 108기능**(나열은 결과보고서 §2 기능 목록표) |
| 다단계 워크플로 | 선정산 1~4단계(한도 산정 → 리스크 심사 → 약정·상환스케줄 → 실행) 스텝퍼 |

**UI/UX 제작 사양**

- 톤: Apple×Toss 기준([`design-guide/디자인_애플_토스.md`](./design-guide/디자인_애플_토스.md)). 점주 언어·3탭 이내 핵심 동선.
- 반응형: `<meta viewport>` 필수. 모바일은 데스크톱 축소판이 아니라 레이아웃 재구성(사이드바→바텀탭+드로어, 그리드→스택, 차트 고정높이). 좁은 표는 가로 스크롤로 처리(글자단위 줄바꿈 금지).
- 데모 무로그인 즉시 체험, 데모 모드 안내 표기.

**사용 부품·라이브러리·외부 의존성**

- Tailwind(CDN), 바닐라 JS. 차트·한글 PDF는 캔버스 직접 렌더(무거운 외부 라이브러리 미사용).
- API 키 부재 시 mock 매출 데이터로 동작(오프라인 시연 보장, 키 하드코딩 금지).

**품질·성능 기준(정량)**

| 항목 | 기준 |
|:---|:---|
| 기능 실 동작 | 핵심 액션 토스트 mock 금지 — PDF 실 생성·CSV 실 입출력·상태 실 저장 |
| 다단계 워크플로 | 1개 이상(선정산 4단계) |
| 상태 지속성 | localStorage 새로고침 후 복원 |
| 뷰 수 | 4종 이상(실제 13뷰) |
| 반응형 | 390px 가로 overflow 0, 1280px 레이아웃 정상 — 양 뷰포트 실 캡처 눈검수 |
| 신규 캡처 | PC·모바일 각 6장 이상(실제 PC 23·모바일 24) |

- **발주자 협력창구**: 이름 `<TODO: 사용자 입력>` / 연락처 `<TODO: 사용자 입력>` / 소속 `<TODO: 사용자 입력>`
- **작업 방법론·형상관리**: Git, 사이클별 브랜치/태그 고정(`release/2026-fintech-cashbridge/v1` 또는 `2026-fintech-cashbridge-v1` 태그, CLAUDE.md §3.5). 단일 HTML은 `v1.html` 파일 보존(덮어쓰기 금지)으로 버전별 배포 병행.
