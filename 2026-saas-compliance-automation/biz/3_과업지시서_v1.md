# 과업지시서 v1 — 「가드레일」 컴플라이언스 자동화 SaaS 시제품(MVP)

> 본 문서는 발주자(가드레일 사업팀)가 외주 용역사에 시제품 v1 제작 분을 위탁하기 위해 작성한 지시서다. 작성 근거는 [`2_개발계획서.md`](./2_개발계획서.md) §작업 분해의 외주 구분 행(3·7) + 자체 개발 분 통합. 제안서([`1_제안서.md`](./1_제안서.md)) §2 솔루션·§8 로드맵 v1과 정합한다.

---

## 1. 과업의 개요

### 1.1 과업의 목적

1. **직접 목적**: CSAP·ISMS-P 인증 준비를 자동화하는 가드레일 시제품 v1(MVP)을 제작한다. v1은 **단일 인증 프레임워크 기준**으로 통제항목 체크리스트·갭분석 대시보드·증적 업로드/관리·정책문서 자동생성·인증 준비율 스코어·다단계 준비 워크플로가 **실제로 동작하는 PoC**를 목표로 한다.
2. **사업 연계**: 본 시제품은 제안서([`1_제안서.md`](./1_제안서.md)) §2 솔루션과 §8 로드맵 v1(MVP, M1~3)의 핵심 가치명제("통제-증적-정책문서를 단일 모델로 통합")를 동작으로 입증하는 산출물이다. 파일럿(§7 GTM)·고객 검증(§17)·후속 투자 마일스톤(§6.7 Seed)의 근거가 된다.

### 1.2 과업의 범위

**외주사가 수행하는 업무(In Scope)** — 개발계획서 외주 구분 행(3·7) 기준:

1. **통제항목 데이터셋 구축(작업 #3)**: 단일 인증 프레임워크(ISMS-P 또는 CSAP 중 발주자 지정)의 통제항목을 구조화된 데이터(통제 ID·분류·항목명·요구사항·증적유형·평가 메모)로 작성. 인증 기준 원문에 정합하도록 검수.
2. **정책문서 템플릿 코퍼스 작성(작업 #7)**: 통제항목에 매핑되는 정책·지침 문서 템플릿(정보보호정책·접근통제·자산관리 등 대표 N종)을 작성. 자동생성 엔진이 채울 치환 변수(`{{조직명}}`·`{{시행일}}` 등)를 포함한 템플릿 형태로 납품.
3. **(위 데이터를 자체 앱에 탑재 가능한 형식으로 정규화)**: JSON/CSV 등 발주자 지정 스키마로 납품, import 검증 통과.

**범위 외(Out of Scope)** — 외주사가 수행하지 않음(자체 또는 후속):

- 멀티테넌트 백엔드·갭분석·증적관리·준비율·워크플로 등 **애플리케이션 코드 개발은 발주자 자체 수행**(작업 #1·2·4·5·6·8·9).
- 다중 프레임워크 매핑·자동수집 커넥터·심사 패키지 PDF 등 **v2 범위**(별도 [`4_과업지시서_v2.md`](./4_과업지시서_v2.md)).
- 실 클라우드 연동·자사 인증 취득·법률 자문(전문가 자문 범위).
- 통제 데이터의 **최종 인증 적합성 보증**(인증 합격은 심사기관 판단 — 본 과업은 "준비 보조용 데이터" 정합성까지).

### 1.3 v1 자체 산출물 (참고 — 외주 범위 아님)

> 외주 데이터셋을 탑재해 발주자가 자체 개발하는 v1 앱 기능. 검수 시 외주 데이터가 이 앱에서 정상 동작함을 확인한다.

| # | 기능 | 동작 요건 |
|:---:|:---|:---|
| 1 | 통제항목 체크리스트 | 외주 데이터셋 로드, 항목별 충족/미흡/해당없음 상태 토글 |
| 2 | 갭분석 대시보드 | 분류별 충족 현황·미흡 항목 우선순위 시각화 |
| 3 | 증적 업로드/관리 | 파일 실 업로드·미리보기·통제항목 매핑·버전·만료일 |
| 4 | 정책문서 자동생성 | 외주 템플릿 + 조직 변수 → 정책문서 실 생성·다운로드 |
| 5 | 인증 준비율 스코어 | 충족 통제/전체 통제 기반 % 산출·게이지 |
| 6 | 다단계 준비 워크플로 | 프레임워크 선택→통제 점검→증적 첨부→정책 생성→준비율 확인 4단계+ |

## 2. 과업의 세부 내용

### 2.1 통제항목 데이터셋 사양(작업 #3)

| 필드 | 설명 | 비고 |
|:---|:---|:---|
| control_id | 통제항목 고유 ID | 인증 기준 체계 반영 |
| category | 통제 분류(영역) | 대분류·중분류 |
| title | 통제항목명 | 기준 원문 표현 |
| requirement | 요구사항 서술 | 충족 기준 |
| evidence_types | 요구 증적 유형(배열) | 예: 정책문서·접근기록·설정값 |
| review_note | 평가/점검 메모 | 심사 포인트 |
| framework | 소속 프레임워크 | v1은 단일(ISMS-P 또는 CSAP) |

- 통제항목 수량: 발주자 지정 프레임워크의 **대표 통제 영역 전체**(전수 또는 합의된 대표 셋).
- 출력 형식: 발주자 지정 JSON 스키마. UTF-8, 한글 깨짐 없음.

### 2.2 정책문서 템플릿 코퍼스 사양(작업 #7)

- 템플릿 종류: 정보보호정책·접근통제 지침·자산관리 지침 등 **대표 N종**(발주자 합의).
- 형식: 치환 변수 포함 마크다운/HTML 템플릿(`{{조직명}}`·`{{시행일}}`·`{{책임자}}` 등).
- 각 템플릿은 1개 이상의 통제항목(control_id)에 매핑되어 납품.

### 2.3 품질·정합성 기준

- 통제항목-증적유형 매핑이 인증 기준 원문에 정합(매핑 누락 0 목표 — 제안서 §10.5).
- 데이터셋이 발주자 import 검증을 통과(파싱 오류 0).
- 정책 템플릿 치환 변수가 자동생성 엔진에서 정상 치환.
