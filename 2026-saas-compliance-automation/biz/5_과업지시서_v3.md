# 과업지시서 v3 — 「가드레일」 컴플라이언스 자동화 SaaS 고도화(Series A 데모)

> 본 문서는 v2 검수 후 추가/심화된 과업을 정의한다. v1([`3_과업지시서_v1.md`](./3_과업지시서_v1.md))·v2([`4_과업지시서_v2.md`](./4_과업지시서_v2.md))를 수정하지 않고 신규 파일로 분리한다. 작성 근거는 [`2_개발계획서.md`](./2_개발계획서.md) §작업 분해의 v3 행(자체 v3 산출물) + 외주 구분이다. 제안서([`1_제안서.md`](./1_제안서.md)) §8 로드맵 v3·§6.7 Series A 게이트와 정합하며, §0·§10은 접촉하지 않는다. 문서 말미에 변경 이력(v2→v3)을 둔다.

## 1. 과업의 개요

### 1.1 과업의 목적

1. **직접 목적**: 가드레일 시제품을 v2(다중 프레임워크·다중 조직 베타)에서 **v3(연속 모니터링·외부 연동·심사관 협업·다국어 심사패키지를 갖춘 Series A 데모)**로 고도화한다. 단순 준비율 산출을 넘어 **실시간 컴플라이언스 상태 추적·정량 리스크 스코어링·외부 GRC/알림 시스템 연동·심사관 협업**을 동작으로 입증해 **시리즈A 투자 게이트 데모 가능 수준**을 목표로 한다.
2. **사업 연계**: 제안서 §8 로드맵 v3(GA 직전, M7~9)와 §3.2 차별점(연속 모니터링·교차매핑 자동화)·§5.3 해자(전환비용·통합 네트워크)를 동작으로 입증한다. §6.7 Series A 게이트 데모의 직접 근거다.

### 1.2 과업의 범위

**외주사가 수행하는 업무(In Scope)** — 개발계획서 v3 외주 구분 행:

1. **다국어(영문) 심사패키지 템플릿·통제 영문 대역 데이터(작업 #v3-out)**: ISO/IEC 27001 해외 심사 제출용 **영문 심사패키지 템플릿**과 공통통제 16종의 **영문 명칭·도메인 영문 대역(en) 데이터**를 작성·검수한다. 국문/영문 PDF 출력 시 깨짐 0, ISO Annex A 통제ID 정합.
2. **연속 모니터링 프로브 룰셋 검수**: 통제 프로브 5종(MFA·암호화·로그보존·방화벽·퇴직자계정)의 **점검 기준·이탈(drift) 판정 임계치**가 각 프레임워크 기준 원문에 정합하는지 도메인 자문 검수.

**범위 외(Out of Scope)**:

- 연속 모니터링 엔진·교차매핑 충족계산·리스크 스코어링 알고리즘·Jira/Slack 커넥터·심사관 협업 포털·정책↔증적 일치성 검사·KPI/SLA 대시보드 등 **애플리케이션 코드는 발주자 자체 수행**(v3 자체 산출물 §5).
- 실 클라우드 API·실 Jira/Slack API 실연동(키 발급·OAuth 정식 연동은 GA 범위). v3는 키 부재 시 mock fallback으로 동작.
- 자사 인증 취득·법률 자문·영문 번역의 법적 효력 보증(심사 준비 보조용 정합성까지).

## 2. 과업의 세부 내용

### 2.1 다국어(영문) 심사패키지 사양(작업 #v3-out)

| 항목 | 사양 |
|:---|:---|
| 입력 | 통제 충족 현황·증적 메타·정책 목록·리스크 등록부(발주자 앱 데이터) |
| 구성 | 표지·요약·통제별 충족 현황(영문)·증적 인덱스·정책 목록·위험 등록부 |
| 언어 | 국문(KO)/영문(EN) 2종, i18n 사전 기반 라벨 치환 |
| 통제 영문 | 공통통제 16종 영문 명칭(en)·도메인 영문(domainEn), ISO Annex A 통제ID 정합 |
| 폰트 | 한글·영문 모두 깨짐 0(jsPDF canvas 렌더 임베드) |
| 출력 | PDF 1식, 프레임워크별·언어별 분리 생성 |

### 2.2 연속 모니터링 프로브 룰셋

- 프로브 5종: MFA 적용률·저장 암호화·로그 보존기간·방화벽 개방·퇴직자 계정.
- 각 프로브의 **목표 기준값**과 **이탈(drift) 판정**(충족 통제가 미흡으로 강등되는 조건)이 기준 원문에 정합(자문 검수).

### 2.3 품질·정합성 기준

- 국문/영문 PDF 생성 성공·한글/영문 깨짐 0(제안서 §10.5).
- 영문 통제 명칭이 ISO/IEC 27001 Annex A 통제와 의미 정합(누락 0).
- 프로브 판정 임계치가 각 프레임워크 통제 요구수준에 정합.

## 5. 성과품 목록

> v3 자체 산출물 핵심 8가지(발주자 자체 개발)를 명시한다. 외주(영문 패키지·프로브 룰셋)는 본 목록의 #7·#1과 관련 데이터로 검수에 통합한다.

### 5.1 v3 자체 산출물 핵심 8가지 (검수 대상)

| # | 산출물 | 검수 합격 조건(측정 가능) |
|:---:|:---|:---|
| 1 | **실시간 연속 모니터링(continuous compliance·drift 감지 알림)** | 프로브 5종 주기 점검 실행, 설정 변경 시뮬레이션 시 충족 통제가 미흡으로 강등되는 **drift 감지** + 자동 알림(Slack mock) 발생, drift 조치완료 시 통제 복구. 이벤트 타임라인 기록 |
| 2 | **다중 프레임워크 교차매핑 자동(통제 1건 → 동시 충족 계산)** | 공통통제 1건 충족 입력 시 CSAP·ISMS-P·ISO27001 매핑 control_id 가 **동시 충족**으로 계산, 전 프레임워크 준비율이 단일 모델에서 동시 산출. 최소 3개 프레임워크 동시 표기 |
| 3 | **외부 GRC/티켓·알림 연동 mock(Jira·Slack)** | Jira 이슈 생성(GRC-키 발번)·Slack 알림 발송이 mock 으로 동작, 전 호출이 연동 로그에 기록. 티켓 생성·완료 시 Jira/Slack 연계 호출. 최소 2개 외부 시스템 |
| 4 | **위험 스코어링·우선순위 알고리즘(통제 미흡 → 리스크 점수)** | 노출도×영향도×드리프트×증적부재 산식으로 통제별 0~100 리스크 점수 산출, 레벨(심각/높음/보통/낮음) 분류·우선순위 정렬, 잔여 리스크 지수 집계, 도메인 히트맵 |

| 6 | **정책↔증적 일치성 자동검사** | 정책·통제·증적 3자 교차검증으로 (정책 미생성/증적 누락/통제 미충족/정합) 판정 자동 산출, 누락 탐지 표 |
| 7 | **영문 ISO 심사패키지(다국어)** | 언어 전환(KO/EN)으로 국문·영문 심사패키지 PDF 각각 생성, 영문 통제 명칭·라벨 치환, 한글/영문 깨짐 0(외주 영문 템플릿 탑재) |

## 변경 이력 (v2 → v3)

| 구분 | v2 | v3 (변경/추가) |
|:---|:---|:---|
| 모니터링 | 준비율 스냅샷·추세 | + **실시간 연속 모니터링**(프로브 주기 점검·drift 감지·자동 알림) |

| 외부 연동 | 클라우드 스캔·알림 mock | + **Jira 이슈 생성 + Slack 알림 GRC 연동**(호출 로그·티켓 동기화) |
| 위험 분석 | 위험가중 갭 우선순위 | + **리스크 스코어링 알고리즘**(노출도×영향도×드리프트×증적, 잔여리스크 지수·레벨·히트맵) |

| 대시보드 | 준비율 게이지·갭 표 | + **KPI·SLA 대시보드**(6 KPI·전 프레임워크·SLA 게이지·리스크 히트맵·모니터링 요약) |
