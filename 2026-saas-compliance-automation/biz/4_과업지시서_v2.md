# 과업지시서 v2 — 「가드레일」 컴플라이언스 자동화 SaaS 심화(Beta·Series A 데모)

> 본 문서는 v1 검수 후 추가/심화된 과업을 정의한다. v1([`3_과업지시서_v1.md`](./3_과업지시서_v1.md))을 수정하지 않고 신규 파일로 분리한다. 작성 근거는 [`2_개발계획서.md`](./2_개발계획서.md) §작업 분해의 v2 행 + 외주 구분(14) + 자체 v2 산출물. 제안서([`1_제안서.md`](./1_제안서.md)) §8 로드맵 v2와 정합한다. 문서 말미에 변경 이력(v1→v2)을 둔다.

---

## 1. 과업의 개요

### 1.1 과업의 목적

1. **직접 목적**: 가드레일 시제품을 v1(단일 프레임워크 PoC)에서 **v2(다중 프레임워크·다중 조직 베타)**로 심화한다. 다중 인증 동시 대응·실 백엔드 시뮬레이션·역할별 권한·심사 대응 패키지 생성을 갖춰 **파일럿 투입·Series A 데모 가능 수준**을 목표로 한다.
2. **사업 연계**: 제안서 §8 로드맵 v2(Beta, M4~6)와 §3.2 차별점("공통 통제 단일 입력·다중 재사용")·§5.3 해자(전환비용·네트워크)를 동작으로 입증한다. §6.7 Series A 게이트 데모의 근거다.

### 1.2 과업의 범위

**외주사가 수행하는 업무(In Scope)** — 개발계획서 v2 외주 구분 행(14):

1. **심사 대응 패키지 PDF 생성 엔진(작업 #14)**: 통제 충족 현황·증적 목록·정책문서 요약·준비율을 종합한 **심사 대응 패키지 문서를 PDF로 자동 생성**하는 엔진/템플릿 제작. 한글 폰트 임베드, 표·목차·증적 인덱스 포함.
2. **(v2 다중 프레임워크용 통제 데이터 확장)**: v1 단일 프레임워크 데이터셋을 CSAP·ISO27001로 확장하고 **공통 통제 매핑표**(프레임워크 간 동일/유사 통제 연결)를 작성·검수.

**범위 외(Out of Scope)**:

- 다중 프레임워크 매핑 엔진·테넌트 전환·자동수집 커넥터·티켓·권한·감사로그 등 **애플리케이션 코드는 발주자 자체 수행**(v2 자체 산출물 §5).
- 실 클라우드 API 실연동(v3 범위).
- 자사 인증 취득·법률 자문.
- 통제 데이터의 최종 인증 적합성 보증(준비 보조용 정합성까지).

## 2. 과업의 세부 내용

### 2.1 심사 대응 패키지 PDF 엔진 사양(작업 #14)

| 항목 | 사양 |
|:---|:---|
| 입력 | 통제 충족 현황·증적 메타·정책문서 요약·준비율(발주자 앱 데이터) |
| 구성 | 표지·목차·요약(준비율·프레임워크)·통제별 충족 현황표·증적 인덱스·정책문서 목록 |
| 폰트 | 한글 폰트 임베드(깨짐 0) |
| 출력 | PDF 1식, 프레임워크별 분리 생성 가능 |
| 생성 방식 | jsPDF 또는 서버 PDF(발주자 스택 정합), 비동기 잡 큐 연동 가능 형태 |

### 2.2 다중 프레임워크 통제 데이터 확장

- v1 데이터셋(단일) → CSAP·ISO27001 추가.
- **공통 통제 매핑표**: 프레임워크 간 동일/유사 통제를 연결(예: ISMS-P 접근통제 ↔ ISO27001 A.9 ↔ CSAP 접근통제)해 1회 입력·다중 재사용을 가능케 함.

### 2.3 품질·정합성 기준

- PDF 생성 성공률 ≥ 99%(제안서 §10.5), 한글 깨짐 0.
- 공통 통제 매핑이 각 프레임워크 기준 원문에 정합(누락 0, 자문 검수).

## 5. 성과품 목록

> v2 자체 산출물 핵심 8가지(발주자 자체 개발)를 명시한다. 외주(PDF 엔진·데이터 확장)는 본 목록의 #5·관련 데이터로 검수에 통합한다.

### 5.1 v2 자체 산출물 핵심 8가지 (검수 대상)

| # | 산출물 | 검수 합격 조건(측정 가능) |
|:---:|:---|:---|
| 1 | **다중 프레임워크 매핑** | CSAP·ISMS-P·ISO27001 통제를 단일 모델에 로드, 공통 통제 1건 입력 시 매핑된 프레임워크에 동시 반영. 최소 3개 프레임워크 전환 동작 |
| 2 | **다중 조직(테넌트)** | 2개 이상 조직 생성·전환, 조직별 통제 상태·증적 격리(타 조직 데이터 비노출) |
| 3 | **증적 자동수집 커넥터 mock** | 클라우드 설정 스캔 mock 실행 → 스캔 결과를 증적 후보로 자동 등록 → 통제 매핑. 스캔 로그 기록 |
| 4 | **갭→과제 티켓·담당자 배정** | 미흡 통제에서 과제 티켓 생성 → 담당자 배정 → 상태(예정/진행/완료) 변경 → 증적 첨부로 통제 충족 전파 |
| 5 | **심사 대응 패키지 PDF 생성** | 통제 현황·증적·정책·준비율 종합 PDF 실 생성·다운로드(외주 엔진 탑재). 한글 깨짐 0, 생성 성공 |
| 6 | **정책문서 버전관리** | 정책문서 수정 시 버전·diff·이력 보존, 이전 버전 조회·롤백 |
| 7 | **역할 권한(CISO·실무자·심사관 뷰)** | 3개 역할별 화면·권한 분기(CISO 전체·실무자 작업·심사관 읽기/감수), 권한 외 기능 비노출 |
| 8 | **준비율 추세·감사로그** | 준비율 시계열 추세 차트 + 주요 액션(상태변경·증적등록·정책수정) 감사로그 기록·조회 |

### 5.2 외주 성과품 (5.1에 통합 검수)

| # | 성과품 | 수량/형태 | 검수 합격 조건 |
|:---:|:---|:---|:---|
| A | 심사 패키지 PDF 엔진 | 엔진 코드 + 샘플 PDF | 앱 연동 생성 성공률 ≥ 99% / 한글 깨짐 0 / 목차·증적 인덱스 포함 |
| B | 다중 프레임워크 통제 데이터(CSAP·ISO27001) | JSON 1식 | import 파싱 오류 0 / 기준 원문 정합(자문 검수) |
| C | 공통 통제 매핑표 | CSV 1식 | 프레임워크 간 매핑 누락 0 / #1 매핑 기능에서 동작 |

## 변경 이력 (v1 → v2)

| 구분 | v1 | v2 (변경/추가) |
|:---|:---|:---|
| 프레임워크 | 단일(ISMS-P 또는 CSAP) | **다중**(CSAP·ISMS-P·ISO27001) + 공통 통제 매핑 |
| 조직 | 단일 | **다중 조직(테넌트)** 격리·전환 |
| 증적 | 수동 업로드/관리 | + **자동수집 커넥터 mock**(클라우드 설정 스캔) |
| 갭 처리 | 갭분석 표시 | + **과제 티켓·담당자 배정** 워크플로 |
| 산출물 | 정책문서 생성 | + **심사 대응 패키지 PDF**(외주 엔진) |
| 정책 | 단일 버전 | + **버전관리(diff·이력·롤백)** |
| 권한 | 단일 사용자 | + **역할 권한(CISO·실무자·심사관)** |
| 분석 | 준비율 스코어 | + **준비율 추세·감사로그** |
| 외주 범위 | 통제 데이터셋·정책 템플릿(v1) | 심사 패키지 PDF 엔진 + 다중 프레임워크 데이터·공통 매핑 |
