## 0. AI 정직성 고지 (§2.6 — 먼저 밝힌다)

본 데모의 **"AI 생성·채점"은 외부 LLM(ChatGPT/Claude 등) 호출이 아니다.** 커머스 도메인 지식을 코드화한 **자체 결정론적 룰/템플릿 엔진**이다. 이는 제안서([`1_제안서.md`](./1_제안서.md)) §데이터 정직성 선언·§구매동인 ⑤(라)와 정합한다.

**"AI 래퍼가 아님"의 코드 레벨 근거** (제안서 §구매동인 ⑤ 논증을 데모가 어떤 형태로 시연하는가):

| 근거 | v1.html 구현 | 의미 |
|:---|:---|:---|
| 도메인 사전 | `CAT_LEX` — 카테고리 6종별 훅·베네핏·파워워드 사전(예: beauty→"속보습"·"저자극") | 모델 밖 도메인 자산(제안서 ⓐ) |
| eval 레이어 | `scoreText()` — 5지표(길이·파워워드 밀도·구체수치·CTA·광고규정) 가중합 채점 + 항목별 사유 | 생성물 채점은 래퍼에 없음(제안서 ⓒ) |
| 금칙어 검수 | `BANNED` 정규식 5패턴 + `state.brand.banExtra` — 의약품 오인·최상급·단정·배타·과장 차단 | 정책 검수 레이어(제안서 ⓐ·ⓒ) |
| 결정론성 | `rng()` = FNV-1a 시드 해시 → 동일 입력 동일 출력 | LLM 비결정성 아님, 외부 호출 0 |

→ v1 데모의 룰 엔진은 제안서가 가리키는 **실제 독자 엔진(도메인 코퍼스·전환 라벨·eval·검수 루프)을 결정론 버전으로 시연**해 *제품 형태와 통합 지점*을 입증하는 골격이다. LLM 실연동은 v2 이후 범위(과업지시서 §1.2 Out of Scope).

## 4. 실행/구동 방법

1. `projects/seller-studio/v1.html` 을 브라우저로 연다(파일 더블클릭 또는 `open v1.html`).
2. 첫 구동 시 데모 시드데이터(제품·브랜드·히스토리)가 localStorage 에 주입된다.
3. 좌측 사이드바(모바일: 햄버거 드로어/바텀탭)로 12개 뷰 이동.
4. 제품 선택 → 상세/카피/이미지 생성 → 품질·금칙어 검수 → A/B 비교 → 내보내기 순으로 4단계 워크플로 시연.
5. 새로고침 후에도 상태 유지(localStorage). 초기화는 관리자/설정의 데모 초기화.
6. 키·네트워크 불필요(오프라인 동작, jsPDF만 PDF 내보내기 시 CDN).
