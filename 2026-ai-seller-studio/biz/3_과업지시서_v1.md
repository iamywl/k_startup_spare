## 1. 과업의 개요

### 1.1 과업의 목적

1) **직접 목적** — 셀러스튜디오 시제품(단일 HTML 자체완결 데모, `projects/seller-studio/v1.html`)을 외부 인프라 없이 단독 구동 가능한 형태로 **통합·안정화하고, PC·모바일 양 뷰포트에서 정상 동작을 검수**한다. 동시에 커머스 도메인 룰셋(금칙어·채널 글자수 제약)을 보강하고 결과 문서화를 지원한다.
2) **제안서 연결** — 본 시제품은 [`1_제안서.md`](./1_제안서.md) §2.3 사용자 여정(입력→생성·검수→집행→측정 4단계 워크플로)과 §구매동인 ⑥(데모 정합)이 가리키는 산출물이다. 데모는 제안서가 주장하는 **검수·등록·측정 완결성**(must-have 동인)을 실제 작동하는 형태로 입증하기 위한 PoC다. 단, 데모의 "AI 생성·채점"은 **외부 LLM 호출이 아닌 자체 결정론적 룰/템플릿 엔진**임을 화면·문서에 정직 표기한다(제안서 §데이터 정직성 선언·§구매동인 ⑤(라)와 정합).

### 1.2 과업의 범위

**1) 외주사가 수행할 업무**
- 단일 HTML 데모의 **빌드/구동 무결성 검증** — 외부 네트워크 0(jsPDF CDN 1건 제외) 상태에서 단독 구동, 콘솔 오류 0.
- **반응형 교차 뷰포트 검수** — 데스크톱(1280±)·모바일(390×844) 양쪽에서 12개 뷰 전수 점검(사이드바→드로어/바텀탭 전환, 가로 overflow 0, 차트·표·모달 깨짐 0). `CLAUDE.md` §3.3 [반응형_검수_필수] 절차 준수(스크린샷 눈검수).
- **커머스 도메인 룰셋 보강** — 금칙어 정규식 패턴(의약품 오인·최상급·단정·배타·과장 프로모션) 및 채널별(네이버·인스타·구글·카카오) 제목/본문 글자수 제약 enforcement 점검·보완.
- **실 구동 캡처 산출** — PC·모바일 폴더 분리 저장(`captures/v1/`·`captures/mobile/v1/`).
- 결과 문서화 지원(성과품↔납품 매핑 근거 제공).

**2) 범위 외 (Out of Scope)** — 분쟁 예방 목적 명시
- **외부 LLM(ChatGPT/Claude 등) 실연동** — v1 데모는 결정론 엔진이며 실 LLM 호출은 v2 이후. 본 과업에 포함하지 않는다.
- **실 채널(스마트스토어·쿠팡 등) API 등록 연동** — 데모는 등록 페이로드/발송 로그 시뮬레이션까지. 실 OAuth·실 발행은 범위 외.
- **실 전환 데이터 수집·서버 백엔드·DB·과금 결제** — v1은 localStorage 단일 기기 영속까지.
- **팀·서명·계약 행정 양식 작성** — 발주자 영역(`<TODO: 사용자 입력>`).

## 2. 과업의 세부 내용

### 2.1 시제품 사양 (SW)
- **형태**: 단일 HTML 파일 자체완결(`v1.html`). 외부 의존성은 jsPDF CDN 1건뿐, 그 외 네트워크 0.
- **뷰 12종**: 대시보드 / 제품 / 상세페이지 / 광고카피 / 이미지 변형 / A/B 비교 / 품질·전환 스코어 / 브랜드 톤·금칙어 / 생성 히스토리 / 내보내기 / 사용량 / 관리자.
- **워크플로**: 제품 입력 → 콘텐츠 생성(상세·카피·이미지) → eval/금칙어 검수 → A/B 비교 → 내보내기(집행 형식) → 사용량·히스토리(측정) 의 4단계 골격.

### 2.2 UI/UX 제작 사양
- 디자인 가이드: `design-guide/디자인_애플_토스.md`(톤)·`디자인_모바일.md`·`디자인_PC.md`.
- 데스크톱: 좌측 사이드바 + 콘텐츠. 모바일: 사이드바를 햄버거 오프캔버스 드로어 + 하단 바텀탭으로 재구성(상단 패널 하이재킹 금지).
- 좁은 화면(≈360~430px) 가로 overflow 0, 그리드 스택, 차트 고정높이.
- 상단에 **AI 정직성 고지 배너**(결정론 엔진임을 사용자에게 명시) 노출.

### 2.3 사용 라이브러리·외부 의존성
- jsPDF 2.5.1 (CDN, 카탈로그 PDF 생성). 그 외 외부 라이브러리·네트워크 호출 없음.
- canvas 2D(이미지 실픽셀 변환), Web `localStorage`(상태 영속).

### 2.4 품질·성능 기준 (정량)
- 기능 단위 **114개** 동작(`window.__FEATURES__` 카운트와 일치).
- 12개 뷰 전부 인터랙션 가능(단순 표시 아님).
- 모바일(390)·PC(1280) **양 뷰포트 가로 overflow 0**, 콘솔 오류 0.
- 금칙어 검수: 5개 정규식 패턴 + 브랜드 추가금칙어 실 매칭, 차단 카운트 집계.
- 품질 eval: 5지표(길이·파워워드 밀도·구체수치·CTA·광고규정) 가중합 종합점수 + 예상전환율 추정.
