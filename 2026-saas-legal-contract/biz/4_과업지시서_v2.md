# 과업지시서 v2 — 리걸체크 계약서 검토·협업 플랫폼

> 발주자가 외주 용역사에 「리걸체크」 v2 시제품 개발을 위탁하기 위한 지시서.
> 작성 근거: [`2_개발계획서.md`](./2_개발계획서.md) WBS의 v2 외주 행 → 본 §1 과업의 범위.
> v1([`3_과업지시서_v1.md`](./3_과업지시서_v1.md))은 수정하지 않으며, v2는 신규 과업(역할분기·룰셋확장·표준 대비 diff·협상 랭킹·외부 통합)을 추가한다.

---

## 1. 과업의 개요

### 과업의 목적

1) **직접 목적:** v1(룰기반 단일 사용자 검토 도구) 위에, **다중 사용자(3역할 RBAC)·케이스 관리·표준 대비 조항 diff·협상 우선순위 랭킹·전자서명/알림톡 mock·CSV/JSON 입출력**을 갖춘 **계약서 검토·협업 플랫폼 시제품(`v2.html` 단일 HTML 1식)**으로 확장한다.
2) **사업 연결:** 본 v2는 [`1_제안서.md`](./1_제안서.md) §3.3 로드맵의 v2 단계(계약 유형 분류·다중 사용자·심화 분석)를 실증한다. 베타 출시·시리즈A 데모용으로, "단순 점검 도구"가 아닌 **검토 워크플로 전체를 흐르는 협업 SaaS**임을 보인다.

> ⚠️ **정직성 고지:** v2의 분석 엔진도 **결정론적 룰/알고리즘**(가중치 모델·토큰 유사도·ROI 랭킹)이며 **LLM 추론이 아니다.** 전자서명·알림톡·webhook은 **외부 키 없이 동작하는 mock**이다.

### 과업의 범위

**In Scope (외주사 수행)**
- 위험 룰셋 **10종 → 20종 확장** + **가중치 모델**(심각도 × 계약유형 가중 × 발생빈도 × 등급계수) 산출.
- **표준 대비 조항 diff** — 텍스트 정규화·토큰(어절+한글 bigram) Jaccard 유사도로 내 계약 ↔ 균형 표준계약 조항 비교, **독소/누락/정합** 자동 분류.
- **협상 우선순위 랭킹** — 리스크 임팩트 × 교섭난이도로 ROI 산출·순위 부여, 임팩트–난이도 산점도.
- **3역할 RBAC**(의뢰인·검토자·관리자) — 역할별 화면·권한 분기(검토자 작업 큐, 관리자 전용 리포트, 코멘트 작성자 분기).
- **케이스 관리 보드** — 다중 계약 케이스, 단계(초안·검토중·협상중·서명중·완료) 칸반/필터, KPI.
- **협업 코멘트** — 조항별 스레드, 작성자 역할 색 구분, 해결/미해결 토글.
- **전자서명 요청 mock + 상태 추적** — 서명자 목록·타임라인·외부 발송 로그(전자서명 API/알림톡/webhook mock).
- **CSV / JSON 입출력** — 케이스 포트폴리오 CSV·JSON 내보내기, JSON 백업 가져오기(병합).
- 검토 리포트 PDF(v1 계승, 협상 우선순위·표준정합도 반영, **한글 보존**).
- localStorage 케이스 영속, 반응형(데스크톱 사이드바 ↔ 모바일 햄버거 드로어 + 바텀탭 + 더보기 시트).

**Out of Scope (외주사 미수행)**
- 서버·DB·실 계정/로그인·실 결제(시제품은 클라이언트 단일 HTML, RBAC는 클라이언트 역할 스위치 시뮬레이션).
- 실 LLM/외부 AI API 연동(v2도 결정론적 룰/알고리즘).
- 실 전자서명 사업자(KICA·모두싸인 등) 연동·법적 효력 보증·실제 법률 자문(전자서명·알림톡은 mock).
- 캡처 스크립트·문서 작성(발주자 자체 수행).

## 2. 과업의 세부 내용

### 2.1 기능 사양

| 기능 | 사양 |
|:---|:---|
| 룰셋 | **20종**(v1 10종 + 신규 10종: 지연이자율·하자담보·일방상계·불공정 불가항력·일방양도·과도감사·포괄독점·광범위 면책·부당 의무존속·일방 단가인상) |
| 가중치 모델 | `weight = severity(1~10) × typeWeight(유형별) × freqFactor(출현빈도) × qualFactor(정량신호) × levelCoef(등급계수)`; 점수 = Σweight × 0.62 정규화(0~100) |
| 유형 가중 | 계약 유형(용역·공급·임대차)별 치명 조항 가중(예: 용역=IP귀속 1.4·대금 1.3, 임대차=자동갱신 1.4) |
| 표준 대비 diff | 정규화(소문자·기호제거·공백정리) → 토큰(어절+한글 bigram) Jaccard 유사도. 유사도 ≥0.34=정합, <0.34=독소. 핵심 보호조항(책임제한·비밀유지·불가항력) 부재 시 **누락**. 평균 유사도=표준 정합도(%) |
| 협상 랭킹 | ROI = 리스크 임팩트(weight) ÷ 교섭난이도(1~5). ROI 내림차순 우선순위 #1.. 부여, 산점도(x=난이도, y=임팩트) |
| RBAC | 역할 3종(의뢰인·검토자·관리자) 스위치. 검토자=작업 큐·코멘트 검토자 작성, 관리자=관리자 리포트·CSV/JSON 입출력 노출 |
| 케이스 보드 | 다중 케이스 카드(제목·유형·점수·등급·단계·미해결 코멘트), 단계 필터(전체/초안/검토/협상/서명/완료), KPI 4종 |
| 협업 코멘트 | 조항 선택 + 본문 작성, 작성자 역할 색(의뢰인 인디고/검토자 블루/관리자 퍼플), 해결·미해결 토글, 미해결 카운트 배지 |
| 전자서명 mock | 서명자 목록(상태: 대기→발송됨→완료), 진행 타임라인, 발송 로그(POST /esign·KAKAO alimtalk·esign webhook), 전원 서명 시 케이스 '완료' 전이 |
| CSV/JSON | CSV 내보내기(BOM+따옴표 이스케이프, 한글), JSON 백업 내보내기, JSON 가져오기(id 중복 제외 병합) |
| PDF | jsPDF + 캔버스 한글 렌더(v1 계승), 협상 우선순위(ROI 순)·표준정합도·단계 반영, 다중 페이지 |
| 영속 | localStorage `legalcheck_cases_v2`, 새로고침 후 케이스·코멘트·서명상태 유지 |

### 2.2 UI/UX 사양

- 디자인 톤: [디자인_애플_토스.md](../../design-guide/디자인_애플_토스.md) — 브랜드 단일 포인트(신뢰 인디고), 위계=여백·타이포, WCAG AA 대비. 그림자는 떠 있는 레이어(드로어·시트·토스트)에만.
- 데스크톱: [디자인_PC.md](../../design-guide/디자인_PC.md) — 좌측 사이드바 252px(섹션 그룹+역할 스위처) + 본문 `min-w-0`, KPI 4열·차트 2열.
- 모바일: [디자인_모바일.md](../../design-guide/디자인_모바일.md) — 사이드바 숨김·**햄버거 오프캔버스 드로어**(메뉴 多) + **바텀탭 5개**(보드·입력·분석·협상·더보기 시트). KPI 2열, 차트 고정높이, diff는 좌우 분할→상하 스택, 표는 가로 스크롤, 탭 ≥44px, overflow 0.
- 화면 흐름(**10뷰**): 케이스 보드 → 계약서 입력 → 위험 분석 → 표준 대비 diff → 협상 우선순위 → 표준계약서 생성 → 협업 코멘트 → 전자서명 → 관리자 리포트 → 검토 이력.
- 워크플로(3개 이상): ① 입력→분석→diff→협상 채택→서명(검토 파이프라인) ② 검토자 코멘트→해결→단계 전이(협업) ③ 케이스 포트폴리오→CSV/JSON 입출력(관리/백업).

### 2.3 라이브러리·의존성

| 항목 | 버전 | 비고 |
|:---|:---|:---|
| Chart.js | 4.x (CDN) | 위험분포·카테고리 가중·게이지·ROI 산점도·관리자 분포 |
| jsPDF | 2.5.x (CDN) | 리포트 PDF(캔버스 한글 렌더) |
| 외부 API 키 | **없음** | 전자서명·알림톡 mock, 전부 클라이언트, 키 하드코딩 금지 |

### 2.4 품질·성능 기준 (정량)

- 좁은 화면(390px) 가로 overflow **0px**, 콘솔 pageerror **0건**, 탭/버튼 터치 타깃 **≥44px**.
- 예시 계약서 분석(20룰 매칭+diff+ROI) **1초 이내**.
- diff 표준 유사도·독소/누락/정합 카운트가 룰 매칭 결과와 **정합**.
- 전자서명 mock: 발송 → 상태 '발송됨' → 전원 완료 → 케이스 '완료' 전이 + 로그 기록 **확인 가능**.
- CSV/JSON 내보내기 **실제 다운로드**(한글 깨짐 0), JSON 가져오기 **병합 동작**.
- PDF 한글 깨짐 **0건**, 새로고침 후 케이스·코멘트·서명상태 **유지율 100%**.
- 역할 전환 시 화면·권한 분기 **즉시 반영**(검토자 큐·관리자 리포트·코멘트 작성자).
