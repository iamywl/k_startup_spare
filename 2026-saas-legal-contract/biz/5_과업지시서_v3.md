# 과업지시서 v3 — 리걸체크 계약서 검토·협업·협상 플랫폼 (시리즈A 데모)

> 발주자가 외주 용역사에 「리걸체크」 v3 시제품 개발을 위탁하기 위한 지시서.
> 작성 근거: [`2_개발계획서.md`](./2_개발계획서.md) WBS의 v3 외주 행 → 본 §1 과업의 범위.
> v1([`3_과업지시서_v1.md`](./3_과업지시서_v1.md))·v2([`4_과업지시서_v2.md`](./4_과업지시서_v2.md))는 수정하지 않으며, v3는 신규 과업(TF-IDF 코사인 diff·등급 캘리브레이션·협상 시뮬레이터·법령/판례 매칭 DB·전자서명 OAuth·webhook 콜백·팀 협업 알림·KO/EN i18n·감사 로그·계약 버전관리)을 추가한다.

---

## 1. 과업의 개요

### 과업의 목적

1) **직접 목적:** v2(다중 사용자·표준 diff·협상 랭킹·전자서명 mock 플랫폼) 위에, **고급 분석 알고리즘(TF-IDF 코사인·등급 캘리브레이션·협상 시뮬레이터)·법령/판례 매칭 DB·전자서명 OAuth 흐름 + webhook 콜백·실시간 팀 협업 알림·KO/EN 다국어·감사 로그·계약 버전관리**를 갖춘 **시리즈A 데모급 계약 협상 플랫폼 시제품(`v3.html` 단일 HTML 1식)**으로 확장한다.
2) **사업 연결:** 본 v3는 [`1_제안서.md`](./1_제안서.md) §3.3 로드맵의 v3 단계(협업·협상·전자서명·B2B 법무 대시보드)를 실증하고, 같은 제안서 §차별화 기술의 구매동인 논증 ⑤에서 가리킨 데모 구현 지점(위험탐지·표준 diff·협상 ROI·시뮬레이터·버전관리·KO/EN)을 **실제 동작으로 시연**한다.

> ⚠️ **정직성 고지:** v3의 분석 엔진도 **결정론적 룰/알고리즘**(22룰 가중치 모델·TF-IDF 코사인 유사도·등급 캘리브레이션·협상 시뮬레이터)이며 **LLM 추론이 아니다.** 전자서명 OAuth·알림톡·webhook·팀 알림·법령/판례 매칭은 **외부 키 없이 동작하는 mock/내장 룰DB**다.

### 과업의 범위

**In Scope (외주사 수행)**
- 위험 룰셋 **20종 → 22종 확장**(+포괄적 자동출금·담보·연대보증, +포괄적 데이터·개인정보 권리) + 각 룰에 **법령/판례 매칭 근거(`lawRef`)** 내장.
- **TF-IDF 코사인 유사도 diff** — 표준조항 코퍼스로 IDF 산출 → TF-IDF 벡터 코사인으로 내 계약 ↔ 표준 조항 비교, 독소/누락/정합 분류 정교화(v2 토큰 Jaccard 대체).
- **위험점수 등급 캘리브레이션** — 고위험 조항 수에 따라 등급 임계(고위험/중위험)를 동적 조정하여 점수→등급 매핑을 보정.
- **협상 시뮬레이터** — 조항별 ‘표준대로 수정’ 토글 → 위험 가중을 표준 수준으로 낮춘 위험점수를 실시간 재계산, before/after 차트·등급 변화 표시, 선택 수정안 일괄 채택.
- **전자서명 OAuth 2.0 흐름 mock** — authorize → callback(state 검증) → token 교환 핸드셰이크 로그, 연결 후에만 서명 요청 가능, **webhook 콜백**으로 완료 상태 추적.
- **실시간 팀 협업 알림 mock** — 코멘트 작성 시 `collab.comment.created` webhook + 팀 알림 로그 적재.
- **KO/EN 다국어(i18n)** — 사이드바·모바일 토글로 전 UI(룰 카테고리·법령 근거·표준 조항 포함) 즉시 전환.
- **감사 로그(Audit Trail)** — 케이스 생성·분석·코멘트·서명·버전 이벤트를 시간순 기록, 유형 필터, 관리자 전용.
- **계약 버전관리(diff 히스토리)** — 협상 반영본을 버전 스냅샷으로 저장, 버전 간 조항 추가·삭제·위험점수 변화 추적.
- v2 계승: 3역할 RBAC·케이스 보드·협업 코멘트·표준계약서 생성·CSV/JSON 입출력·검토 리포트 PDF(한글 보존, 법령 근거 반영)·localStorage 영속·반응형(드로어+바텀탭+시트).

**Out of Scope (외주사 미수행)**
- 서버·DB·실 계정/로그인·실 결제(시제품은 클라이언트 단일 HTML, RBAC·OAuth는 클라이언트 시뮬레이션).
- 실 LLM/외부 AI API 연동(v3도 결정론적 룰/알고리즘).
- 실 전자서명 사업자(KICA·모두싸인 등) 연동·법적 효력 보증·실제 법률 자문(전자서명·알림톡·webhook·법령매칭은 mock/내장 룰DB).
- 캡처 스크립트·문서 작성(발주자 자체 수행).

## 2. 과업의 세부 내용

### 2.1 기능 사양

| 기능 | 사양 |
|:---|:---|
| 룰셋 | **22종**(v2 20종 + 자동출금·담보·연대보증, 데이터·개인정보 권리) + 각 룰 `lawRef`(법령·판례 근거) 내장 |
| TF-IDF diff | 표준조항 코퍼스로 IDF 산출 → 어절+한글 bigram TF-IDF 벡터 코사인 유사도. ≥0.30=정합, <0.30=독소. 핵심 보호조항 부재 시 누락. 평균 코사인=표준 정합도(%) |
| 등급 캘리브레이션 | 고위험 조항 ≥3 시 임계 하향(고위험 52/중위험 26), 거의 없을 시 상향(68/36)으로 점수→등급 매핑 동적 보정. 분석 화면에 적용 임계 표기 |
| 협상 시뮬레이터 | 조항별 현행/표준대로수정 토글 → 수정 조항 가중을 18%로 낮춘 위험점수 즉시 재계산, before/after 막대 + 등급 변화, 선택 수정안 일괄 채택 |
| 법령/판례 매칭 | 22룰 각각에 KO/EN 법령·판례 근거(예: 위약금=민법 제398조, 지연이자=상법 제54조, IP=저작권법 제45·14조)를 협상 카드·PDF에 노출 |
| 전자서명 OAuth | OAuth 2.0 mock 핸드셰이크(authorize→callback state 검증→token), 연결 후 서명요청 활성, webhook `esign.completed` 콜백으로 완료 추적, 알림톡 로그 |
| 팀 협업 알림 | 코멘트 작성 시 `collab.comment.created` webhook + 팀 알림(2명) 로그 적재, 협업 화면에 실시간 로그 표시 |
| i18n | KO/EN 사전 기반, 사이드바·모바일 앱바 토글. UI·룰 카테고리·법령 근거·표준 조항·차트 라벨·PDF까지 전환, localStorage 언어 기억 |
| 감사 로그 | 케이스/코멘트/서명/버전 이벤트 시간순 기록(최대 300), 유형 필터 5종, 관리자 전용 뷰 + KPI |
| 버전관리 | 케이스별 버전 배열(스냅샷: 본문·위험점수·조항 시그니처), 버전 간 조항 add/remove·점수 변화 diff 표시 |
| RBAC | 의뢰인·검토자·관리자 3역할 스위치, 관리자=리포트·감사로그·CSV/JSON 노출 |
| PDF | jsPDF + 캔버스 한글 렌더, 협상 우선순위(ROI 순)·법령 근거·표준정합도 반영, KO/EN, 다중 페이지 |
| 영속 | localStorage `legalcheck_cases_v3`·`legalcheck_audit_v3`·언어, 새로고침 후 케이스·코멘트·서명·버전·감사 유지 |

### 2.2 UI/UX 사양

- 디자인 톤: [디자인_애플_토스.md](../../design-guide/디자인_애플_토스.md) — 브랜드 단일 포인트(신뢰 인디고), 위계=여백·타이포, WCAG AA 대비.
- 데스크톱: [디자인_PC.md](../../design-guide/디자인_PC.md) — 좌측 사이드바 256px(KO/EN·역할 스위처·섹션 그룹 13뷰) + 본문 `min-w-0`, KPI 4열·차트 2열.
- 모바일: [디자인_모바일.md](../../design-guide/디자인_모바일.md) — 사이드바 숨김·**햄버거 오프캔버스 드로어** + **바텀탭 5개**(보드·분석·시뮬·협상·더보기 시트). KPI 2열, 차트 고정높이, diff 좌우→상하 스택, 시뮬 토글 행 스택, 표 가로 스크롤, 탭 ≥44px, overflow 0.
- 화면 흐름(**13뷰**): 케이스 보드 → 계약서 입력 → 위험 분석 → 표준 대비 diff → 협상 우선순위 → 협상 시뮬레이터 → 표준계약서 생성 → 협업 코멘트 → 전자서명(OAuth) → 버전 관리 → 관리자 리포트 → 감사 로그 → 검토 이력.
- 워크플로(3개 이상): ① 입력→분석→diff→협상 채택/시뮬레이션→OAuth 서명(검토·협상·체결 파이프라인) ② 검토자 코멘트→팀 알림→해결→단계 전이(실시간 협업) ③ 케이스 포트폴리오→버전 스냅샷→감사 로그→CSV/JSON 입출력(거버넌스/백업).

### 2.3 라이브러리·의존성

| 항목 | 버전 | 비고 |
|:---|:---|:---|
| Chart.js | 4.x (CDN) | 위험분포·카테고리 가중·게이지·ROI 산점도·시뮬 before/after·관리자 분포 |
| jsPDF | 2.5.x (CDN) | 리포트 PDF(캔버스 한글 렌더, KO/EN, 법령 근거) |
| 외부 API 키 | **없음** | OAuth·전자서명·알림톡·webhook·팀 알림·법령매칭 전부 mock/내장, 키 하드코딩 금지 |

### 2.4 품질·성능 기준 (정량)

- 좁은 화면(390px) 가로 overflow **0px**, 콘솔 pageerror **0건**, 탭/버튼 터치 타깃 **≥44px**.
- 예시 계약서 분석(22룰 매칭+TF-IDF diff+ROI+캘리브레이션) **1초 이내**.
- 협상 시뮬레이터: 조항 토글 시 위험점수 **즉시(<100ms) 재계산** + before/after 차트 갱신.
- 전자서명 OAuth: 연결 전 서명요청 **비활성**, 연결 후 활성 + webhook 콜백 로그 **기록 확인 가능**.
- 코멘트 작성 시 팀 알림 webhook 로그 **적재 확인**.
- KO/EN 토글 시 전 UI(룰 카테고리·법령 근거·표준 조항·차트·PDF) **즉시 전환**, 한글/영문 깨짐 0.
- CSV/JSON 내보내기 **실제 다운로드**(한글 깨짐 0), PDF 한글 깨짐 **0건**.
- 새로고침 후 케이스·코멘트·서명·버전·감사 로그·언어 **유지율 100%**.
- 버전 스냅샷: 버전 간 조항 add/remove·위험점수 변화 **정합 표시**.
