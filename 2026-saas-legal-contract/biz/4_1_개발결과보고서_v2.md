## v2 신규/심화 산출물

### 1) 실 알고리즘 심화
- **가중치 모델:** `weight = severity(1~10) × typeWeight(계약유형별) × freqFactor(출현빈도) × qualFactor(정량신호: 위약금 %·연 24% 등) × levelCoef(등급계수)`. 종합점수 = Σweight × 0.62 정규화(0~100). 분석 화면에 산식·rawSum·정규화값을 노출.
- **표준 대비 토큰 유사도 diff:** 텍스트 정규화(소문자·기호제거·공백정리) → 토큰(어절 + 한글 bigram) **Jaccard 유사도**. 내 조항 ↔ 룰별 균형 표준 문구를 비교해 유사도 ≥0.34=정합, <0.34=**독소**로 분기. 핵심 보호조항(책임제한·비밀유지·불가항력)이 본문에서 발견 안 되면 **누락**으로 탐지. 평균 유사도=표준 정합도(%).
- **협상 ROI 랭킹:** ROI = 리스크 임팩트(weight) ÷ 교섭난이도(1~5). ROI 내림차순으로 우선순위 #1.. 부여, 임팩트–난이도 산점도로 시각화(좌상단=고ROI 협상 1순위).

### 2) 다중 사용자/다중 테넌트
- **3역할 RBAC**(의뢰인·검토자·관리자) 사이드바·드로어 스위처로 전환. 역할별 분기:
  - **검토자:** 보드 헤더 '검토자 작업 보드' + **검토자 작업 큐**(검토중·협상중 케이스), 코멘트가 '검토자'로 작성.
  - **관리자:** **관리자 리포트**(단계 분포·카테고리 누적·완료율·CSV/JSON 입출력) 전용 메뉴 노출.
  - **의뢰인:** 기본 워크스페이스.
- **케이스 관리 보드:** 다중 계약 케이스를 단계(초안·검토중·협상중·서명중·완료) 플래그·필터로 관리.

### 3) 외부 시스템 통합 2건 이상 (CDN·라이브러리 제외)
- **① 전자서명/알림톡 mock:** 서명 요청 발송 시 `POST /esign/v1/requests`·`KAKAO alimtalk send`·`WEBHOOK esign.completed` 로그를 실제로 적재, 서명자 상태(대기→발송됨→완료) 전이, 전원 서명 시 케이스 '완료'.
- **② CSV/JSON 외부 입출력:** 케이스 포트폴리오를 **CSV**(BOM+따옴표 이스케이프, 한글)·**JSON 백업**으로 실제 파일 다운로드, **JSON 가져오기**로 외부 백업을 병합(id 중복 제외).

## 2. 구현·제작 범위

- **10개 뷰** 실동작: ① 케이스 보드 ② 계약서 입력 ③ 위험 분석 ④ 표준 대비 diff ⑤ 협상 우선순위 ⑥ 표준계약서 생성 ⑦ 협업 코멘트 ⑧ 전자서명 ⑨ 관리자 리포트 ⑩ 검토 이력.
- **워크플로 3개:** (a) 입력→분석→diff→협상 채택→서명 검토 파이프라인, (b) 검토자 코멘트→해결→단계 전이 협업, (c) 케이스 포트폴리오→CSV/JSON 입출력 관리/백업.
- **심화 알고리즘:** 가중치 모델, 토큰(어절+한글 bigram) Jaccard 유사도 diff, 협상 ROI 랭킹.
- **다중 사용자:** 의뢰인·검토자·관리자 3역할 RBAC, 케이스 보드(다중 케이스·단계).
- **외부 통합 2건:** 전자서명/알림톡 mock 발송 로그, CSV/JSON 입출력(가져오기 병합 포함).
- **PDF/영속:** jsPDF 캔버스 한글 렌더(협상 우선순위·표준정합도 반영), localStorage 케이스 영속(새로고침 유지).

## 4. 실행/구동 방법

```bash
# 1) 앱 실행 — 브라우저로 v2.html 열기 (빌드 불필요)
open projects/legal-check/v2.html

# 2) 캡처 재현 (선택)
cd projects/legal-check
npm i -D playwright && npx playwright install chromium   # 또는 글로벌 chromium 사용
node capture.mjs            # PC→biz/captures/v2, 모바일→biz/captures/mobile/v2
rm -rf node_modules package*.json
```

앱 사용: ‘데모 케이스 채우기’로 보드 채움 → ‘계약서 입력’ → ‘예시 불러오기’ → ‘조항 분석 시작’ → 분석·표준 diff·협상 우선순위 확인 → 역할을 검토자로 바꿔 코멘트 → 의뢰인으로 전자서명 발송·완료 → 관리자로 CSV/JSON 입출력.
