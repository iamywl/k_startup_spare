## 1. 과업의 개요

**과업의 목적**

1) 수출 초보 중소·소상공인이 매 수출 건마다 마주치는 4관문(HS코드 분류·FTA 원산지 판정·수출신고서·관세 환급)을 **품목 1회 입력으로 릴레이 처리**하는 웹 시제품(PoC) 1식을 외주로 제작한다.
2) 본 PoC 는 [`1_제안서.md`](./1_제안서.md) §S-3 핵심 워크플로(그림 4)와 §S-4 아키텍처(그림 5)를 실 구현으로 입증해, 실전창업 심사·베타 시연·시리즈 검증의 근거(PoC)로 쓴다.

**과업의 범위**

수행 업무(개발계획서 WBS 외주 행과 1:1):
1) 수출 대시보드 — 단계별 건수·미환급 금액·FTA 절감액 KPI + 추세 차트
2) 품목·HS코드 분류 보조 — 품목 설명 → 후보 HS(6/10자리) + 신뢰도 + 휴먼검수 확정
3) FTA 원산지 판정 — 협정 선택 → 결정기준 체크 → 충족 여부·절감액 산정 → 원산지증명서(C/O) 초안
4) 수출신고서·커머셜 인보이스 자동생성 — 품목·금액·원산지 자동 채움 → PDF
5) 관세 환급 추적 — 수출완료 건 → 간이정액환급 추정 → 신청 상태 전이 추적
6) 데이터 입출력 — 주문 CSV 업로드 일괄 분류 / 신고·환급 내역 CSV 내보내기
7) 공통: localStorage 상태 지속, 정합 점검, 외부 API(UNI-PASS 등) mock 폴백, 반응형(PC·모바일)

Out of Scope (v1 미포함):
- 실 백엔드·DB 서버, 다기업 테넌트, 실 사용자 인증/결제
- 관세청 UNI-PASS·품목분류 API 실 연동(정식 버전 — v1 은 mock)
- 실 관세율·HS 전체 DB 적재(v1 은 데모용 대표 품목·협정 룰셋)

## 2. 과업의 세부 내용

- **형태:** 단일 자체완결 HTML(`projects/trade-customs/index.html`), 오프라인 `file://` 단독 구동. 빌드 불필요.
- **화면(뷰) 6종 이상:** 대시보드 / 품목·HS / 원산지 / 신고서 / 환급 / 데이터(설정). 각 뷰는 단순 표시가 아니라 인터랙션 가능.
- **다단계 워크플로 1개 이상:** 품목 등록 → HS 분류·확정 → 원산지 판정 → 신고서 생성 → 환급 추적(상태 전이).
- **HS 분류 로직:** 품목 설명 키워드 → 룰·점수 매칭으로 후보 HS 다건 + 신뢰도(%) + 선택 사유. 사용자가 최종 확정(휴먼인더루프). 자기주장 AI 아님 — 룰 근거 노출.
- **원산지 판정 로직:** 협정별 결정기준(예: 세번변경·부가가치) 체크리스트 → 충족 시 절감액(적용 전후 관세 차) 산정 → C/O 초안 필드 채움.
- **문서 생성:** jsPDF 로 수출신고서·커머셜 인보이스·C/O 초안 PDF 실 생성(한글 깨짐 없이). CDN 부재 시 안내 폴백.
- **CSV:** 주문 CSV 업로드 → 행별 품목 자동 분류 / 신고·환급 내역 CSV 내보내기(외부 입출력 실동작).
- **라이브러리/의존성:** jsPDF(CDN, 폴백 처리) 외 추가 의존성 최소화. API 키 하드코딩 금지, 키 부재 시 mock.
- **품질·성능 기준(정량):** 좁은 화면(≈390px) 가로 overflow 0, 콘솔 pageerror 0, 새로고침 후 상태 유지, 핵심 액션은 토스트 mock 이 아니라 실 데이터 변경.
