## v2 신규/심화 산출물

1. **다중 테넌트 포트폴리오(`02`)** — 관세사 한길관세사무소가 화장품·패션·건기식 3개 수출기업을 관리. 기업별 수출 건수·FTA 절감·환급 추정·신고 완료율 미터를 한 화면에. 상단 셀렉터로 전체↔개별 테넌트 전환 시 대시보드·목록이 그 기업 데이터로 좁혀진다.
2. **RBAC 역할 전환(`11`·`10`)** — 사이드바 셀렉터로 수출기업/관세사/포워더 전환. 수출기업은 자사(글로우셀러) 데이터만, 포워더는 선적·신고서만(환급·신고·테넌트 메뉴 비노출). 권한 밖 뷰는 자동으로 허용 첫 뷰로 우회.
3. **HS 가중 점수 엔진(`03`)** — "데님 청바지 jeans denim 팬츠 바지" 입력 → 6204.62 신뢰도 97%(매칭 근거 6개 노출). 음성키워드로 인접 품목 오분류를 감점 처리. 후보별 신뢰도 미터.
4. **다협정 절감 최적화(`04`)** — 베트남행 건은 RCEP·한아세안FTA 두 협정을 비교, 절감액·RVC 문턱·발급유형 순으로 추천 협정을 띄운다. 중국행은 한중FTA·RCEP 비교.
5. **원산지 RVC 정량 판정(`05`)** — 협정 문턱(예: 한아세안 40%) 대비 입력 RVC 를 미터로 비교, CTH 체크와 OR 결합해 충족/미충족 + 마진(%p) 산출.
6. **착지원가 시뮬레이션(`06`)** — 상품가+물류비(중량×국가요율)+보험+관세를 MFN/FTA 적용 전후 표로 비교, FTA 관세 절감액을 강조.
7. **환급 정밀 산정(`08`)** — 동일 수출금액이라도 HS 품목군별 1만원당 환급액이 달라 건별 환급액이 다르게 계산된다(3304 vs 6109 비교 가시).
8. **외부 연동 4건 + 감사 로그(`07`·`09`)** — UNI-PASS 전송 시 접수번호 발급·접수→수리, 환급 접수번호, 환율 실시간 드리프트, 포워더 B/L·ETA. 모든 전송이 통합 로그(채널 필터)에 아티팩트로 남는다.

## 4. 실행/구동 방법

1. `projects/trade-customs/v2.html` 을 브라우저로 연다 (더블클릭 또는 `file://`).
2. 기본 관세사 역할로 시드(3개 수출기업 6건)가 표시된다. 사이드바(모바일=더보기 드로어)에서 역할·테넌트를 전환한다.
3. 워크플로 시연: 품목·HS → 가중 점수 분류 확정 → FTA 원산지·최적화(다협정 비교·RVC 판정) → 수출신고서 생성·착지원가 시뮬 → UNI-PASS 전자신고 전송→수리 → 관세 환급 접수 → 포워더 선적 부킹. 연동 로그에 전 전송 아티팩트가 누적된다.
4. 새로고침 후에도 진행 상태·역할·테넌트가 유지된다(localStorage). 데이터·설정에서 초기화 가능.
5. 재캡처: `cd projects/trade-customs && node capture-v2.mjs` (워크스페이스 루트 `node_modules` 심링크로 playwright 해석). v1 데모는 `node capture.mjs` 로 별도 보존.

## 7. 미흡·잔여 사항

- 외부 연동(UNI-PASS·환급·환율·포워더)은 **mock 전송 로그**로, 정식 버전은 관세청 전자인증서·실 API·실 환율 피드로 승격한다(v3 범위).
- HS·관세율·환급 고시는 데모 대표 품목·협정 룰셋 확장판이며, 정식 버전은 관세청 품목분류 사례·전 품목 관세율 DB 적재가 필요하다.
- 인증·결제·실 백엔드 다중 테넌트 분리(현재는 클라이언트 localStorage 시뮬)는 정식 버전 과업.
