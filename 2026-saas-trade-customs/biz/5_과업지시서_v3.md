## 1. 과업의 개요

**과업의 목적**

1) v2(다중테넌트·실 알고리즘 5종·RBAC 3역할·외부연동 4건)를 **시리즈A 글로벌 확장 데모**로 끌어올린다. 구체적으로 ① 관세 의사결정을 돕는 **고급 알고리즘 4종**(관세율 변동 시나리오·다국가 공급망 원산지 누적·FTA 사후검증 리스크·수출 채산성 최적화), ② 해외 바이어·영문 서류 대응을 위한 **KO/EN 다국어(i18n)**, ③ 해외바이어·은행을 포함한 **역할 5종·외부연동 7건**(바이어 포털·무역금융 L/C·선사 EDI 추가), ④ **뷰 15종·워크플로 5개** 로 확장하는 것을 외주 과업으로 위탁한다.
2) 본 산출물은 [`1_제안서.md`](./1_제안서.md)의 GTM(B2B 관세사 채널 + 직수출기업)·글로벌 확장 로드맵을 데모로 입증해, **시리즈A 투자·해외 바이어 연계·은행 무역금융 제휴**의 근거로 쓴다.

**과업의 범위**

수행 업무(개발계획서 WBS v3 외주 행과 1:1):
1) **고급 알고리즘 4종(실 연산)** — ① 관세율 변동 시나리오 시뮬(현행·FTA실효·부분복귀·추가관세 시나리오별 착지원가·마진율) ② 다국가 공급망 원산지 누적(roll-up) 판정(부품 트리 역내/역외 집계 → RVC) ③ FTA 사후검증 리스크 스코어(가중 요인 0~100) ④ 수출 채산성 최적 협정·국가 추천(도착국×협정 순채산 정렬)
2) **KO/EN 다국어(i18n)** — 언어 토글(KO↔EN), 대시보드·메뉴·핵심 뷰·서류 라벨 번역, 영문 PDF
3) **역할 5종(RBAC 확장)** — v2의 수출기업/관세사/포워더 + **해외바이어·은행** 2역할 추가(역할별 메뉴·데이터·권한 분기, 바이어 진입 시 자동 EN)
4) **외부 연동 7건(mock, 아티팩트 로그)** — v2의 UNI-PASS·환급·환율·포워더 4건 + **⑤ 무역금융 L/C 개설(SWIFT MT700) ⑥ 해외 바이어 포털 접근 ⑦ 선사 EDI 적하목록(Manifest)** 3건. 통합 감사 로그(채널 7종 필터)
5) **신규 뷰·워크플로** — 관세율 시나리오·공급망 누적·채산성 최적화·무역금융(L/C)·바이어 포털·사후검증 6뷰 추가(총 15뷰), L/C 개설→선적→네고 워크플로 추가(총 5워크플로)
6) 공통: localStorage(`traderelay-v3`) 상태 지속, 반응형(PC·모바일), 키 없이 mock 동작, 로그인 없이 즉시 시연, v1/v2 불변 보존

## 2. 과업의 세부 내용

- **형태:** 단일 자체완결 HTML(`projects/trade-customs/v3.html`), 오프라인 `file://` 단독 구동. v2(`v2.html`)를 base 로 복제·확장하되 별도 파일·별도 localStorage 키(`traderelay-v3`). 빌드 불필요.
- **화면(뷰) 15종:** 대시보드 / 관리 수출기업 / 품목·HS / FTA 원산지·최적화 / **관세율 시나리오** / **공급망 원산지 누적** / **채산성 최적화** / 수출신고서 / UNI-PASS 전자신고 / 관세 환급 / 포워더 선적 / **무역금융(L/C)** / **해외 바이어 포털** / **FTA 사후검증** / 연동 로그·설정. 역할별 노출 분기.
- **다단계 워크플로 5개:**
  - ① 분류→다협정 최적화→원산지 정량 판정→신고서→UNI-PASS 전송→수리(전자신고)
  - ② 신고서→환급 접수(mock)→심사→지급(환급)
  - ③ 신고서→포워더 선적 부킹→B/L·ETA·선사 EDI 송신(선적)
  - ④ **L/C 개설(MT700)→선적→선적서류 매입(네고)(무역금융)**
  - ⑤ **공급망 부품 누적→원산지 반영→사후검증 리스크 점검(원산지 검증대응)**
- **고급 알고리즘(정량 연산, 토스트 mock 아님):**
  - 관세율 시나리오: 시나리오별 `관세 = 상품가 × 변동관세율`, `착지원가 = 상품가 + 물류비 + 보험 + 관세`, `마진율 = (목표판매가 − 착지원가)/목표판매가`. 목표판매가 = 상품가×(1+목표마진 30% `[추정]`).
  - 공급망 누적 roll-up: `RVC = 역내가치 / 총가치 ×100`, 역내(국내)+누적가능국 부품을 역내로 인정, 협정 문턱 비교·마진(%p).
  - 사후검증 리스크: `score = Σ(가중요인)` — RVC 마진 얇음↑, CTH 미충족·자율발급↑, 서류 미비↑, 누적 의존↑ → 0~100, 낮음/보통/높음.
  - 채산성 최적화: 도착국×협정별 `순채산 = 목표판매가 − 착지원가 + 환급` 내림차순 추천.
- **i18n:** `I18N{ko,en}` 사전 + `t(key)`/`L()` 헬퍼. 언어 토글(사이드바·모바일 상단·드로어). 바이어 역할 진입 시 자동 EN. 영문 PDF(바이어·L/C 필드 포함).
- **외부 연동 mock(7건):** 각 연동은 접수번호/참조번호/SWIFT/로그 라인을 **아티팩트로 남긴다**(연동 로그 뷰 채널 7종 필터). CDN·라이브러리는 연동으로 카운트하지 않는다.
- **문서 생성:** jsPDF 로 수출신고서·인보이스·C/O 초안 PDF(수출기업·관세사·UNI-PASS·RVC·Landed·**바이어·L/C(SWIFT)** 필드). CDN 부재 시 텍스트 폴백.
- **품질·성능 기준(정량):** 좁은 화면(≈390px) 가로 page overflow 0(넓은 차트·표는 카드 내부 가로 스크롤), 콘솔 pageerror 0, 새로고침 후 상태·역할·테넌트·언어 유지, 핵심 액션은 실 데이터/실 연산 변경.
