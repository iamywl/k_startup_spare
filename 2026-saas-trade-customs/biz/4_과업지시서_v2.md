## 1. 과업의 개요

**과업의 목적**

1) v1 PoC(단일 수출기업·룰 기반·오프라인)를 **베타/시리즈A 데모 수준**으로 끌어올린다. 구체적으로 ① 관세사 1곳이 다수 수출기업을 관리하는 **다중 테넌트** 구조, ② 수출기업·관세사·포워더 **역할별 권한(RBAC)** 화면, ③ HS 분류·FTA 절감·관세/물류비·환급의 **실 알고리즘 정교화**, ④ 관세청 UNI-PASS·환급·환율·포워더 **외부 시스템 연동(mock)** 을 신규 과업으로 위탁한다.
2) 본 산출물은 [`1_제안서.md`](./1_제안서.md) §S-4 아키텍처(그림 5)의 "정식 버전 확장 경로"(백엔드 API·다기업 테넌트·UNI-PASS 전자신고·환급 연계)를 데모로 입증해, 시리즈A·B2B 관세사 채널 영업의 근거로 쓴다.

**과업의 범위**

수행 업무(개발계획서 WBS v2 외주 행과 1:1):
1) **다중 테넌트** — 관세사 ↔ 다수 수출기업(테넌트) 데이터 분리, 테넌트 셀렉터, 관리 수출기업 포트폴리오 뷰(기업별 절감·환급·신고완료율)
2) **RBAC 역할 전환** — 수출기업/관세사/포워더 3역할, 역할별 메뉴·데이터 가시성·액션 권한 분기
3) **HS 분류 실 알고리즘** — 키워드 가중 점수(핵심어 3·보조어 1) + 오분류 방지 음성키워드 감점 + 신뢰도 정규화·미터 시각화
4) **다협정 절감 최적화** — 도착국에 적용 가능한 모든 협정을 전수 비교해 절감 최대 협정 추천(절감액·RVC 문턱·발급유형·양허 잔여기간 정렬)
5) **원산지 RVC 정량 판정** — 입력 RVC vs 협정 문턱 비교, CTH(세번변경) '또는' RVC 결정 로직, 마진(%p) 산출
6) **관세·물류비 착지원가(Landed Cost) 시뮬레이션** — 상품가+물류비(중량×요율)+보험+관세를 MFN/FTA 적용 전후로 비교
7) **환급 정밀 산정** — 품목군(HS 4단위)별 1만원당 환급액 고시(mock) 기반 산정
8) **외부 연동 4건(mock, 아티팩트 로그)** — ① UNI-PASS 전자신고 전송(접수번호·접수→수리) ② 관세청 환급 접수(접수번호) ③ 환율 API 실시간 갱신(드리프트) ④ 포워더 선적 부킹(B/L·ETA). 통합 감사 로그 + 채널 필터
9) 공통: localStorage(`traderelay-v2`) 상태 지속, 반응형(PC·모바일), 키 없이 mock 동작, 로그인 없이 즉시 시연

Out of Scope (v2 미포함):
- 실 백엔드·DB 서버·실 사용자 인증/결제(연동은 mock — 정식 버전 과업)
- 관세청 UNI-PASS·품목분류·환급 **실 API** 연동(키·전자인증서 필요 — v2 는 mock 전송 로그)
- HS 전체 DB·전 품목 관세율 적재(v2 는 데모 대표 품목·협정 룰셋 확장판)
- v1.html(`index.html`) 수정 — v1 데모는 불변 보존, v2 는 `v2.html` 신규 파일

## 2. 과업의 세부 내용

- **형태:** 단일 자체완결 HTML(`projects/trade-customs/v2.html`), 오프라인 `file://` 단독 구동. v1(`index.html`)을 base 로 복제·확장하되 별도 파일·별도 localStorage 키. 빌드 불필요.
- **화면(뷰) 8종 이상:** 대시보드 / 관리 수출기업 / 품목·HS / FTA 원산지·최적화 / 수출신고서 / UNI-PASS 전자신고 / 관세 환급 / 포워더 선적 / 연동 로그·설정 (역할별 노출 분기). 각 뷰 인터랙션 가능.
- **다단계 워크플로 3개 이상:**
  - ① 분류→다협정 최적화→원산지 정량 판정→신고서→UNI-PASS 전송→수리(전자신고)
  - ② 신고서→환급 접수(mock)→심사→지급(환급 상태 전이)
  - ③ 신고서→포워더 선적 부킹→B/L·ETA 발급(선적)
- **실 알고리즘(정량 연산, 토스트 mock 아님):**
  - HS 분류: `score = Σ(키워드 가중치) − Σ(음성키워드 2점)`, 후보별 정규화 신뢰도(상한 97%).
  - 다협정 최적화: 적용 협정별 `절감 = MFN% × 수출금액` 내림차순 + 동률 시 양허 잔여기간·발급난이도 보조 정렬.
  - 원산지: `충족 = CTH ∨ (RVC ≥ 협정문턱)`, 마진 = RVC − 문턱.
  - 착지원가: `상품가 + 물류비(중량×국가요율) + 보험(0.3%) + 관세(MFN 또는 0)`.
  - 환급: 품목군(HS4)별 1만원당 환급액 × (수출금액원/1만).
- **다중 테넌트·RBAC:** 관세사 역할은 전체/개별 테넌트 전환, 수출기업 역할은 자사 데이터만, 포워더 역할은 선적·신고서 한정(환급·신고 접근 차단).
- **외부 연동 mock:** 각 연동은 접수번호/참조번호/로그 라인을 **아티팩트로 남긴다**(연동 로그 뷰에서 채널 필터 조회). CDN·라이브러리는 연동으로 카운트하지 않는다.
- **문서 생성:** jsPDF 로 수출신고서·인보이스·C/O 초안 PDF 생성(수출기업·관세사·UNI-PASS 접수번호·RVC·Landed Cost 필드 포함). CDN 부재 시 텍스트 폴백.
- **CSV:** 주문 CSV 업로드 → 가중 점수 엔진 자동 분류 일괄 등록(현재 테넌트) / 신고·환급 내역 CSV 내보내기(테넌트·RVC·UNIPASS 포함 14열).
- **품질·성능 기준(정량):** 좁은 화면(≈390px) 가로 overflow 0, 콘솔 pageerror 0, 새로고침 후 상태·역할·테넌트 유지, 핵심 액션은 실 데이터/실 연산 변경.
