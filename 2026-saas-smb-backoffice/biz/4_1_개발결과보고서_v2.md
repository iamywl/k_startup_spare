# 개발결과보고서 v2 — SMB 통합 백오피스 SaaS

> v2 사이클: v1(1천만원 PoC) → **5억 베타** 격상. 한글 PDF·다중 테넌트·QR 계약 검증·ICS·알림톡 mock API·직원 포털·노무사 챗을 모두 실 구현.
> 신규 산출물: [`projects/smb-backoffice/v2.html`](../projects/smb-backoffice/v2.html) + [`app.js`](../projects/smb-backoffice/app.js) + [`nanum-font.js`](../projects/smb-backoffice/nanum-font.js)(NanumGothic Base64 번들). v1(`index.html`)은 미수정.

## v1 한계 및 v2 개선 매핑

| v1 한계 | v2 개선 방식 | 증거 캡처 |
|:---|:---|:---|
| PDF 본문이 **영문(transliteration)** — 인명·조항 latin 표기 | jsPDF에 **NanumGothic TTF Base64 임베드**(`addFileToVFS`+`addFont`). 계약서/급여명세 본문·인명·조항 전부 한글 정상 렌더. 로컬 번들 우선, 실패 시 Google Fonts TTF fetch 폴백 | `05_workflow_korean_pdf.png`, `05b_korean_pdf_zoom.png` |
| **단일 회사**만 지원 | **다중 회사(테넌트) 스위처** — (주)스몰박스/봄날카페/테크랩 3사. 회사별 직원·계약·급여·알림톡 데이터 완전 분기. 전환 시 전 화면 즉시 갱신 | `01_dashboard.png`, `02_tenant_switch_techlab.png` |
| 카카오 알림톡 **로그만** 존재(API 흐름 없음) | NHN Toast 알림톡 REST 엔드포인트 형태의 **mock API 요청/응답 흐름** — POST 바디·헤더·HTTP 200·`MESSAGE_ACCEPTED` 응답 시뮬레이션 + 로그 적재 | `07_workflow_kakao_api.png`, `16_kakao_api_log.png` |
| 계약 **위·변조 검증 수단 없음** | **서명 해시(djb2+FNV-1a 혼합 16자리) → QR 인코딩**. 검증 페이지에서 토큰 입력 시 유효/불일치 판정 | `04_contract_qr_modal.png`, `06_workflow_sign_qr.png`, `08_verify_valid.png`, `09_verify_tampered.png` |

## 2. 구현/제작 범위

- 단일 HTML(`v2.html`) + 외부 로직(`app.js`) + 폰트 번들(`nanum-font.js`) — 빌드 불필요, `file://` 오프라인 자체완결.
- **11개 라우트**: 대시보드 / 전자계약 / 계약 워크플로 / QR 검증 / 근태·캘린더 / 급여 / 직원 관리 / 직원 대시보드 / 알림톡 발송 / 노무사 챗 / 문서함.
- **워크플로 3개+**: ① 계약 발행 4단계(템플릿→변수→한글PDF→서명·QR·발송) ② QR 검증(코드 입력→해시 대조→판정) ③ 직원 포털 전자서명(계약 선택→캔버스 서명→계약 확정).
- **3개 테넌트** 데이터 격리, 영속 계층 `localStorage[smb_state_v2]`.
- 로그인 없이 게스트 모드 (CLAUDE.md §3.4).
