# 과업지시서 v3 — SMB 통합 백오피스 SaaS

> v3 위탁 분: **국세청 e-세금계산서 연동 인증·다국어 번역 감수·노무사 룰 확장 자문** 가정. SW 본체(8가지)는 자체.
> 제안서 [§8 로드맵](./1_제안서.md) v3(M7~12, 시리즈A 데모) 행을 구현 범위로 삼는다.

## 1. 과업의 개요

### 과업의 목적

1) v2 5억 베타 통과 후, **시리즈A 데모(수준)** 단계로 격상.
2) 제안서 §8 v3 로드맵(노무사 자문 챗·전자세금계산서 발급 API·다국어(한·영)·근태/급여 자동화)을 실제 동작하는 단일 데모로 입증.
3) v2 한계 — 세금계산서 미지원·단일 언어·룰 단순·연차 미정산·급여 이체 수작업·권한 무·분석 평면 — 해소.
4) 외주에 맡기는 직접 목적: **국세청 e-세금계산서 발급 인증 연동(인증서·API 키), 영문 번역 감수, 노무 도메인 룰 정합성 검토** — 발주자 자체 SW에 결합할 외부 전문 영역.

### 과업의 범위

1) 외주 수행 업무 (가정)
   - **국세청 홈택스/팝빌 e-세금계산서 발급 API 인증 연동** — 사업자 공동인증서 등록·API 키 발급·정발행 검증(샌드박스 1건)
   - **다국어(영문) UI 카피 번역 감수** — 회계·노무 도메인 용어 영문 표준화(Supply value/VAT/Statutory leave 등)
   - **노무사 자문 룰 정합성 검토** — 확장 12개 카테고리의 근거 조항·수치 감수(2026 개정 반영)
2) 범위 외 (Out of Scope) — 자체 수행
   - 전자세금계산서 세액 산출·발급내역 UI·발급 API mock 흐름
   - i18n 토글 엔진·번역 키 구조
   - 노무사 챗 룰 엔진·후속질문 로직
   - 연차/근태 정산·펌뱅킹 CSV·분석 차트·권한/감사로그·다중회사 콘솔
   - 한글 PDF (v2 NanumGothic 번들 재사용)

## 2. 과업의 세부 내용

- 세금계산서 API: 국세청 e-세금계산서 표준(정발행 issueType=01, 과세/영세율/면세 구분, 공급가액·세액·합계). 인증 부재 시 자체 mock 흐름으로 대체 가능해야 함(오프라인 데모 보장).
- 영문 번역: 회계(e-Tax Invoice, Supply value, VAT, NTS code)·노무(Statutory leave, Severance, Audit log)·메뉴 카피 전반. KO/EN 1:1 키 매핑.
- 노무 룰: 연차(근기법 §60)·최저임금(최임법 §6)·수습(최임령 §3)·해고·부당해고(근기법 §23·26·27·28)·4대보험·주휴(§55)·퇴직금(퇴직급여법 §8)·임금체불(§36·43)·육아휴직(고평법 §19)·통상임금(근기령 §6)·연장근로(§56)·직장내괴롭힘(§76의2·3).
- 권한 모델: 오너/관리자/직원 3역할, 라우트별 접근 제어 + 활동 감사로그.

## 5. 성과품 목록

| 성과품 | 수량 | 형태 | 검수 합격 조건 |
|:---|:---|:---|:---|
| e-세금계산서 인증 연동(외주 가정) | 1 | 키 + 검증서 | 샌드박스 정발행 1건 승인 (인증 부재 시 mock 흐름으로 대체 입증) |
| 영문 번역 감수표(외주 가정) | 1 | CSV/XLSX | 회계·노무 도메인 용어 표준화 |
| 노무 룰 검토 의견서(외주 가정) | 1 | PDF | 12개 카테고리 근거 조항 정합 |
| **자체 산출물 — v3 SaaS 본체** | 1 | URL(`v3.html`) | 아래 v3 자체 산출물 핵심 8가지 모두 포함·실동작 |

> **v3 자체 산출물 핵심 8가지** (검수 기준 = 측정 가능):
> 1. **전자세금계산서 발급 API 시뮬레이션** — 공급가액 입력 → 세액(10%) 자동계산 → 국세청 e-세금계산서 발급 API(mock) POST 요청/응답(승인번호 NTS-AC-xxxx) → 발급내역 적재 + 한글 PDF. 검수: 7,000,000 입력 시 세액 700,000·합계 7,700,000, 응답에 ntsConfirmNum 표시.
> 2. **다국어(한·영) i18n 토글** — 주요 화면 KO/EN 전환(메뉴·KPI·폼·차트 라벨·회사명/직원명). 검수: EN 전환 시 nav가 Analytics/e-Tax Invoice/Payroll 등 영문, 토글로 즉시 복귀.
> 3. **노무사 자문 챗 고도화** — 12개 룰 + 근거 조항 인용(📎 근로기준법 제N조) + 후속질문 칩(↳). 검수: '임금체불' 질의 시 근기법 제36·43조 + 후속칩 노출.
> 4. **연차·근태 자동 정산 + ICS** — 근속연수 기반 법정 연차 자동 산정(근기법 §60), 잔여/소진/미사용 정산금 계산, 연차 ICS 내보내기. 검수: 3년차 16일·1년차 15일, ICS VEVENT 생성.
> 5. **급여 이체 파일(은행 펌뱅킹 CSV) 생성** — 4대보험·소득세 공제 후 실지급액 → 표준 펌뱅킹 대량이체 CSV(순번·은행·계좌·예금주·금액·적요) + 합계행. 검수: 4건 + 합계행, BOM 포함 한글 정상.
> 6. **대시보드 분석 고도화** — 인건비 6개월 추세 바차트 + 4대보험 사업주 부담 구성(국민연금/건강+장기요양/고용/산재) 스택바 + 월 총부담. 검수: 부담 합계 = 항목 합, 회사 전환 시 재계산.
> 7. **권한/감사로그** — 오너/관리자/직원 3역할, 라우트별 접근 제어, 활동 감사로그(역할 배지·시각·활동·상세). 검수: 직원 역할 시 nav 3개로 축소, 세금계산서 발급/CSV 생성이 감사로그에 자동 기록.
> 8. **다중 회사 통합 관리 콘솔** — 3사 직원·인건비·사업주 보험·세금계산서 합산 비교 + 회사별 카드 전환. 검수: 총 직원 12명·총 인건비 합산, 카드 전환 시 활성 회사 변경.

각 성과품의 외부 통합 검수: ① e-세금계산서 API mock 요청/응답 흐름 ② 펌뱅킹 CSV 외부 출력 ③ ICS(.ics) 외부 캘린더 출력 — **외부 통합 3건+** 충족.

## 변경 이력 (v2 → v3)

| 구분 | v2 | v3 변경점 |
|:---|:---|:---|
| 세금계산서 | 없음(문서함에 정적 카운트만) | **전자세금계산서 발급 모듈 신설** — 세액 자동산출 + 국세청 발급 API mock 흐름 + 발급내역 + PDF |
| 언어 | 한국어 단일 | **i18n(KO/EN) 토글** 신규 |
| 노무사 챗 | 8룰, 근거 단문 | **12룰 + 근거 조항 인용 + 후속질문 칩** 으로 고도화 |
| 연차/근태 | 출퇴근 예시 표 + 시프트 ICS | **근속 기반 연차 자동 정산 + 미사용 정산금 + 연차 ICS** 신규 |
| 급여 | 명세 계산 + PDF | **은행 펌뱅킹 대량이체 CSV 생성** 추가 |
| 대시보드 | KPI 4 + 패널 | **인건비 추세·4대보험 부담 구성 차트**로 분석 고도화 |
| 권한 | 게스트 단일 | **오너/관리자/직원 RBAC + 감사로그** 신규 |
| 다중 회사 | 스위처만 | **통합 관리 콘솔(합산·비교)** 신규 |
| 산출물 | `v2.html`+`app.js` | **`v3.html`+`app-v3.js` 신규** (v1·v2 파일 미수정, nanum-font.js 재사용) |
