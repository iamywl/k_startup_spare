## 1. 과업의 개요

### 과업의 목적

1) **직접 목적**: 원데이클래스 양면 마켓플레이스 "클래스나우"의 **거래 코어**(예약 상태머신·에스크로 결제 mock·취소/환불 정책 엔진·추천/평점/수요 랭킹 알고리즘·정산 수수료 엔진·PDF/CSV 입출력)를 단일 HTML 자체완결 시제품으로 제작·납품한다.
2) **제안서 연결**: [`1_제안서.md`](./1_제안서.md) §2 Solution 이 정의한 "호스트의 빈 시간을 한 장의 클래스로 만들고 게스트가 검색→예약→결제→후기까지 끝내는 양면 거래 플랫폼"의 핵심 구매동인(즉시 예약·신뢰·자동 정산)을 실 동작으로 입증하는 PoC. 양면시장의 매칭·신뢰·정산 통합(§경영혁신 프레임워크)이 데모에서 실제로 구현됨을 보인다.

### 과업의 범위

1) **외주사 수행 업무**
   - 예약 4단계 상태머신(슬롯 선택 → 인원 → 결제수단 → 확정) + 슬롯 정원 차감·영속
   - 에스크로 결제 mock(카드·카카오·토스·계좌이체 선택 → PG 승인 모사 → 예약코드 발급)
   - 취소·환불 정책 엔진(수업 3일 전 100% / 1일 전 50% 등 차등 환불율 자동계산 + 슬롯 환불 복원)
   - 콘텐츠기반 추천 점수 엔진(관심 카테고리·내 지역·잔여석·인기 결합), 추천 근거(reason) 태그
   - 베이지안 평점 엔진(원점수 vs 신뢰 보정 평점 병기), 수요/인기 랭킹 엔진(예약률·조회·찜 결합)
   - 정산 수수료 엔진(플랫폼 수수료 12% + PG 결제수수료 2.9% 차감 → 순지급액 → 주간 지급 스케줄)
   - 영수증 PDF·정산서 PDF(jsPDF, 한글 정상 렌더) + 카탈로그 CSV 내보내기(UTF-8 BOM)
2) **범위 외 (Out of Scope)** — 외주사가 수행하지 않음
   - 실 PG사(토스페이먼츠·카카오페이) 라이브 연동 및 정산 송금(실 자금 이동 없음, mock 승인까지)
   - 회원가입·본인인증·사업자 검증 등 정식 인증(시제품은 무로그인 3역할 전환, `CLAUDE.md` §3.4)
   - 서버·DB·앱스토어 배포(단일 HTML·localStorage 영속으로 갈음)
   - 호스트/게스트 화면 UI 골격 디자인(발주자 자체 분, 외주는 거래 로직 + 결과 화면 연동)

## 2. 과업의 세부 내용

### SW 사양
- **형태**: 단일 `v1.html`(빌드리스 바닐라 JS), 외부 인프라 없이 브라우저 단독 구동. 키 부재 시 mock fallback.
- **역할 모델**: 게스트(수강생)·호스트(강사)·관리자 3역할, 무로그인 전환. 역할별 네비게이션·홈 뷰 분기.
- **상태 영속**: 전 거래 상태(예약·정원·찜·리뷰·정산·정책)를 localStorage JSON 으로 직렬화, 새로고침 후 복원.

### UI/UX 제작 사양 (거래 흐름)
- 예약: 우측 슬라이드 시트의 4단계 스텝퍼, 각 단계 검증(잔여석·최대 인원). 결제 요약에 단위경제(수강료·할인) 표기.
- 환불: 취소 시 정책 기준 환불율·환불액을 화면에 명시하고 슬롯 정원을 복원.
- 정산: 호스트 정산 화면에 거래액·플랫폼 수수료·PG 수수료·순지급액을 항목별로 분해, 주간 지급 스케줄 표시.
- 디자인 가이드는 발주자 `design-guide/`(Apple×Toss 톤)를 따른다 — 문서 그림자료의 흑백 규약과 무관(앱은 컬러).

### 사용 라이브러리·외부 의존성
- jsPDF(CDN) — 한글 폰트 임베드로 영수증·정산서 PDF 생성. 그 외 외부 의존성 최소화(바닐라 우선).

### 품질·성능 기준 (정량)
| 항목 | 기준 |
|:---|:---|
| 실 동작 기능 수 | 100개 이상(토스트·정적 mock 제외, 실제 동작만) |
| 화면/뷰 수 | 4종 이상(다역할 포함) |
| 다단계 워크플로 | 1개 이상(예약 4단계) |
| 상태 지속성 | localStorage 새로고침 후 100% 복원 |
| 반응형 | 360~430px 가로 overflow 0, 1280px 데스크톱 레이아웃 정상 |
| 정산 계산 정확도 | 거래액 × (1 − 0.12 − 0.029) = 순지급액, 반올림 오차 없음 |
| PDF | 한글 깨짐 없이 영수증·정산서 생성 |
