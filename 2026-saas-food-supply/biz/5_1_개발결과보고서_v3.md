## v2 한계 및 v3 개선 매핑

v3는 v2의 부족함을 메우는 사이클이다. v2에서 부족했던 항목과 v3에서 해결한 방식을 1:1로 매핑한다.

| # | v2 한계 | v3 개선 | 입증 캡처 |
|:--:|:---|:---|:---|
| 1 | 예측이 **EWMA(레벨만)** — 추세·계절을 가중으로만 보정, 추세 성분이 화면에 없음 | **Holt-Winters**(레벨 α·추세 β·요일 계절 γ 분리 추정) + EWMA/HW **토글**, 예측 표에 추세·계절(×) 컬럼 노출 | v3/03-forecast-hw, v3/01-dashboard(HW 막대+선) |

## v3 신규/심화 산출물

### 신규 알고리즘

- **Holt-Winters 삼중 지수평활** — 레벨 ℓ·추세 b·요일 계절 s(m=7)를 분리 추정. 예측 μ = (ℓ + b)×계절지수. EWMA와 **방법 토글**로 직접 비교 가능. 예측 표에 μ·추세·계절(×)·σ·안전재고·ROP·제안수량 노출.
- **예산제약 발주최적화(greedy LP relaxation)** — 품목별 가치/원 = (품절위험 감소)÷(단가) 내림차순으로 예산 한도까지 채우고, 경계 품목은 부분 배정. 전량/부분/제외 결과와 배정액·충족 품목 KPI 산출.
- **폐기·마진 분석** — 폐기위험액 = max(0, 현재고 − 14일×μ)×단가×폐기율, 품절 기회손실 추정, 식재료비 절감 1%p가 영업이익에 미치는 **레버리지 곡선**.
- **통합정산 롤업** — 체인 전 매장 정산을 공급사×매장 행렬로 합산, 부가세=Σ정산×10%, 공급사/매장별 비중 차트.

### 신규 멀티테넌시·외부 통합·국제화

- **본사 통합정산 뷰**(본사 역할 전용) — 공급사×매장 롤업 표, 전자세금계산서 발행·오픈뱅킹 이체 mock.
- **외부 연동 허브 6종** — POS 매출 동기화 / 카카오·네이버 OAuth(mock 토큰, 키 0) / 오픈뱅킹 정산 이체→입금완료 webhook 콜백 / 전자세금계산서 발행 / 재고·예측 CSV 입출력 / 알림톡 발주서 발송·공급사 확인 콜백. 모든 이벤트를 연동 이벤트 로그에 집계.
- **i18n KO/EN 토글** — `data-i18n` 매핑으로 사이드바·바텀탭·뷰 제목·KPI 라벨·차트 제목을 즉시 전환, localStorage로 언어 유지.

## 2. 구현/제작 범위

- **14뷰**: 대시보드(HW) · 다점포 현황 · 본사 통합정산 · 재고 현황 · 품목·공급사 · 수요예측(HW) · 예산제약 발주최적화 · ABC·EOQ 분석 · 폐기·마진 분석 · 발주·발송 · 입고·검수 · 정산 · 공급사 평가·추천 · 외부 연동 허브(+알림 센터·공급사 포털).

- 모두 키 없이 오프라인 mock으로 동작.

## 4. 실행/구동 방법

```bash
# 1) 데모 실행 (빌드 불필요)
open projects/food-supply/v3.html      # 또는 브라우저로 file:// 열기

# 2) 캡처 재현 (선택)
cd projects/food-supply
npm i -D playwright && npx playwright install chromium
node capture-v3.mjs                     # PC→biz/captures/v3, 모바일→biz/captures/mobile/v3
rm -rf node_modules package*.json       # 잔존물 제거
```

- 키 0으로 동작: POS·OAuth·오픈뱅킹·전자세금계산서·알림톡은 mock, 폰트 fetch 실패 시 canvas 폴백.
- 역할/매장/언어 전환은 사이드바(모바일은 드로어) 셀렉트·토글로 로컬 동작.
