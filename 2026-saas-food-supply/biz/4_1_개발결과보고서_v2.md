## v2 신규/심화 산출물

### 신규 알고리즘

- **EWMA + 요일/계절 가중**: `ewma(arr, α)`가 14일 이력을 지수평활하고, `dowFactor()`가 오늘 요일의 과거 평균/전체 평균으로 보정(0.6~1.6 클램프). 예측 표에 `EWMA μ`·`요일가중(×)`·`σ`가 분리 표시(v2/03-forecast).
- **서비스레벨 안전재고**: 안전재고 = Z·σ·√L (Z=90/95/98%, σ=최근 7일 표준편차).
- **ABC 분류(파레토)**: `classifyABC()`가 연간 사용액(μ·365·단가) 내림차순으로 누적 비중을 계산해 ≤80%=A, ≤95%=B, 그 외 C. 파레토 차트(막대+누적% 선)·등급 도넛(v2/06-abc-eoq).
- **EOQ**: `eoq()` = √(2·D·S ÷ (단가·H)), D=연간수요, S=발주비용, H=단가×보관비율. 권장 주문주기·관리정책 산출.
- **공급사 평가 스코어**: `supplierScores()`가 납기준수율·단가경쟁력·품질(검수통과율)을 가중 평균(가중치 정규화)해 0~100 종합점수. 검수 부분입고/불량이 품질 점수를 낮추는 것을 실측 입증(신선수산 품질 50점, 대구축산유통 납기 50점).

### 신규 멀티테넌시·외부 통합

- **3역할**: 점주(운영·예측·발주), 본사(다점포), 공급사(수주 포털). `applyRole()`이 역할별 사이드바·바텀탭 가시성을 분기.
- **다점포**: 3개 매장 시드, 본사 다점포 대시보드에서 합산·비교·매장 전환.
- **외부 통합(실제 입출력)**: ① POS 매출→식자재 소비 환산(mock, 재고·이력 갱신) ② 재고 CSV 가져오기(헤더 스킵·미등록 품목·음수 검증) + 재고/예측 CSV 내보내기(BOM 포함) ③ 알림톡 발송 mock 로그(채널·템플릿·HTTP 200). 연동 이벤트 로그 누적.

## 2. 구현/제작 범위

- **뷰 13종**: ① 대시보드 ② 다점포 현황(본사) ③ 재고 현황 ④ 품목·공급사 ⑤ 수요예측(EWMA) ⑥ ABC·EOQ ⑦ 발주·발송 ⑧ 입고·검수 ⑨ 정산 ⑩ 공급사 평가 ⑪ 외부 연동 ⑫ 알림 센터 ⑬ 공급사 포털.
- **워크플로 3개**: 발주 사이클(소비/POS→EWMA→EOQ 제안→발주서·알림톡→입고·검수→정산), 공급망 평가 사이클(발주→출고→검수→스코어 반영), 다점포 운영 사이클(본사→매장 전환→매장별 운영).
- **실 알고리즘**: EWMA+요일가중, 서비스레벨 안전재고, ABC 파레토, EOQ, 공급사 가중 스코어. 모두 브라우저 내부에서 실제 계산.
- **실 동작(목업 아님)**: 품목 CRUD·소비/POS 기록은 localStorage 즉시 반영, 차트(Chart.js)는 실 데이터 렌더, 발주서는 jsPDF 실 생성, CSV는 실제 파싱·검증·다운로드, 검수는 부분/불량 수량을 재고·정산·평가에 반영.
- **반응형**: 데스크톱 사이드바 252px, 모바일 역할별 5탭 바텀바 + 햄버거 드로어, KPI 2열, 차트 고정높이, 표 가로 스크롤, 검수 모달은 별도 오버레이(backdrop 미디어 규칙과 분리).

## 4. 실행/구동 방법

```bash
# 1) 앱 직접 열기 (빌드 불필요)
open projects/food-supply/v2.html      # 또는 브라우저로 file:// 로 열기

# 2) 캡처 재현 (Playwright 임시 설치 후 삭제)
cd projects/food-supply
npm i -D playwright && npx playwright install chromium
node capture.mjs                       # captures/v2, captures/mobile/v2 생성
rm -rf node_modules package*.json      # 잔존물 제거
```
