# 과업지시서 v2 — 발주메이트 식자재 자동발주·재고·공급망 SaaS 고도화

> 본 문서는 외주 용역 기업에 발주메이트 **v2 시제품(웹 데모)** 제작을 위탁할 때 발주자 측이 작성하는 지시서다. v1([`3_과업지시서_v1.md`](./3_과업지시서_v1.md)) 검수 후 추가·심화된 과업(수요예측 고도화·ABC/EOQ·다중 테넌트·외부 통합·공급사 평가)을 정의한다. **v1 문서는 수정하지 않고 본 신규 파일로 분리**한다. [`2_개발계획서.md`](./2_개발계획서.md)의 후속 WBS(외주 행)를 §1 과업의 범위로 옮겨 확정했다.

## 1. 과업의 개요

### 과업의 목적

1) **직접 목적**: v1의 단일 매장·기초 발주 PoC를 **베타 출시·시리즈A 데모 가능한 공급망 SaaS**로 고도화한다. 구체적으로 ① 수요예측을 지수평활(EWMA)+요일/계절 가중으로, 안전재고를 서비스레벨(Z) 기반으로 심화, ② ABC 재고분류(파레토)·경제적주문량(EOQ) 산출, ③ 식당주/본사(체인)/공급사 **3역할·다점포** 운영, ④ 외부 통합(POS·CSV·알림톡) 2건 이상, ⑤ 공급사 평가 스코어카드를 동작 가능한 시제품으로 구현한다.
2) **사업과의 연결**: [`1_제안서.md`](./1_제안서.md) §2(Solution)의 예측 파이프라인과 §3(Scale-up)의 확장 시나리오(다매장·정산 리포트 구독, 도매상-식당 양면 네트워크)를 실증한다. v2는 제안서의 "데이터 효과·전환비용·네트워크 효과(Moat)"가 제품에서 어떻게 구현되는지를 보이는 단계다.

### 과업의 범위

외주 용역사가 수행할 업무(v1 위에 **추가/심화**):

1) **수요예측 엔진 고도화**: 14일 소비 이력 기반 EWMA(평활계수 α 선택) + 요일·계절 가중(오늘 요일 패턴 반영) + 서비스레벨(Z) 기반 안전재고·ROP·목표재고·제안수량.
2) **ABC·EOQ 분석(신규 뷰)**: 연간 사용액 파레토(누적 80/95%)로 A·B·C 등급 분류 + EOQ=√(2DS/(단가·H)) 산출 + 권장 주문주기·관리정책 매핑. 파레토 차트·등급 도넛.
3) **다중 테넌트·역할 분기**: 식당주(점주)/본사(체인 관리자)/공급사 3역할. 역할별 사이드바·바텀탭 가시성 분기, 본사는 다점포(매장 포트폴리오·매장 전환) 화면, 공급사는 수주 포털.
4) **다점포 대시보드(신규 뷰)**: 본사 관점 전 매장 재고·발주 제안액·정산 합산 + 매장별 비교 차트 + 매장 전환.
5) **공급사 평가(신규 뷰)**: 납기준수율·단가경쟁력·품질(검수 통과율) 가중 스코어카드. 가중치 조정·재계산.
6) **입고·검수 고도화**: 정상 입고 + 검수 모달(입고 비율·품질 기록) → 부분입고·불량 반영, 공급사 평가에 연동. 검수 이력 표.
7) **외부 통합 2건 이상**: ① POS 매출 동기화(mock, 매출→식자재 소비 환산) ② 재고/예측 **CSV 입출력**(파싱·검증) ③ 알림톡 발주서 발송(mock 로그). 연동 이벤트 로그.
8) **알림 센터(신규 뷰)**: 재주문 경보·과잉재고·검수 이슈·연동 결과 통합.
9) **공급사 포털(신규 뷰)**: 공급사 역할로 들어온 발주 확인 + 출고 처리(식당 입고 대기로 전송).
10) **반응형 셸 재정비**: 역할 전환 시 모바일 바텀탭 동적 재구성, 좁은 화면(390~430px) 가로 overflow 0, 사이드바→드로어/바텀탭.

### 범위 외 (Out of Scope)

- 실 백엔드/DB·실 인증(데모는 로그인 없이 동작, [CLAUDE.md](../../CLAUDE.md) §3.4). 역할/매장 전환은 로컬 시뮬레이션.
- 실 카카오 알림톡·실 POS·실 결제/정산 연동(본 과업은 **mock**까지). 실 OAuth.
- 머신러닝 기반 예측(딥러닝). 본 과업의 예측은 통계적(EWMA·서비스레벨)까지.
- 다국어.

## 2. 과업의 세부 내용

### 2.1 알고리즘 사양 (고도화)

| 알고리즘 | 공식/방법 | 산출물 |
|:---|:---|:---|
| 수요예측 EWMA | sₜ = α·xₜ + (1−α)·sₜ₋₁, α ∈ {0.2, 0.35, 0.5} | 일평균 μ |
| 요일·계절 가중 | (오늘 요일 과거 평균 ÷ 전체 평균), 0.6~1.6 클램프 | dowFactor |
| 안전재고 | Z·σ·√L (Z=서비스레벨 90/95/98%, σ=최근 7일 표준편차) | 안전재고 |
| 재주문점 ROP | μ·L + 안전재고 | ROP |
| 제안수량 | (목표재고 − 현재고) → MOQ 올림 | 제안수량 |
| ABC 분류 | 연간 사용액(μ·365·단가) 내림차순 파레토, 누적 ≤80%=A, ≤95%=B, 그 외 C | A/B/C 등급 |
| EOQ | √(2·D·S ÷ (단가·H)), D=μ·365, S=발주비용, H=단가×보관비율 | EOQ·주문주기 |
| 공급사 평가 | 납기준수율·단가경쟁력·품질(검수통과율)의 가중 평균(가중치 정규화) | 0~100 종합점수 |

### 2.2 뷰·워크플로 사양

- **뷰 13종**: ① 대시보드 ② 다점포 현황(본사) ③ 재고 현황 ④ 품목·공급사 ⑤ 수요예측(EWMA) ⑥ ABC·EOQ 분석 ⑦ 발주·발송 ⑧ 입고·검수 ⑨ 정산 ⑩ 공급사 평가 ⑪ 외부 연동 ⑫ 알림 센터 ⑬ 공급사 포털.
- **워크플로 3개 이상**:
  1. **발주 사이클**: 소비/POS 입력 → EWMA 예측 → EOQ 제안 → 발주서 생성·알림톡 발송 → 입고·검수 → 정산.
  2. **공급망 평가 사이클**: 발주 → 출고(공급사 포털) → 입고·검수(부분/불량 기록) → 공급사 스코어카드 반영.
  3. **다점포 운영 사이클**: 본사 다점포 대시보드 → 매장 전환 → 매장별 재고·예측·발주.
