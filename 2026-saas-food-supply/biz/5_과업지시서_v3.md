# 과업지시서 v3 — 발주메이트 식자재 자동발주·재고·공급망 SaaS 고도화

> 본 문서는 외주 용역 기업에 발주메이트 **v3 시제품(웹 데모)** 제작을 위탁할 때 발주자 측이 작성하는 지시서다. v2([`4_과업지시서_v2.md`](./4_과업지시서_v2.md)) 검수 후 추가·심화된 과업(Holt-Winters 예측·예산제약 발주최적화·폐기/마진 분석·본사 통합정산·공급사 대안 추천·외부 통합 확장·국문/영문 i18n)을 정의한다. **v1·v2 문서는 수정하지 않고 본 신규 파일로 분리**한다. [`2_개발계획서.md`](./2_개발계획서.md)의 후속 WBS(외주 행)를 §1 과업의 범위로 옮겨 확정했다.

## 1. 과업의 개요

### 과업의 목적

1) **직접 목적**: v2의 EWMA·다중 테넌트 SaaS를 **실서비스(시리즈A~출시) 수준의 의사결정 엔진**으로 고도화한다. 구체적으로 ① 수요예측을 **Holt-Winters(레벨+추세+요일 계절성)** 로 심화하고 EWMA와 비교 토글, ② 발주 예산이 한정될 때 품절위험을 최소화하는 **예산제약 발주최적화(선형계획 근사)**, ③ 과잉재고 폐기손실·식재료비 절감 레버리지를 금액화하는 **폐기·마진 분석**, ④ 체인 전 매장을 공급사×매장으로 롤업하는 **본사 통합정산**(부가세·전자세금계산서·오픈뱅킹 mock), ⑤ 품목별 **대안 공급사 추천**, ⑥ **외부 통합 6종**(POS·OAuth·오픈뱅킹 webhook·전자세금계산서·CSV 입출력·알림톡 양방향), ⑦ **국문/영문 i18n 토글**을 동작 가능한 시제품으로 구현한다.
2) **사업과의 연결**: [`1_제안서.md`](./1_제안서.md) §2(Solution)의 예측·최적화 파이프라인과 §3(Scale-up)의 다매장·정산 리포트 구독·양면 네트워크, 그리고 제안서 §차별성의 **"구매동인 논증"**(예측 정확도→폐기·결품 절감→이익 레버리지)을 제품에서 실증한다. v3는 차별화 기술이 *실제 비용절감·이익개선이라는 구매동인*으로 연결됨을 폐기·마진·최적화 화면에서 가시화하는 단계다.

### 과업의 범위

외주 용역사가 수행할 업무(v2 위에 **추가/심화**):

1) **Holt-Winters 수요예측(고도화)**: 레벨 α·추세 β·요일 계절 γ를 분리 추정. 예측 μ에 추세·요일 계절지수 반영. **EWMA/Holt-Winters 방법 토글**, 서비스레벨(Z) 안전재고·ROP·제안수량 산출. 결과 표에 추세·계절(×) 컬럼 노출.
2) **예산제약 발주최적화(신규 뷰)**: 발주 예산 한도 입력 → 단위당 품절위험 감소(가치/원)가 큰 품목부터 채우는 **greedy LP relaxation**. 전량/부분/제외 배정 결과, 배정액·충족 품목 KPI.
3) **폐기·마진 분석(신규 뷰)**: 커버일수 14일 초과 과잉분의 **폐기 위험 금액화**, 품절 기회손실, **식재료비 절감 1%p → 영업이익 레버리지** 곡선. 품목별 권고(발주/적정/감량).
4) **본사 통합정산(신규 뷰)**: 체인 전 매장 정산을 **공급사×매장 롤업** 표로 집계 + 통합 정산액·부가세(10%)·정산 건수 KPI + 공급사별/매장별 비중 차트 + **전자세금계산서 발행·오픈뱅킹 이체(mock)**.
5) **공급사 대안 추천(심화)**: 공급사 스코어카드(납기·단가·품질 가중) 위에 **품목별 현재 공급사 ↔ 추천 공급사** 비교(점수 델타·사유) 표.
6) **외부 통합 6종(확장)**: ① POS 매출 동기화(mock) ② **카카오/네이버 OAuth 로그인(mock 토큰)** ③ **오픈뱅킹 정산 이체 → 입금완료 webhook 콜백(mock)** ④ **전자세금계산서 발행 API(mock)** ⑤ 재고/예측 CSV 입출력(파싱·검증) ⑥ 알림톡 발주서 발송 + 공급사 확인 콜백. 연동 이벤트 로그 통합.
7) **국문/영문 i18n 토글**: 사이드바·바텀탭·뷰 제목·KPI 라벨을 KO/EN 즉시 전환(localStorage 유지).
8) **반응형·역할 셸 유지·강화**: 3역할(점주/본사/공급사) 분기 + 신규 뷰(최적화·폐기·통합정산)의 모바일 바텀탭/드로어 편입, 390~430px 가로 overflow 0.

### 범위 외 (Out of Scope)

- 실 백엔드/DB·실 인증(데모는 로그인 없이 동작, [CLAUDE.md](../../CLAUDE.md) §3.4). OAuth·webhook·세금계산서·오픈뱅킹은 **mock 토큰·mock 콜백**까지. v1.html·v2.html 보존.
- 실 카카오 알림톡·실 POS·실 결제/정산·실 국세청 연동(본 과업은 mock 로그까지).
- 딥러닝 기반 예측. 본 과업의 예측은 통계적(Holt-Winters·EWMA·서비스레벨)까지. LP는 greedy 근사(정수계획 solver 미사용).
- 한·영 외 다국어.

## 2. 과업의 세부 내용

### 2.1 알고리즘 사양 (고도화)

| 알고리즘 | 공식/방법 | 산출물 |
|:---|:---|:---|
| Holt-Winters | 레벨 ℓₜ=α(xₜ/sₜ₋ₘ)+(1−α)(ℓₜ₋₁+bₜ₋₁), 추세 bₜ=β(ℓₜ−ℓₜ₋₁)+(1−β)bₜ₋₁, 계절 sₜ=γ(xₜ/ℓₜ)+(1−γ)sₜ₋ₘ (m=7, 요일 주기) | 예측 μ·추세·계절지수 |
| EWMA(비교) | sₜ = α·xₜ + (1−α)·sₜ₋₁ | 일평균 μ |
| 안전재고 | Z·σ·√L (Z=90/95/98%) | 안전재고 |
| 재주문점 ROP | μ·L + 안전재고 | ROP |
| 제안수량 | (목표재고 − 현재고) → MOQ 올림 | 제안수량 |
| 예산제약 최적화 | 품목별 가치/원 = (품절위험 감소)÷(단가) 내림차순 greedy 배정, Σ배정액 ≤ 예산 | 배정수량·배정액(전량/부분/제외) |
| 폐기 위험 | max(0, 현재고 − 14일×μ) × 단가 × 폐기율 | 폐기위험액 |
| 이익 레버리지 | 절감액(=식재료비×절감%p) ÷ 영업이익률 기준 배수 | 레버리지 곡선 |
| ABC 분류 | 연간 사용액(μ·365·단가) 파레토, 누적 ≤80%=A, ≤95%=B, 그 외 C | A/B/C 등급 |
| EOQ | √(2·D·S ÷ (단가·H)), D=μ·365 | EOQ·주문주기 |
| 공급사 평가 | 납기준수율·단가경쟁력·품질의 가중 평균(정규화) | 0~100 종합점수 |
| 통합정산 롤업 | 공급사×매장 정산 합산, 부가세=Σ정산×10% | 롤업 표·VAT |

### 2.2 뷰·워크플로 사양

- **뷰 14종**: ① 대시보드(HW) ② 다점포 현황(본사) ③ 본사 통합정산(본사) ④ 재고 현황 ⑤ 품목·공급사 ⑥ 수요예측(HW) ⑦ 예산제약 발주최적화 ⑧ ABC·EOQ 분석 ⑨ 폐기·마진 분석 ⑩ 발주·발송 ⑪ 입고·검수 ⑫ 정산 ⑬ 공급사 평가·추천 ⑭ 외부 연동 허브 / 알림 센터 / 공급사 포털.
- **워크플로 4개 이상**:
  1. **발주 사이클**: 소비/POS 입력 → Holt-Winters 예측 → 예산제약 최적화 → 발주서 생성·알림톡 발송 → 입고·검수 → 정산.
  2. **공급망 평가 사이클**: 발주 → 출고(공급사 포털) → 입고·검수(부분/불량) → 스코어카드·대안 추천 반영.
  3. **본사 통합정산 사이클**: 다점포 현황 → 본사 통합정산 롤업 → 전자세금계산서·오픈뱅킹 이체(mock).
  4. **이익 최적화 사이클**: 폐기·마진 분석(폐기·결품 손실 금액화) → 예산제약 최적화로 발주 재배분 → 이익 레버리지 확인.
