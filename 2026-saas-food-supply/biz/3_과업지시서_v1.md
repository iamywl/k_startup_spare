# 과업지시서 v1 — 발주메이트 식자재 자동발주·재고관리 SaaS 시제품

> 본 문서는 외주 용역 기업에 발주메이트 v1 시제품(웹 데모) 제작을 위탁할 때 발주자 측이 작성하는 지시서다. [`2_개발계획서.md`](./2_개발계획서.md) §WBS의 외주 행(3~9)을 §1 과업의 범위로 옮겨 확정했다.

---

## 1. 과업의 개요

### 과업의 목적

1) **직접 목적**: 소상공인 식당↔식자재 공급사 B2B 자동발주·재고관리 SaaS의 동작 가능한 시제품(웹 데모) 1식을 제작한다. 재고 추적 → 소비예측 → 재주문 제안 → 발주서 생성·발송(mock) → 입고·정산의 5단계 워크플로가 실제로 동작해야 한다.
2) **사업과의 연결**: 본 시제품은 [`1_제안서.md`](./1_제안서.md)에서 제시한 "예측 기반 자동발주" 가설을 검증하는 MVP다. 제안서 §2(Solution)의 알고리즘(이동평균·안전재고·ROP)과 §3(Scale-up)의 활성화 지표("첫 발주서 생성")가 시제품에서 실증 가능해야 한다.

### 과업의 범위

외주 용역사가 수행할 업무:

1) **소비예측 엔진**: 이동평균(μ)·표본표준편차(σ)·안전재고(Z·σ·√L)·재주문점(ROP=μL+안전재고)·목표재고·제안수량(MOQ 올림) 실제 계산 구현.
2) **재고 대시보드(뷰①)**: KPI 4종 + 소진추이 라인차트 + 커버일수 막대차트 + 재고 현황 표.
3) **품목·공급사 관리(뷰②)**: 품목 CRUD(현재고·단가·리드타임·MOQ·Z), 공급사 추가, 소비 기록(재고 차감), localStorage 지속.
4) **소비예측·재주문 제안(뷰③)**: 파라미터(이동평균 윈도우·목표커버일수) 입력 → 품목별 계산 결과 표 + 부족 품목 발주 제안 카드.
5) **발주서·발송(뷰④)**: 공급사별 발주서 초안 → jsPDF 발주서 생성·다운로드(한글 깨짐 금지) → mock 알림톡 발송 로그 → 발송 이력 추적.
6) **입고·정산(뷰⑤)**: 입고확정 시 재고 증가 + 정산 내역 누적, KPI·정산 표.
7) **반응형 셸**: 데스크톱 사이드바 + 모바일 바텀탭/햄버거 드로어. 좁은 화면 가로 overflow 0.

### 범위 외 (Out of Scope)

- 실 백엔드/DB 구축, 회원가입·인증(데모는 로그인 없이 동작, §3.4).
- 실 카카오 알림톡 발송(본 과업은 **mock 로그**까지). 실 결제·정산 연동.
- POS·외부 ERP 실 연동(로드맵). 다국어.

## 2. 과업의 세부 내용

### 2.1 기능 사양

| 기능 | 사양 |
|:---|:---|
| 소비예측 | 이동평균 윈도우 선택(5/7/14일), μ·σ 계산, 안전재고=Z·σ·√L, ROP=μL+안전재고 |
| 재주문 판정 | 현재고 ≤ ROP → "발주 필요". 제안수량=(목표재고−현재고) MOQ 올림 |
| 서비스수준 Z | 품목별 선택(1.28/1.65/2.05 = 90/95/98%) |
| 발주서 | 공급사별 그룹핑, 발주번호 자동(PO-YYYYMMDD-seq), PDF 다운로드 |
| 발송 | mock 알림톡(대체 SMS) 발송 로그, 발송/입고완료 상태 |
| 정산 | 입고확정→재고 증가+정산 누적, 누적 정산액 표시 |
| 지속성 | localStorage, 새로고침 후 상태 유지 |

### 2.2 UI/UX 제작 사양

- 화면 흐름: 대시보드 → 품목/공급사 → 소비예측·재주문 → 발주서·발송 → 입고·정산 (5뷰).
- 디자인 톤: [디자인_애플_토스.md](../../design-guide/디자인_애플_토스.md) — 단일 브랜드 hue(블루/인디고), 위계는 여백·타이포, WCAG AA(4.5:1)+.
- 모바일: [디자인_모바일.md](../../design-guide/디자인_모바일.md) — 사이드바 금지, 바텀탭 5개 + 햄버거 드로어, KPI 2열 스택, 차트 고정높이.
- 데스크톱: [디자인_PC.md](../../design-guide/디자인_PC.md) — 사이드바 248px + 본문 min-w-0, 2열 그리드.

### 2.3 사용 부품·라이브러리·외부 의존성

| 항목 | 버전/출처 |
|:---|:---|
| Chart.js | 4.4.x (CDN) |
| jsPDF | 2.5.1 (CDN) + 한글 TTF(임베드) / canvas 폴백 |
| 저장소 | 브라우저 localStorage |
| 빌드 | 없음(단일 HTML, file:// 동작) |

> API 키 하드코딩 금지. 외부 네트워크 없이도 핵심 기능 동작(폰트 fetch 실패 시 canvas 폴백).

### 2.4 품질·성능 기준 (정량)

| 항목 | 기준 |
|:---|:---|
| 뷰 수 | 5종 이상, 각 뷰 인터랙션 가능 |
| 워크플로 | 5단계(재고→예측→제안→발송→정산) 실제 동작 |
| 반응형 | 390~430px 가로 overflow 0, 1280px 사이드바 정상 |
| 예측 정확성 | 시드 데이터로 μ·σ·ROP·제안수량이 수식과 일치 |
| PDF | 한글 깨짐 0(□ 금지), 발주서 항목 누락 0 |
| 상태 지속 | 새로고침 후 입력·정산 데이터 유지 |
| 캡처 | PC 6장+ / 모바일 6장+ 실 구동 |
