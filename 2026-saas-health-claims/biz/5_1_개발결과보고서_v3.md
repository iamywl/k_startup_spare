## v3 신규/심화 산출물

8대 자체 산출물을 캡처로 입증한다(§5 캡처 단위 설명 참조). 신규 캡처 22장(>12장 기준).

1. **반려 예측 룰엔진(실 계산)** — `predictDenial`가 7개 위험룰(진찰료 누락·중복·상병 미기재·비급여 동의·고액·DTx 근거·실손 한도)을 가중치로 평가, 위험점수를 로지스틱 함수 `P=1/(1+e^-(k·risk-b))`로 정규화해 반려확률(%)·위험등급·사유별 기여도(%)를 산출한다. 워크플로 step2에서 제출 전 사전검증, 심사 콘솔에서 청구별 예측, 청구 목록에 예측 컬럼 표기(캡처 04·08·09).
2. **보험사 다중연동(EDI+API)** — `ediSubmit`가 채널을 분기: 심평원은 EDI 텍스트 레이아웃(HEAD/LINE/TAIL), 보험사(삼성화재·현대해상·DB손해보험)는 REST API JSON 페이로드. 채널별 엔드포인트·접수번호·통신 로그 보존(캡처 07·17·20).
3. **다기관 통합 정산 대시보드** — `GroupSettlement`가 3개 테넌트의 청구/지급/미수/반려를 롤업, 기관별 지급률·반려율·평균 소요일·청구액 비교 막대 + 통합 합계(캡처 15).
4. **DTx 처방→급여 연동** — 코드북에 DTx 급여 코드(DTX10 불면증·DTX20 만성통증) 추가, 처방 자동매핑·수동 추가로 급여 산정·청구서 DTx 라벨 표기(캡처 03·04·06).
5. **환자 앱(청구 추적·동의·전자서명)** — 모바일 환자 앱에서 청구 진행바(접수→심사→지급, 반려 시 적색)로 추적, 캔버스 전자서명→PNG 저장, 실손 보험사 청구 제출(캡처 19·20·21·22).
6. **수가/코드 고시 자동업데이트** — `CODE_BOOK_VERSIONS` 2024·2026 고시 버전, effective date·단가 diff(증감 ▲▼) 비교표, 활성 버전 전환 시 신규 청구 단가 자동 적용(캡처 13·14).
7. **정산·미수금 + 펌뱅킹** — `settleClaim` 정산액·미수금 산출, 청구일 경과 미수금 aging 구간 분류, `firmBankingFile`로 전자금융 펌뱅킹 이체파일(헤더/명세/트레일러) 생성·다운로드(캡처 11·12).
8. **감사로그·KPI** — 모든 제출·심사·정산·내보내기·고시적용·동의 행위를 주체·역할·기관·시각과 함께 감사 트레일에 기록, 통합 지급률·반려율·평균 소요일·감사 이벤트 KPI 시각화(캡처 16).

## 2. 구현/제작 범위

- **반려 예측(실 알고리즘 1)**: 7 위험룰 가중 스코어링 → 로지스틱 정규화(`k=1.05, b=2.4`) → 반려확률·등급(낮음/주의/높음)·사유 기여도. 심사 시 확률 50%+ 또는 진찰료 누락이면 자동 반려.
- **정산 산출(실 알고리즘 2)**: `settleClaim` 상태별 정산액·미수금 산출 + 다기관 롤업 + aging 구간 분류 + KPI(지급률=Σ지급/Σ청구, 반려율=반려/건수, 평균 소요일=작성→지급 일수 평균).
- **보험사 다중연동**: 채널 2종(심평원 EDI 텍스트 / 보험사 REST JSON), 보험사 4곳 라우팅(국민건강보험·삼성화재·현대해상·DB손해보험), REQ/RES 통신 로그.
- **다기관(테넌트 3곳)**: 서울365의원(11010001)·부산항운정형외과(21030045)·대구메디케어내과(31050088). 각 환자·청구·동의·시퀀스 격리, 통합 정산은 본부관리자 롤업.
- **RBAC 4역할**: 의료진·청구담당·본부관리자·환자. 본부관리자는 다기관 통합 정산·감사·KPI 전용, 권한 외 화면은 `AccessDenied` 차단.
- **DTx 급여 연동**: DTx 코드 자동매핑·수동 추가, 청구서·코드북 DTx 라벨, 2026 고시 신규 급여 단가.
- **수가 고시 버전관리**: 2024/2026 고시 버전, 단가 diff·effective date, 활성 버전 단가 자동 적용.
- **정산·미수금·펌뱅킹**: 정산 명세·미수금 aging·펌뱅킹 이체파일(텍스트 레이아웃) 다운로드.
- **환자 앱**: 청구 추적 진행바·동의·캔버스 전자서명·실손 청구.
- **감사로그·KPI**: 행위·주체·시각 감사 트레일 + 통합 KPI.
- **외부 통합 3건+**: ① 보험사 REST API mock(JSON 페이로드·로그) ② 심평원 EDI mock(텍스트 레이아웃) ③ 펌뱅킹 이체파일(텍스트 외부 출력) ④ CSV 입출력. 모두 키 부재 프론트 내장 동작.
- **상태 지속성**: 전 테넌트·버전·감사로그 `claimbridge_v3` 키 localStorage 직렬화.
- **로그인 불요**: CLAUDE.md §3.4 — 역할/테넌트 전환 스위치로 인증 우회(demo).

뷰/화면 종류(10종+): ① 대시보드 ② 다기관 통합 정산 ③ 신규 청구·DTx 워크플로(4스텝) ④ 청구 목록(+상세·연동 로그) ⑤ 심사·반려예측 콘솔 ⑥ 정산·미수금·펌뱅킹 ⑦ 환자 관리(+동의 모달) ⑧ 수가 고시 관리 ⑨ 감사로그·KPI ⑩ 기관 관리 ⑪ 환자 앱(모바일, 청구 추적·동의·실손). 워크플로(4개+): 청구 제출(자동매핑·반려예측·DTx) / 심사·정산 / 펌뱅킹 이체 / 환자 동의·실손 청구.

## 데이터 정직성 선언

본 보고서의 모든 화면·수치는 실제 구동된 클레임브릿지 v3 SPA(`projects/claimbridge-spa/v3.html`)를 Playwright로 캡처한 결과다. 합성·목업 이미지는 사용하지 않았다. 청구코드 단가·EDI 코드·요양기관기호·고시 단가·지급률·반려확률은 데모용 예시값이며 실제 건강보험 수가 고시값·실 EDI/보험사 API 규격·실 펌뱅킹 포맷이 아니다(시리즈A 데모 시제품 표기). 심평원/보험사 API·펌뱅킹은 mock 시뮬레이션이다. 외부 통계·인용은 본 결과보고서에 포함하지 않는다.
