## v2 신규/심화 산출물

8대 자체 산출물을 캡처로 입증한다(§5 캡처 단위 설명 참조). 신규 캡처 20장(>10장 기준).

1. **반응형** — 데스크톱 관리자 콘솔(1440px)과 모바일 환자 포털(414px 폰 프레임)을 같은 코드베이스에서 분기 렌더(캡처 16·17). Tailwind `lg:` 그리드로 카드/패널이 뷰포트별 1·2·4열로 재배치.
2. **다중 기관(테넌트)** — `state.tenants[tenantId]` 구조로 환자·청구·시퀀스를 테넌트별 분리. 사이드바 테넌트 셀렉터 전환 시 전 화면 데이터 교체, 타 테넌트 데이터 비노출(캡처 02·03).
3. **자동매핑 알고리즘·검증** — `autoMapPrescription`이 처방 텍스트를 토큰화→키워드 가중 스코어링→EDI 코드 산출, 매핑 점수·근거 표기. `validateClaim` 5룰 엔진이 통과/경고/오류 판정(캡처 05).
4. **EDI API mock** — `ediSubmit`(제출→접수 RECEIVED)·`ediAdjudicate`(심사→지급/반려) mock. 청구 상세에 REQ/RES JSON 통신 로그(캡처 07·09·10·11).
5. **동의·전자서명** — 동의서 + HTML Canvas 서명을 마우스/터치로 그려 PNG dataURL로 저장, 동의 이력·서명 썸네일 표기(캡처 14·15).
6. **실손 자동화 mock** — 실손 보험자 환자가 포털에서 청구 제출→보험사 mock 접수, 자기부담 반영 지급률(캡처 18·19).
7. **역할 권한(RBAC)** — `ROLES.can` 화이트리스트로 역할별 메뉴/기능 노출·차단. 의료진은 심사/기관 메뉴 미노출, 환자는 모바일 포털 전용(캡처 16·17).
8. **상태·지급률·반려사유 대시보드** — 상태별 분포 바·지급률(%)·반려사유 분석 패널·정산 처리 현황(캡처 01·10·20).

## 2. 구현/제작 범위

- **멀티테넌트 격리**: 2개 의료기관(서울365의원 11010001 / 부산항운정형외과 21030045). 각 테넌트 환자·청구·동의·시퀀스 분리. `updateTenant`는 활성 테넌트만 수정.
- **RBAC 3역할**: 의료진(처방·환자)·청구담당(제출·심사·정산·기관)·환자(모바일 포털). 접근 불가 화면은 `AccessDenied` 차단.
- **자동매핑 + 검증**: 11개 코드북(급여/비급여, EDI 코드·가중치). 토큰 스코어링 정렬·매핑 근거·5룰 검증.
- **EDI 청구 흐름(mock)**: 작성→제출→접수→심사→지급완료/반려 상태머신. 심평원·보험사 채널 분기. 통신 로그 JSON 보존.
- **동의·전자서명**: 캔버스 서명(마우스·터치)→PNG, 동의 텍스트·타임스탬프·서명 이미지 영속.
- **실손 자동화**: 환자 포털에서 미청구 실손 건을 보험사로 제출, 자기부담 90% 지급률 산정.
- **CSV 입출력**: 청구 내보내기(BOM 포함 UTF-8)·가져오기(파싱 후 신규 청구 추가).
- **대시보드**: 상태별 분포·금액 기준 지급률·반려사유 집계·최근 청구·정산 현황.
- **상태 지속성**: 전 테넌트 상태 `claimbridge_v2` 키 localStorage 직렬화.
- **로그인 불요**: CLAUDE.md §3.4 — 역할/테넌트 전환 스위치로 인증 우회(demo).

뷰/화면 종류(8종+): ① 대시보드 ② 신규 청구 워크플로(4스텝) ③ 청구 목록(+상세 모달·EDI 로그) ④ 심사·정산 콘솔 ⑤ 환자 관리(+동의 모달) ⑥ 기관 관리 ⑦ 전자서명 캔버스 ⑧ 모바일 환자 포털(+실손 청구). 워크플로(3개+): 청구 제출 워크플로 / 심사·정산 워크플로 / 환자 동의·실손 청구 워크플로.

## 데이터 정직성 선언

본 보고서의 모든 화면·수치는 실제 구동된 클레임브릿지 v2 SPA(`projects/claimbridge-spa/v2.html`)를 Playwright로 캡처한 결과다. 합성·목업 이미지는 사용하지 않았다. 청구코드 단가·EDI 코드·요양기관기호·지급률은 데모용 예시값이며 실제 건강보험 수가 고시값·실 EDI 규격이 아니다(베타 시제품 표기). 심평원/보험사 API는 mock 시뮬레이션이다. 외부 통계·인용은 본 결과보고서에 포함하지 않는다.
