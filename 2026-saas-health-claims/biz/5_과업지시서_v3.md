# 과업지시서 v3 — 클레임브릿지 청구 자동화 플랫폼 시리즈A 데모(스케일업)

> v2([`4_과업지시서_v2.md`](./4_과업지시서_v2.md)) 검수 후 추가/변경된 과업. v1·v2를 수정하지 않고 신규 파일로 분리한다. 본 문서는 [`2_개발계획서.md`](./2_개발계획서.md) §작업분해의 v3 외주 구분 행을 §1 범위로 옮기고, v3 자체 심화 산출물 8가지를 §5에 명시한다.

## 1. 과업의 개요

### 1.1 과업의 목적

1) **외주에 맡기는 직접 목적**: v3 시리즈A 데모에서 다기관(테넌트) 통합 정산·반려 예측 룰엔진·보험사 다중연동·펌뱅킹 이체파일·수가 고시 자동반영을 지탱하는 **반려예측 룰셋 정합성 검증 자문 + 펌뱅킹/EDI 파일 포맷(전자금융·심평원 EDI 레이아웃) 규격 자문 + 감사로그 무결성 설계 자문**을 외주 위탁한다.
2) **사업·제안 연결**: [`1_제안서.md`](./1_제안서.md) §2.5 청구 정확성 검증, §5.3 해자 3층, §8 로드맵 Series A(v3)에 해당. v2의 멀티테넌트·EDI mock 위에 **반려 사전예측·다기관 통합 정산·미수금/펌뱅킹·수가 고시 반영·감사로그/KPI**를 얹어 시리즈A 데모 가능 수준으로 스케일업한다.

### 1.2 과업의 범위

**외주사가 수행할 업무 (In Scope)**

1) 반려 예측 룰엔진(denial prediction) 룰셋 정합성·가중치 검토 자문(반려 사유 코드 체계·심평원 심사기준 매핑 정합성).
2) 펌뱅킹 이체파일(전자금융공동망 CMS/펌뱅킹 텍스트 레이아웃) 포맷 규격 자문 + 보험사 EDI/API 다중연동 메시지 스키마(EDI 4010 유사 레이아웃·REST API 페이로드) 설계 자문.
3) 다기관 통합 정산 집계 산식(기관별 지급/미수/반려 롤업) 검증 자문.
4) 감사로그(audit trail) 무결성 설계 자문(행위·주체·시각·해시체인 개념) + 청구 KPI(지급률·반려율·소요일) 산식 검증.
5) 수가/코드 고시 반영 버전관리(고시 effective date·코드 버전 diff) 데이터 모델 설계 자문.
6) 설계서·ER/시퀀스 다이어그램·인계 문서.

**범위 외 (Out of Scope)**

- 8대 v3 자체 산출물의 프론트 구현 (자체 개발 — §5 참조).
- 반려 예측 스코어링 로직·정산 산출 알고리즘 구현 (자체 개발).
- 실 보험사/심평원/은행 운영망 연계 (정식 연동은 별도 사업, mock은 자체).
- ISMS-P 인증 취득 실행·실 PG/펌뱅킹 계약.

### 1.3 자체 개발 v3 산출물 개요

§5 성과품 목록의 8대 자체 산출물이 v3 핵심이며, 외주는 그 기반 룰셋·파일 포맷·정산 산식·감사 무결성을 보강한다.

## 2. 과업의 세부 내용

### 2.1 반려 예측 룰엔진 사양

| 항목 | 사양 |
|:---|:---|
| 입력 | 청구 항목(코드·구분·금액)·상병·환자 동의·기관·채널 |
| 산출 | 반려확률(%)·위험등급(낮음/주의/높음)·사유별 기여도·교정 권고 |
| 알고리즘 | 가중 룰 스코어링(룰별 위험가중치 합산→로지스틱 정규화) — 실 계산 |
| 사전검증 | 제출 전 "사전검증" 실행 시 반려 위험 항목 하이라이트 |

### 2.2 보험사 다중연동(EDI+API) mock 사양

- 채널 2종 이상: 심평원 EDI(텍스트 레이아웃) + 보험사 REST API(JSON 페이로드).
- 보험사 3곳 이상 라우팅(삼성화재·현대해상·국민건강보험) — 채널별 응답·접수번호 분기.
- 연동 로그(REQ/RES) 보존, 채널별 처리 상태 추적.

### 2.3 다기관 통합 정산·미수금·펌뱅킹 사양

- 다기관(테넌트 3곳 이상) 통합 정산 대시보드 — 기관별 지급/미수/반려 롤업.
- 미수금 관리(aging: 0~30/31~60/60일+) + 정산 배치 생성.
- 펌뱅킹 이체파일(CSV/텍스트 레이아웃) 생성·다운로드(수취인·계좌·금액·적요).

### 2.4 DTx 처방→급여 연동·수가 고시 반영·환자앱·감사/KPI 사양

- DTx(디지털치료제) 처방 코드 → 급여 산정 워크플로 확장(처방→코드→급여→청구).
- 수가/코드 고시 반영 시뮬레이션(고시 effective date 적용 시 단가 diff·버전 전환).
- 환자 앱(모바일): 청구 추적·동의·전자서명(캔버스→PNG).
- 감사로그(행위·주체·시각 해시체인) + KPI(지급률·반려율·평균 소요일) 시각화.

### 2.5 품질·성능 기준 (정량)

| 기준 | 목표 |
|:---|:---|
| 반려 예측 | 청구 1건당 반려확률·등급·기여도 산출, 사전검증 결과 표시 |
| 다기관 정산 | 테넌트 3곳 통합 롤업, 기관 격리 유지(타 테넌트 조회 0건) |

| 뷰/워크플로 | 뷰 10종+, 워크플로 4개+ |

## 6. 변경 이력 (v2 → v3)

| 구분 | v2 | v3 변경/추가 |
|:---|:---|:---|
| 검증 | 5룰 검증(pass/warn/error) | **반려 예측 룰엔진** — 반려확률(%)·위험등급·사유 기여도·제출 전 사전검증 |
| 외부 연계 | EDI API mock(단일 채널·CSV) | **보험사 다중연동**(보험사 3곳·EDI 텍스트+REST API JSON) + **펌뱅킹 이체파일** |
| 정산 | 단일 기관 정산 현황 | **다기관 통합 정산 대시보드**(기관별 지급/미수/반려 롤업) + **미수금 aging** |
| 처방 | 자유텍스트 자동매핑 | + **DTx 처방→급여 연동** 워크플로 확장 |
| 환자 | 모바일 포털(조회·실손·동의) | **환자 앱** 청구 추적·동의·전자서명(유지·심화) |
| 코드북 | 고정 단가 | **수가/코드 고시 자동업데이트**(고시 버전 diff·effective date) |
| 운영 | 토스트·상태 이력 | **감사로그**(행위·주체·시각) + **청구 KPI**(지급률·반려율·소요일) |
