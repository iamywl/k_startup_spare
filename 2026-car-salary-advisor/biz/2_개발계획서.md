# 개발계획서 — 연봉 기반 차량 옵션 추천기 (가칭 카픽 / CarPick)

> 이 문서는 **설계 단계 산출물**이다. 앱 코드·실데이터 수집은 아직 진행하지 않으며(사용자 지시),
> 본 설계가 확정되면 데이터 수집 → `projects/car-advisor/` 구현 순으로 진행한다.

| 항목 | 값 |
|---|---|
| 프로젝트 | `2026-car-salary-advisor` |
| 한 줄 목표 | **내 연봉을 넣으면, 감당 가능한 차들 중에서 원하는 옵션(2열 엉따·통풍 등)으로 거르고 예상 유지비까지 보여주는 추천 웹앱** |
| 산출물 | standalone 데모 웹앱 1개 (`projects/car-advisor/`) — 추후 구현 |
| 플랫폼 | 단일 HTML + Tailwind(CDN) + Pretendard + vanilla JS, 백엔드 없음(mock 데이터) |
| 디자인 기준 | [`../../design-guide/디자인_애플_토스.md`](../../design-guide/디자인_애플_토스.md) (Apple × Toss) 단일 준수 |

## 1. 핵심 기능 (사용자 요청 4종 + 부가)

| # | 기능 | 설명 | 사용자 요청 |
|---|---|---|:---:|
| F1 | **연봉 기반 추천** | 연봉을 입력하면 감당 가능한 예산 구간을 산정하고, 그 안의 차량을 추천·정렬 | ✅ |
| F2 | **옵션별 필터** | 차량 옵션(앞 열선/통풍, **2열 엉따(열선)·2열 통풍**, 선루프, HUD, 어라운드뷰 등)을 개별 토글로 AND 필터 | ✅ |
| F3 | **예상 유지비** | 연료비 + 자동차세 + 보험(추정) + 소모품/정비를 합산해 월/연 단위로 표시, 항목별 분해 | ✅ |
| F4 | **주행거리 입력** | 사용자별 **연간 주행거리(km)** 를 입력 → 유지비(연료·소모품)에 반영, 슬라이더로 즉시 재계산 | ✅ |
| F5 | 부담률 표시 | 차량가/월 유지비가 연봉(실수령) 대비 몇 %인지 신호등으로 표시 | 부가 |
| F6 | 비교함 | 관심 차량 2~3대 나란히 비교 (가격·연비·옵션·유지비) | 부가(v2 후보) |

## 2. 옵션 분류 체계 (필터 대상 taxonomy)

> 📐 **상세 도메인·필터 설계는 [`2_1_옵션_도메인_필터설계.md`](./2_1_옵션_도메인_필터설계.md) 참조.**
> 거기서 (A) 엔지니어링 정식 분류(정규화 저장) ↔ (B) 소비자 체감 라벨(UI) 2계층 분리,
> 페르소나 태그(#초보운전·#장거리출퇴근·#캠핑차박·#수족냉증·#패밀리카), 검색어 동의어 매핑,
> 옵션 가치(가성비·잔존가치) 점수, 정규화 스키마(ERD)까지 정의한다.
>
> 아래는 데모 v1 구현을 위한 핵심 옵션 요약(boolean 모델). 구어("엉따"=좌석 열선)를 라벨에 병기.

| 그룹 | 옵션 키 | 표시 라벨 | 비고 |
|---|---|---|---|
| 시트 | `frontHeatedSeat` | 앞좌석 열선 (앞 엉따) | |
| 시트 | `frontVentSeat` | 앞좌석 통풍 | |
| 시트 | `rearHeatedSeat` | **2열 열선 (2열 엉따)** | 사용자 명시 |
| 시트 | `rearVentSeat` | **2열 통풍** | 사용자 명시 |
| 시트 | `heatedWheel` | 스티어링 휠 열선 | |
| 시트 | `memorySeat` | 운전석 메모리 시트 | |
| 시트 | `powerPassengerSeat` | 동승석 전동(워크인) | |
| 시트 | `massageSeat` | 마사지 시트 | 상위 트림 |
| 편의 | `sunroof` | 선루프 / 파노라마 선루프 | `sunroofType` 으로 일반/파노라마 구분 |
| 편의 | `hud` | 헤드업 디스플레이(HUD) | |
| 편의 | `wirelessCharger` | 무선 충전 | |

| 편의 | `builtinCam` | 빌트인 캠(블랙박스 내장) | |
| 편의 | `ambientLight` | 앰비언트 라이트 | |
| 안전 | `smartCruise` | 스마트(어댑티브) 크루즈 | |
| 안전 | `hda` | 고속도로 주행보조(HDA/차로유지) | |
| 안전 | `bca` | 후측방 충돌방지 보조 | |
| 안전 | `svm` | 서라운드뷰(어라운드뷰) 모니터 | |
| 안전 | `rcca` | 후방 교차충돌 방지 | |

> 필터 UI는 그룹별 칩(chip) 토글. 선택한 옵션을 **모두 만족(AND)** 하는 차량만 노출.
> 옵션 데이터는 트림 단위로 수집(같은 모델도 트림별 적용 상이)하되, 데모 v1은 **대표 트림 1개/모델**로 단순화.

## 3. 차량 데이터 스키마 (수집 대상 필드)

```jsonc
{
  "id": "string",                 // brand-model-trim 슬러그
  "brand": "현대",                 // 제조사
  "model": "그랜저",
  "trim": "캘리그래피",            // 대표 트림
  "segment": "준대형",            // 경차/소형/준중형/중형/준대형/대형/SUV소형/SUV중형/SUV대형/MPV
  "bodyType": "세단",             // 세단/SUV/해치백/MPV/픽업
  "fuel": "가솔린",               // 가솔린/디젤/하이브리드/EV/LPG
  "price": 4380,                  // 만원 (대표 트림 시작가)
  "displacementCc": 2497,         // 자동차세 계산용 (EV=0, 정액 처리)
  "efficiency": 10.4,             // 복합연비 km/L  (EV는 전비 km/kWh)
  "drivetrain": "FWD",            // FWD/RWD/AWD
  "seats": 5,
  "options": {                    // §2 taxonomy 키 → boolean (대표 트림 기준)
    "frontHeatedSeat": true, "frontVentSeat": true,
    "rearHeatedSeat": true, "rearVentSeat": true,
    "heatedWheel": true, "memorySeat": true, "massageSeat": false,
    "sunroof": true, "sunroofType": "파노라마", "hud": true,
    "wirelessCharger": true, "digitalKey": true, "powerTailgate": true,
    "builtinCam": true, "ambientLight": true,
    "smartCruise": true, "hda": true, "bca": true, "svm": true, "rcca": true
  },
  "source": "제조사 카탈로그"      // 데이터 출처(수집 단계에서 채움)
}
```

- 데모 목표 규모: **약 24~40개 차종**(경차~대형·SUV·하이브리드·EV·수입 일부) 커버.
- 모든 수치는 수집 단계 전까지 **`[추정]`** 으로 표기하고, 실값 확정 시 출처와 함께 교체.

## 4. 추천 로직 — 연봉 → 예산 산정

입력: `연봉(세전, 만원)`, (선택) `보유 예산`, `연료 선호`, `필수 옵션 필터`, `연간 주행거리`.

**예산 성향(사용자 선택, 기본=표준)**

| 성향 | 차량가 상한 | 근거(통념) |
|---|---|---|
| 보수적 | 연봉 × **0.4** | "차값은 연봉 절반 이하" 중에서도 보수적 |
| 표준 | 연봉 × **0.5** | 가장 흔한 권장선 |
| 공격적 | 연봉 × **0.7** | 차에 비중을 크게 두는 경우 |

- 보조 지표: **월 차량 총지출(할부 가정 + 유지비) ≤ 월 실수령액의 20%** 권장선 충족 여부 표시.
- 결과 정렬 옵션: `가성비(옵션수/가격)` · `유지비 낮은 순` · `가격 낮은 순` · `연비 좋은 순`.
- 필수 옵션 필터를 만족하지 못하면 후보에서 제외(없으면 "조건을 만족하는 차가 없어요 → 옵션을 줄여보세요" 안내).

## 5. 예상 유지비 계산식 (연간, 주행거리 반영)

> 모든 단가·계수는 **변수로 분리**해 데이터 수집 단계에서 실값으로 교체. 현재는 `[추정]` 기본값.

```
연_유지비 = 연료비 + 자동차세 + 보험료(추정) + 소모품·정비
```

| 항목 | 식 | 비고 |
|---|---|---|
| 연료비 | `주행거리 / 연비 × 연료단가` | EV는 `주행거리 / 전비 × 전기단가`. 단가는 오피넷/한전 기준 [추정]: 휘발유 1,650 · 경유 1,540 · LPG 1,100 원/L · 전기 320 원/kWh |
| 자동차세 | `배기량 × cc당 세율 × 1.3` | 승용 cc당: ≤1000cc 80원 / ≤1600cc 140원 / >1600cc 200원, 지방교육세 30% 가산. EV는 약 13만원 정액 [추정] |
| 보험료 | `f(차량가, 운전자 연령대)` | 데모는 차량가 비례 간이식 + 연령 계수 [추정]. 실값은 보험개발원 참고 |
| 소모품·정비 | `세그먼트·연료별 연 정액 [추정]` | HEV/EV는 정비비 낮게(엔진오일·미션 없음 반영) |
| (선택) 감가상각 | `차량가 × 10~15%/년` | 토글로 on/off, 총소유비용(TCO) 보기용 |

- **주행거리 슬라이더**(예: 0~40,000 km/년)를 움직이면 연료비·소모품이 즉시 재계산.
- 출력: 월 유지비(큰 숫자, 카운트업) + 연 유지비 + 항목별 막대 분해.
- 입력값(연봉·주행거리·성향)은 `localStorage` 저장으로 새로고침 후에도 유지.

## 6. 화면 설계

```mermaid
flowchart LR
  A[입력 화면\n연봉·주행거리·성향·연료] --> B[추천 결과\n옵션 필터 + 차량 카드 리스트]
  B --> C[차량 상세\n유지비 분해·전체 옵션·부담률]
  B --> D[비교함\n2~3대 나란히 비교]
  C --> B
  D --> B
```

| 화면 | 핵심 행동(Primary) | 주요 요소 |
|---|---|---|
| 입력 | "내 차 찾기" | 연봉 입력(큰 숫자), 연간 주행거리, 예산 성향, 연료 선호 |
| 추천 결과 | 차량 카드 선택 | 옵션 필터 칩(그룹별), 정렬, 차량 카드(가격·연비·옵션 배지·월 유지비), 예산 신호등 |
| 차량 상세 | "비교함에 담기" | 유지비 항목별 분해, 전체 옵션 매트릭스, 연봉 대비 부담률, 주행거리 슬라이더 |
| 비교함(부가) | — | 2~3대 스펙·유지비 나란히 |

- 디자인 톤: 브랜드 블루 #3182F6 단일 강조, ink 그레이스케일, Pretendard, 해요체 카피, 풀폭 주 버튼 52px. (기존 6개 앱과 동일 기준)
