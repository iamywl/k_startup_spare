## 1. 과업의 개요

**과업의 목적**
1) v1 PoC(배출→계근→인계→정산→ESG)를 **베타 출시·시리즈A 데모 가능한 수준**으로 끌어올린다. 즉 ① **실 알고리즘**(배출량 예측·정산 이상 탐지·처리업체 추천·탄소 산정), ② **다중 사업장(멀티테넌트)·역할기반 권한(RBAC)**, ③ **외부 시스템 통합 4종**(올바로 전자인계 신고 전송·계근 IoT 게이트웨이·펌뱅킹 정산 CSV·탄소 리포트 외부 제출)을 탑재한 **단일 HTML 자체완결 웹 데모 1식**을 제작한다.
2) 본 시제품은 제안서([`1_제안서.md`](./1_제안서.md)) §Solution·§차별성(정산 분쟁 방지·올바로 기한 관리·ESG 자동화)을 실증해, 베타 영업·심사 시연·투자 데모의 근거물로 쓰인다. v1(`index.html`)은 그대로 보존하고 v2는 신규 `v2.html`로 분리한다.

**과업의 범위**
1) 외주사 수행 업무
   - **멀티테넌트**: 사업장(테넌트) 3곳 데이터 분리, 상단 테넌트 전환, 테넌트별 현황 집계.
   - **RBAC**: 5개 역할(관리자·배출담당·운반기사·처리장·심사관(읽기)) — 역할별 표시 뷰·권한 분기, 역할 전환 UI, 역할×권한 매트릭스.
   - **실 알고리즘 4종**: (a) 배출량 예측(3개월 이동평균 + 최소제곱 추세 + 잔차 표준편차 예측구간·R²), (b) 정산 이상 탐지(체인 편차 ±3% + 품목 2σ 통계 이상치 + 청구단가 불일치), (c) 처리업체 가중점수 추천(단가·거리·재활용률·정산신뢰도), (d) 탄소(CO₂e) 산정 계수 모델·재활용률 실집계.
   - **3자(배출·운반·처리) 순차 전자서명** 인계서(각 역할만 해당 단계 서명 가능) → 발행 → 올바로 D-day.
   - **외부 통합 4종(mock·실 입출력)**: 올바로 전자인계 신고 전송(접수번호 로그) / 계근 IoT 게이트웨이 수신(디바이스·중량 로그) / 펌뱅킹 이체요청 CSV 내보내기·이체결과 CSV 가져와 정산 반영 / 탄소 리포트 외부 포털 제출(접수 ID 로그).
   - **감사로그**: 모든 상태 변경(배출·계근·서명·정산·연동)을 행위자·역할·테넌트·시각으로 기록 + CSV.
   - 운반·처리 콘솔(역할별 배정 작업), 검증 보류 정산 차단, 정산 검증 전용 뷰.
   - 반응형(모바일 바텀탭/드로어, 390px overflow 0) + localStorage(키 `recyclo-v2`) 지속.
   - PC(1280×900)·모바일(390×844) 실 구동 캡처 각 10장+ 제공.
2) Out of Scope (외주사 미수행)
   - 올바로 **공식 API 실연계**(공공 인증·실 전송) — v2는 신고 전문 매핑·전송 로그까지 mock.
   - 계근기 실 하드웨어 결선, 실 펌뱅킹 이체 실행(은행 API), 실 결제·세금계산서 국세청 발행 연동.
   - 정식 회원 인증·SSO(데모는 로그인 없이 역할 전환, CLAUDE.md §3.4). 서버 백엔드·DB(데모는 localStorage).

## 2. 과업의 세부 내용

**시제품 사양**
- 형태: 단일 `v2.html` 자체완결(외부 인프라 없이 `file://`·정적 호스팅 구동). 외부 라이브러리는 CDN(jsPDF) + 키 부재 시 mock 폴백. **하드코딩 시크릿 없음.**
- 데이터: 브라우저 `localStorage`(키 `recyclo-v2`, v1 키 `recyclo-v1`과 분리) 지속, CSV 내보내기/가져오기.
- 시드: 사업장 3곳·폐기물 품목 6종·처리사 3곳(품목별 단가·거리·재활용률)·위탁단가·인계 이력 17건(예측을 위한 다월 이력 + 이상치 1건 의도 삽입).

**UI/UX 제작 사양**
- 디자인 톤: [`design-guide/디자인_애플_토스.md`](../../design-guide/디자인_애플_토스.md)(컬러 UI 허용 — 흑백 규칙은 `biz/` 문서 그림자료에만 적용).
- 데스크톱: 좌측 사이드바(운영·인텔리전스·연동 3섹션), 상단 테넌트·역할 셀렉트. 모바일: 바텀 탭바 + 햄버거 드로어, 그리드 스택, 넓은 표는 가로 스크롤 래퍼(min-width), 차트 고정높이, 로그박스 `word-break`.
- 뷰 12종: 대시보드 / 배출·계근 / 전자인계서 / 정산 / 배출량 예측 / 처리업체 추천 / 정산 검증 / ESG·탄소 / 외부연동 / 운반·처리 콘솔 / 감사로그 / 테넌트·권한.

**사용 부품·라이브러리**
- jsPDF(인계서·세금계산서·ESG PDF), Canvas(서명·예측/도넛 차트), 표준 Web API(localStorage·Blob/CSV·FileReader). API 키 하드코딩 금지.

**품질·성능 기준(정량)**
- 화면/뷰 **12종 이상**(8종 기준 초과), 다단계 워크플로 **3개 이상**(아래 §5).
- 실 알고리즘 4종 동작(예측 R²·예측구간 산출 / 2σ 이상치 탐지 / 가중점수 순위 / 탄소 계수 환산).
- 멀티테넌트 3곳 데이터 분리 + 역할 5종 권한 분기(권한 없는 액션 버튼 비활성).
- 외부 통합 4건 각각 **가시적 로그/산출물**(접수번호·디바이스 로그·CSV 파일·접수 ID) 생성.
- 모바일 390px 폭 가로 overflow 0(12뷰), 콘솔 pageerror 0.
- 핵심 액션 실동작(토스트 mock 금지): 서명은 캔버스→이미지, PDF는 jsPDF 실생성, 예측·검증·추천·정산은 실 계산.
