## v2 한계 및 v3 개선 매핑

> v3는 v2(베타/시리즈A 데모, 5억)의 부족함을 메워 **시리즈A(투자·글로벌 ESG 공시 시연)** 수준으로 끌어올리는 사이클이다. v2에서 부족했던 항목 ↔ v3 해결 방식 1:1.

| # | v2 한계 | v3 개선 | 증거 |
|---:|:---|:---|:---|
| 1 | **탄소 = 단일 처리방법 계수** — 처리방법별 1계수로 CO₂e 환산(운송·수거·회피배출 미반영) | **LCA 전과정 5단계 정밀 산정** — 수거·운송(거리×계수)·중간처리·최종처분·재활용 회피배출(크레딧)로 분해, 순배출(net)·품목별 탄소강도 | §5-4 / 캡처 04-lca |
| 2 | **순환성 지표 없음** — 재활용률만, 순환경제 정량지표·예측 부재 | **순환경제 지표(MCI) 산정·예측** — `MCI=1−LFI·F(X)`(0~1)·재자원화율·신재대체량 + 분기별 순환성 추세 예측 | §5-2,3 / 02-circular |
| 3 | **업체 추천만(단일 기준 가중합)** — 처리방법 고정, 비용·탄소 동시 최적화·파레토 부재 | **처리경로 최적화** — 처리사×처리방법 경로 생성, 정규화 비용·전과정 탄소 가중합 목적함수 최소 + 파레토 최적해, 탄소가중치 슬라이더 | §5-6 / 06-route |
| 4 | **정산 이상탐지 = 3중 룰(이진 판정)** — 위험도 정량화·다요인 종합 부재 | **부정 리스크 스코어 고도화** — 0~100 가중(체인·z·단가·**Benford 1자리 적합**·처리사이력) + 고/중/저 + 고위험 정산·전송 동시 차단 | §5-7 / 07-risk |
| 5 | **한국어 전용** — 글로벌(영문) ESG·탄소 공시 시연 불가 | **KO/EN 다국어(i18n)** — 상단 토글로 전 뷰 UI·동적텍스트·역할명·품목명·통화 전환, 언어 지속 | §5-10~14 / 10~14-en-* |
| 6 | **역할 5종·연동 4종** | **역할 7종(+ESG심사역·지자체담당)·외부 통합 6종(+지자체 폐기물관리·환경공단 실적)** | §5-9,15,16,17 |
| 7 | **재자원화 수익화 없음** — 처리=비용, 유가물 매각 개념 부재 | **유가물(재생원료) 거래소** — 매도 호가·시세 매칭 체결·정산수익·MCI 반영 | §5-8 / 08-exchange |
| 8 | **뷰 12종·워크플로 3개** | **뷰 17종·워크플로 4개+** | 전 캡처 |

## v3 신규/심화 산출물

> v3에서 새로 추가·심화한 기능을 캡처와 함께 입증한다.

| 분류 | 신규/심화 산출물 | 핵심 |
|:---|:---|:---|
| 알고리즘 | LCA 전과정 탄소 | `lcaStages()` — 수거·운송(거리×`LCA.truck`)·중간처리·최종처분·재활용 크레딧(품목별), net = gross + credit |
| 알고리즘 | 순환경제 지표(MCI) | `siteMci()` — `recovery`·`lfi=(virginShare+landfillShare)/2`·`mci=1−lfi·F(X)`, `quarterlyMci()`→`forecast()` 예측 |
| 알고리즘 | 처리경로 최적화 | `optimizeRoute()` — `obj=(1−wC)·정규화비용 + wC·정규화탄소`, 파레토(비용·탄소 비지배) 식별 |
| 알고리즘 | 부정 리스크 스코어 | `riskScore()` 0~100 다요인 가중 + `benfordFit()=1−평균절대편차`, 고위험 게이트 `highRisk()` |
| 국제화 | KO/EN i18n | `I18N{ko,en}` 사전 + `data-i18n` + `applyI18n()` + 동적 `LANG` 분기(역할명·품목명·통화) |
| 사용자 모델 | 역할 7종 | +ESG심사역·지자체담당, `ROLE_VIEWS`·`PERM_LABELS`(거래소 권한) 확장 |
| 통합 | 외부 6종 | +지자체 폐기물관리 신고(접수번호)·환경공단(KECO) 순환경제 실적 제출(접수ID) |
| 거래 | 유가물 거래소 | 매도 호가·시세 매칭 체결·정산수익·MCI 반영, 거래 로그 |
| 운영 | 고위험 정산·전송 차단 | 리스크 고위험 건 `settle`·`allbaro` 동시 차단(과청구·부정 방지 강화) |

## 3. 환경

- 형태: 단일 `v3.html` 자체완결(외부 인프라·빌드 불필요, `file://` 구동). v1 `index.html`·v2 `v2.html`와 별개 파일·별개 localStorage 키(`recyclo-v3`).
- 라이브러리: jsPDF 2.5.1(CDN). 그 외 표준 Web API(Canvas·localStorage·Blob/CSV·FileReader). **하드코딩 시크릿 없음.**
- 캡처 환경: macOS / Playwright Chromium. PC 1280×900, 모바일 390×844(deviceScaleFactor 2, isMobile).
- 검증: 17뷰 전 화면 390px 가로 overflow **0**(`body.scrollWidth − innerWidth`, **KO·EN 양 언어**), 콘솔 **pageerror 0**.

## 4. 실행/구동 방법

1. `projects/waste-circular/v3.html`를 브라우저로 연다(`file://` 또는 정적 호스팅). 로그인 불필요(시드·역할·언어 자동 탑재).
2. 상단 **언어(EN/한국어)·테넌트(사업장)·역할** 셀렉트로 i18n·멀티테넌트·RBAC를 시연한다.
3. 고급 알고리즘: **순환경제 지표**(MCI·예측) → **LCA 탄소**(5단계·환경공단 제출) → **처리경로 최적화**(탄소가중치 슬라이더·파레토) → **부정 리스크**(스코어·Benford, 달성 물류센터에서 이상 건 확인).
4. 워크플로 ④: 처리경로 최적화 → **유가물 거래소** 매도 등록 → 시세 매칭 체결 → 순환경제 지표 재자원화율 반영.
5. 캡처 재현: 워크스페이스 트리에서 `node projects/waste-circular/capture-v3.mjs` (playwright는 워크스페이스 루트 `node_modules`로 해석).
