## 1. 과업의 개요

**과업의 목적**
1) v2(베타/시리즈A 데모)를 **시리즈A 투자·글로벌 ESG 공시 시연 가능 수준**으로 끌어올린다. 즉 ① v2 알고리즘을 넘어서는 **고급 알고리즘 4종**(LCA 전과정 탄소 정밀 산정·순환경제 지표(MCI) 산정/예측·처리경로 비용·탄소 동시 최적화·부정 정산 리스크 스코어 고도화), ② **KO/EN 다국어(i18n)** 전환(영문 ESG·탄소 공시 맥락), ③ **역할 7종·외부 통합 6종**으로의 확장을 탑재한 **단일 HTML 자체완결 웹 데모 1식**(`v3.html`)을 제작한다.
2) 본 시제품은 제안서([`1_제안서.md`](./1_제안서.md)) §Solution·§차별성(정산 분쟁 방지·올바로 기한 관리·ESG/순환경제 자동화)을 한 단계 심화 실증해, 시리즈A 투자 데모·글로벌(영문) ESG 리포팅 영업의 근거물로 쓰인다. v1·v2 는 그대로 보존하고 v3 는 신규 `v3.html`로 분리한다.

**과업의 범위**
1) 외주사 수행 업무
   - **고급 알고리즘 4종**:
     - (a) **LCA(전과정평가) 탄소 정밀 산정** — v2의 단일 처리방법 계수를 **수거(연료)·운송(거리×트럭계수)·중간처리(전력)·최종처분(매립가스/소각)·재활용 회피배출(크레딧)** 5단계로 분해, 순배출(net) 산출, 품목별 전과정 탄소강도(kg CO₂e/kg).
     - (b) **순환경제 지표(MCI)** — Ellen MacArthur 재단 MCI 기반 단순화 산식(`MCI = 1 − LFI·F(X)`, LFI=신재투입·매립폐기 비중), 재자원화율·신재 대체량 산출 + **분기별 순환성 추세 예측**.
     - (c) **처리경로 최적화** — 후보 경로(처리사 × 처리방법) 생성 → 정규화 비용·전과정 탄소 **가중합 목적함수** 최소화 + **파레토 최적해(지배받지 않는 해)** 식별, 탄소 가중치 슬라이더로 비용↔탄소 선호 조정.
     - (d) **부정 정산 리스크 스코어 고도화** — v2 3중 룰을 **0~100 가중 스코어**로 확장(체인편차·통계 이상치 z·청구단가 괴리·**Benford 1자리 적합도**·처리사 이상이력 등 다요인), 고·중·저 분류, **고위험 건 정산·올바로 전송 동시 차단**.
   - **다국어(i18n) KO/EN**: 상단 토글로 전 뷰(대시보드·ESG/탄소·순환경제·LCA·경로·리스크·연동 등) UI·라벨·역할명·품목명·처리방법·통화(원/KRW) 전환.
   - **역할 확장(7종)**: v2의 5역할(관리자·배출담당·운반기사·처리장·심사관) + **ESG심사역·지자체담당** 2종, 역할별 표시 뷰·권한(거래소 매매 권한 등) 분기.
   - **외부 통합 확장(6종)**: v2의 4종(올바로·계근 IoT·펌뱅킹·탄소포털) + **⑤ 지자체 폐기물관리 연동**(실적 신고·접수번호) + **⑥ 환경공단(KECO) 순환경제 실적 제출**(MCI·재자원화 실적·접수 ID). 각 통합 **가시적 로그/산출물**.
   - **유가물(재생원료) 거래소(mock)**: 재활용 품목 매도 호가 등록 → 시세 매칭·체결 → 정산 수익·MCI 반영.
   - 뷰 **17종**(v2 12 + 순환경제·LCA·처리경로·부정리스크·유가물거래소 5 신설), 워크플로 추가.
   - 반응형(모바일 바텀탭/드로어, 390px overflow 0, KO·EN 양 언어) + localStorage(키 `recyclo-v3`) 지속.
   - PC(1280×900)·모바일(390×844) 실 구동 캡처 각 **15장+**(신규 뷰 + EN i18n 화면 + 신규 역할 + 신규 연동 로그 포함).
2) Out of Scope (외주사 미수행)
   - LCA **공식 DB(예: ecoinvent) 연계·검증된 배출계수 실적용** — v3는 데모용 추정 계수 `[추정]`까지.
   - 올바로/지자체/환경공단 **공식 API 실연계**, 계근기 실 하드웨어·실 펌뱅킹 이체·국세청 발행 연동.
   - 정식 회원 인증·SSO(데모는 로그인 없이 역할 전환, CLAUDE.md §3.4), 서버 백엔드·DB(데모는 localStorage).
   - 다국어 추가 언어(중·일 등)·전문 번역 검수 — v3는 KO/EN 2개 언어.

## 2. 과업의 세부 내용

**시제품 사양**
- 형태: 단일 `v3.html` 자체완결(`file://`·정적 호스팅 구동). 외부 라이브러리는 CDN(jsPDF) + 키 부재 시 mock 폴백. **하드코딩 시크릿 없음.**
- 데이터: 브라우저 `localStorage`(키 `recyclo-v3`, v1 `recyclo-v1`·v2 `recyclo-v2`와 분리) 지속, CSV 내보내기/가져오기.
- 시드: 사업장 3곳·폐기물 품목 6종·처리사 3곳·인계 이력 17건(다월 이력 + 이상치 1건). v3 추가: 거래소 호가/체결·지자체/환경공단 제출 로그 상태.

**알고리즘 사양(정량)**
- LCA: 단계별 계수(`LCA.collect/process/disposal`)·운송계수(`LCA.truck` kg CO₂e/(kg·km))·재활용 크레딧(`LCA.credit` 품목별) → gross/net CO₂e 산출.
- MCI: `recv`(재활용 0.9·소각 0.25·매립 0.0) 회수가치 + 거래소 매각분 가산 → `recovery`·`lfi`·`mci(0~1)`. 분기 시계열 → `forecast()` 예측.
- 처리경로: `obj = (1−wCarbon)·정규화비용 + wCarbon·정규화탄소`, 파레토 플래그(비용·탄소 모두 비지배).
- 리스크: 체인(≤25점)·z 이상치(≤30)·단가괴리(≤30)·처리사이력(≤15) 가중합 → 0~100, ≥60 고위험·≥30 중위험. Benford 1자리 적합도 = `1 − 평균절대편차`.

**UI/UX 제작 사양**
- 디자인 톤: [`design-guide/디자인_애플_토스.md`](../../design-guide/디자인_애플_토스.md)(컬러 UI 허용 — 흑백 규칙은 `biz/` 문서 그림자료에만 적용).
- 데스크톱: 좌측 사이드바(운영·인텔리전스·연동 3섹션, 신규 뷰 포함), 상단 **언어 토글·테넌트·역할** 셀렉트. 모바일: 바텀 탭바(대시보드·배출·순환성·LCA·리스크) + 햄버거 드로어, 그리드 스택, 넓은 표는 가로 스크롤 래퍼(min-width), 차트 고정높이, 로그박스 `word-break`.
- i18n: `data-i18n` 속성 + `I18N{ko,en}` 사전 + `applyI18n()`. 동적 렌더 텍스트는 `LANG` 분기.

**사용 부품·라이브러리**
- jsPDF(인계서·세금계산서·ESG·**LCA** PDF), Canvas(서명·예측/도넛/**순환성** 차트), 표준 Web API(localStorage·Blob/CSV·FileReader). API 키 하드코딩 금지.

**품질·성능 기준(정량)**
- 화면/뷰 **17종 이상**, 다단계 워크플로 **4개 이상**(아래 §5).
- 고급 알고리즘 4종 실동작(LCA net 산출 / MCI 0~1·예측 / 경로 파레토·가중 최적 / 리스크 0~100·Benford).
- KO/EN 토글 시 전 뷰 라벨·동적텍스트·통화 전환, 새로고침 후 언어 유지.
- 역할 7종 권한 분기 + 외부 통합 6건 각 **가시적 로그/산출물**.
- 모바일 390px 가로 overflow 0(17뷰, **KO·EN 양 언어**), 콘솔 pageerror 0.
- 핵심 액션 실동작(토스트 mock 금지): LCA·MCI·경로·리스크 실 계산, PDF 실 생성, 거래소 체결·연동 제출 실 로그.
