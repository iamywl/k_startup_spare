## 1. 과업의 개요

**과업의 목적**
1) 배출 사업장이 **배출 등록 → 수거·계근 → 전자서명 인계서 → 정산 → ESG 리포트**를 한 흐름으로 수행하는 **단일 HTML 자체완결 웹 시제품(PoC) 1식**을 제작한다(오프라인 동작·반응형·실 알고리즘 탑재).
2) 본 시제품은 제안서([`1_제안서.md`](./1_제안서.md)) §2 Solution의 "배출-계근-인계-정산-ESG 파이프라인"을 실증해, 베타 영업·심사 시연·투자 데모의 근거물로 쓰인다.

**과업의 범위**
1) 외주사 수행 업무
   - 정보구조 설계 및 시드 데이터 구성(배출지점·폐기물 품목·처리사·위탁계약 단가·인계 이력).
   - 화면 6종 구현: 대시보드 / 배출·계근 / 전자인계(서명) / 정산 / ESG 리포트 / 외부연동.
   - 인계 체인 중량 검증·정산(계약단가×중량)·재활용률/매립/소각률·탄소 추정 알고리즘 구현.
   - 캔버스 전자서명 → 인계서, jsPDF 정산서·세금계산서·ESG 리포트 생성, 배출량/재활용률 차트.
   - 올바로(ALLBARO) 입력 매핑·입력기한(10일) D-day 알림 **mock** 구현, CSV 입출력.
   - 반응형(모바일 바텀탭/드로어, 390px 가로 overflow 0) 및 localStorage 상태 지속.
   - PC(1280×900)·모바일(390×844) 실 구동 캡처 제공.
2) Out of Scope (외주사 미수행)
   - 올바로 **공식 API 실연계**(공공 인증·전송) — v1은 mock 매핑·내보내기 로그까지만.
   - 계근기(중량계) IoT 실연동, 실 결제·세금계산서 발행 연동.
   - 서버 백엔드·DB·멀티테넌트·정식 회원 인증(시제품은 로그인 없이 동작, CLAUDE.md §3.4).

## 2. 과업의 세부 내용

**시제품 사양**
- 형태: 단일 `index.html` 자체완결(외부 인프라 없이 `file://`·정적 호스팅 구동). 외부 라이브러리는 CDN(jsPDF 등) + 키 부재 시 mock 폴백.
- 데이터: 브라우저 `localStorage` 지속(새로고침·재방문 유지), CSV 내보내기/가져오기.
- 시드: 배출지점 1+·폐기물 품목(폐합성수지·폐금속·슬러지·폐목재 등) 다종·처리사 2+·위탁단가표·최근 인계 이력.

**UI/UX 제작 사양**
- 디자인 톤: [`design-guide/디자인_애플_토스.md`](../../design-guide/디자인_애플_토스.md)(컬러 UI 허용 — 흑백 규칙은 `biz/` 문서 그림자료에만 적용).
- 데스크톱: 좌측 사이드바 내비, 콘텐츠 카드/표. 모바일: 바텀 탭바 + 햄버거 드로어, 그리드 스택, 차트 고정높이.
- 화면 흐름: 대시보드 → 배출 등록 → 수거·계근 → 전자인계(서명) → 정산 → ESG → 연동.

**사용 부품·라이브러리**
- jsPDF(정산서·세금계산서·ESG PDF), Canvas(서명·차트), 표준 Web API(localStorage·Blob/CSV). API 키 하드코딩 금지.

**품질·성능 기준(정량)**
- 화면/뷰 6종 이상, 다단계 워크플로 1개 이상(배출→계근→서명→인계→정산→ESG).
- 모바일 390px 폭에서 가로 스크롤(overflow) 0, 콘솔 pageerror 0.
- 핵심 액션 실동작(토스트 mock 금지): 서명은 캔버스→이미지, PDF는 jsPDF 실생성, 정산·ESG는 실 계산.
- 프로덕션 구동(정적 호스팅/`file://`) 시 정상 렌더.
