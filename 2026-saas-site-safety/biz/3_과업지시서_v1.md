## 1. 과업의 개요

### 과업의 목적

1) **직접 목적** — 소·중규모 건설현장(상시근로자 5~49인, 공사금액 50억 미만) 안전관리 SaaS 「세이프온」의 **시제품(PoC) 1식**을 제작한다. 현장소장·안전관리책임자·작업반장·경영책임자가 사용하는 단일 웹 애플리케이션으로, 현장의 안전 활동(위험성평가·TBM·아차사고) 1회 기록이 법정 서류·전자결재·이행 입증으로 동시에 분기하는 핵심 흐름을 실제 동작으로 구현한다.
2) **사업 연결** — 본 시제품은 [`1_제안서.md`](./1_제안서.md) §2 Solution(S-2 핵심 기능·S-3 핵심 워크플로·S-4 PoC 아키텍처)와 §차별화 기술의 구매동인 논증 ⑤(데모 정합)에서 약속한 "1회 기록 → 법정서류·전자결재·이행입증 동시 충족"을 실증하는 산출물이다. 심사·시연·현장 인터넷 음영지역에서 서버 없이 단독 구동되어야 한다.

### 과업의 범위

1) **수행 업무(In Scope)**
   - 3역할(현장소장/안전관리책임자·작업반장·경영책임자) 전환·뷰 라우팅·localStorage 상태 지속(시드·마이그레이션 포함) 골격 구축
   - 안전 대시보드: KPI 4종(미이수·TBM·고위험·이수율), 최근 7일 아차사고 추세 막대, 의무 이행 체크
   - 위험성평가 4단계 워크플로: 공정 선택(표준 위험요인 자동 로드)·빈도×강도 자동 산정·감소대책 입력·검토·발행(서류 자동 등록 + 결재 상신)
   - TBM 3단계 워크플로: 점검항목 6종 체크·위험요인 주지·참석자 캔버스 서명(이미지 저장)·일지 저장
   - 아차사고 3단계 워크플로: 유형·위험도·위치·현장 사진 실제 업로드·미리보기·신고(대장 등록, 위험도 '상'은 즉시 결재 상신)
   - 안전교육 이수: 근로자×교육 매트릭스, 미이수(법정) 자동 표시·토글, CSV 내보내기, 교육일지 PDF
   - 법정 안전서류: 기록→서류 자동 생성·서류함·PDF 출력
   - 전자결재: 안전관리자→현장소장→경영책임자 3단계 결재 라인·승인 진행·완결
   - jsPDF 한글 문서 4종(위험성평가표·TBM일지·교육일지·아차사고대장), 한글은 캔버스 렌더 후 이미지 삽입으로 깨짐 방지
   - 모바일/PC 반응형: 데스크톱 사이드바 ↔ 모바일 바텀 탭바 + 오프캔버스 드로어
   - 실 구동 캡처(PC·모바일 분리) 및 검수 협조
2) **Out of Scope(범위 외)** — 분쟁 예방을 위해 명시한다.
   - 실 백엔드 서버·DB·다현장(멀티테넌트) 인프라 구축 (v2 이후)
   - 카카오 알림톡 실 발송·전자문서·KOSHA 자료 실 연동·실 OAuth/SSO (PoC는 Mock·로컬로 대체)
   - 회원가입·로그인 정식 인증(시제품은 역할 전환으로 우회, `CLAUDE.md` §3.4)
   - 모바일 네이티브 앱(iOS/Android) 패키징
   - 법적 자문·개인정보 실 운영 처리·보안 인증(ISMS 등)

## 2. 과업의 세부 내용

### 2.1 시제품 사양
- **구동 형태**: 단일 HTML 파일(`index.html`), 빌드리스·오프라인 단독 구동. `<meta name="viewport" content="width=device-width, initial-scale=1">` 필수.
- **상태 저장**: localStorage 기반. 최초 진입 시 데모 데이터(현장·근로자·교육 시드) 주입, 새로고침 후에도 입력 유지. 스키마 변경 시 자동 마이그레이션.
- **역할/권한**: 현장소장/안전관리책임자·작업반장·경영책임자 3역할. 역할 전환이 가능해야 한다(데모 우회).
- **위험성 산정 정확성**: 위험성 = 빈도(1~5) × 강도(1~5). 15↑ '상', 8↑ '중', 그 외 '하'로 자동 등급화. 선택 변경 시 즉시 재계산되어 화면·발행 서류·대시보드 고위험 KPI 간 일치해야 한다.

### 2.2 사용자 UI/UX 제작 사양
- 디자인 톤: Apple×Toss 톤(`design-guide/디자인_애플_토스.md` 기준). 안전/건설 도메인 액센트=세이프티 오렌지, WCAG AA 대비.
- 화면 흐름: 다단계 워크플로(위험성평가 4단계·TBM 3단계·아차사고 3단계)는 진행 단계 표시(스텝퍼)·이전/다음 내비게이션을 제공.
- 반응형: 좁은 화면(360~430px) 가로 overflow 0, 데스크톱 사이드바는 모바일에서 바텀 탭바 + 더보기 오프캔버스 드로어로 재구성(축소판 금지). 출고 전 390px·1280px 스크린샷 눈검수 통과.

### 2.3 사용 부품·라이브러리·외부 의존성
- jsPDF 2.5.1 (CDN). 부재 시 텍스트 다운로드로 폴백(오프라인 시연 보장). 한글은 캔버스 렌더 후 이미지 삽입으로 깨짐 방지.
- 차트는 외부 차트 라이브러리 없이 자체 CSS 막대 구현. 서명은 자체 canvas 구현.
- 카카오 알림톡 등 외부 API는 미사용(정식판 연계). API 키 하드코딩 금지(`CLAUDE.md` §3.1).

### 2.4 품질·성능 기준 (정량)
- 화면/뷰: **8종 이상**(대시보드·위험성평가·TBM·아차사고·교육이수·법정서류·전자결재·현장설정). 다단계 워크플로 **3개 이상**.
- 구현 기능 수: **30개 이상** (코드 하단 `window.__FEATURES.length`로 검증).
- 상태 지속성: 새로고침 후 입력 데이터 100% 유지.
- 한글 표시: 화면·PDF 산출물 한글 깨짐 0건.
- 반응형: 390px·1280px 뷰포트에서 가로 overflow 0, 레이아웃 깨짐(사이드바 하이재킹·잘림·빈 차트) 0건.
- 콘솔 에러(pageerror) 0건.
