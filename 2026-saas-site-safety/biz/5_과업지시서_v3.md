## 1. 과업의 개요

### 과업의 목적

1) **직접 목적** — v2(본사·다현장 베타) 위에 **시리즈A 투자·전국 확산 단계**의 의사결정 도구를 얹는다. ① **고급 예측 알고리즘 4종**(로지스틱 발생확률·K-means 핫스팟 군집·교육효과 통계검정·작업허가-사고 상관)으로 위험을 *사전에* 정량 예측하고, ② **다국어(KO/EN) i18n**으로 건설현장 외국인 근로자 대응을 실증하며, ③ **관계기관·협력사 연동(지자체·산재·KOSHA·협력업체 포털)**으로 원·하청 안전관리를 일원화한다.
2) **사업 연결** — [`1_제안서.md`](./1_제안서.md) §Scale-up(데이터 축적 → AI 예측·전국 SaaS)과 §차별화 기술의 구매동인 논증에서 약속한 "기록 입증 → 본사 통합 → **예측·관계기관 연계**"의 마지막 고리를 실증한다. 시리즈A 데모·실현장 확산 심사에서 단독 구동되어야 한다.

### 과업의 범위

1) **수행 업무(In Scope)** — v2 위에 다음을 추가/심화한다.
   - **고급 알고리즘 4종(실 연산)**: ① 로지스틱 회귀 월간 중대재해 발생확률(`P=1/(1+e^-z)`, 6요인 로그오즈·오즈비 분해) ② K-means(k=3) 위험 핫스팟 군집(현장×공종 2차원 표준화·산점도·히트맵) ③ 안전교육 효과 Welch t-검정(완전이수 vs 미이수 집단 노출률 비교·위험감소율) ④ 작업허가(PTW) 준수율-사고빈도 Pearson 상관.
   - **다국어 KO/EN i18n**: 상단 토글로 전 화면 전환(사이드바·섹션 헤더·핵심 화면·TBM·발생확률·핫스팟). 외국인 근로자 대응 맥락.
   - **신규 뷰 6종**: 중대재해 발생확률(AI)·위험 핫스팟 군집·작업허가(PTW)·교육효과 통계분석·협력업체 포털·지자체·산재 연동센터.
   - **역할 확장**: v2 5역할 → **6역할**(협력업체 안전담당 추가, 결재 4단계로 확장: 협력사→안전관리자→소장→경영책임자).
   - **외부 연동 확장**: v2 4종 → **8종**(추가: 지자체 안전점검 신고·근로복지공단 산재보고·협력업체 포털·KOSHA 위험성평가 등록). 각 연동은 접수번호/처리번호 등 가시 아티팩트·로그 적재.
   - **신규 워크플로**: 작업허가(PTW) 3단계(작업종류→위험통제→승인발행) + 고위험 시 결재·협력사 통지 자동.
   - 모바일/PC 반응형 유지(산점도·히트맵·통계타일·언어토글 모바일 대응), 실 구동 캡처(PC·모바일 분리) 및 눈검수.
2) **Out of Scope(범위 외)**
   - 실 백엔드·DB·실 ML 학습 파이프라인(계수는 자체 가정 `[추정]`, 결정적 시드 — 정식판)
   - 지자체·근로복지공단·KOSHA·카카오 **실 API 연동**·실 OAuth/SSO (v3는 mock·접수번호 발급 시연으로 대체, `CLAUDE.md` §3.1·§3.4)
   - 실 번역 검수(법률 용어 공인 번역)·다국어 5개 이상 확장
   - 회원가입·로그인 정식 인증(역할 전환 우회)·모바일 네이티브 앱·ISMS

## 2. 과업의 세부 내용

### 2.1 시제품 사양
- **구동 형태**: 단일 HTML(`v3.html`), 빌드리스·오프라인 단독 구동. v1(`index.html`)·v2(`v2.html`) 미수정 보존(`CLAUDE.md` §3.5).
- **상태 저장**: localStorage 별도 키 `safeon-v3`(v1·v2와 분리) + 언어 키 `safeon-lang`. 현장별 데이터·PTW·연동 로그·역할·언어 지속, 새로고침 후 유지, 스키마 변경 시 자동 마이그레이션.
- **RBAC**: `ROLES` 맵 6역할(`views`·`apprStep`·`readonly`). 신규 6뷰 권한 분기, 협력업체 역할 추가, 결재 4단계.

### 2.2 알고리즘 사양 (정량 — 실 연산)
- **로지스틱 발생확률**: `z = β₀ + Σ(βᵢ·xᵢ)`, `P=1/(1+e^-z)`. 6요인(교육 미이수·고위험 밀도·아차사고·PTW 미준수·기상·TBM)을 0~1 정규화, 회귀계수·절편은 **자체 가정 `[추정]`**(산업 통계 방향성에 정성 정합)으로 화면 명시. 요인별 로그오즈 기여(βx)·오즈비(e^β) 분해표. 입력 변경 시 즉시 재산정.
- **K-means 핫스팟 군집**: 현장×공종 작업존을 (위험성=빈도×강도, 사고빈도) 2차원 표준화(z-score) 후 k=3 군집, 결정적 초기 중심으로 재현성 보장. 군집을 위험도순 라벨링(고/중/저), 산점도·현장×공종 히트맵·고위험 군집 테이블.
- **교육효과 Welch t-검정**: 전 현장 근로자를 법정교육 완전이수/미이수 두 집단으로 나눠 평균 노출률·표본분산으로 Welch t-통계량 산출, |t|≥1.96 유의 판정, 위험감소율(%) 정량화. 집단 노출은 결정적 프록시 `[추정]`.
- **PTW-사고 Pearson 상관**: 현장별 (PTW 준수율, 정규화 사고빈도) Pearson r 산출, 음의 상관(허가제 효과)으로 해석.

### 2.3 외부 연동 사양 (8종 — 각 가시 산출물 필수)
- v2 4종 유지(카카오 알림톡·기상 API·CSV 입출력·전자결재).
- **지자체 안전점검 신고(mock)**: 발생확률 고위험 시 관할 지자체 신고, **접수번호(GOV-…) 발급**·로그.
- **근로복지공단 산재보고(mock)**: 미결 사고 산재 보고, **처리번호(WCF-…) 발급**·로그.
- **협력업체 포털(mock)**: 고위험작업·핫스팟·안전공지 협력사 통지, 협력사 수·상태 로그.
- **KOSHA 위험성평가 등록(mock)**: 위험성평가 결과 안전보건공단 시스템 연계 동기화 로그.
- 8종 모두 연동센터/연동로그 뷰에서 시점·채널·상태로 적재, 설정에서 ON/OFF.

### 2.4 다국어 i18n 사양
- 상단 KO/EN 토글, `I18N` 사전(ko/en)·`t(key)`·`applyI18n()`로 `data-i18n` 노드 일괄 교체.
- 적용: 사이드바 메뉴·섹션 헤더·브랜드 부제·테넌트/역할 라벨·발생확률/핫스팟/교육효과/PTW/협력사/연동센터 페이지 헤더·TBM 단계 라벨. 언어 선택 localStorage 지속.

### 2.5 UI/UX 사양
- 디자인 톤: Apple×Toss(`design-guide/디자인_애플_토스.md`), 세이프티 오렌지 액센트, WCAG AA.
- 신규: 산점도 플롯(축·군집 색점)·현장×공종 히트맵·통계 타일·로지스틱 게이지·언어 토글 세그먼트.
- 반응형: 좁은 화면(360~430px) 가로 overflow 0, 사이드바→바텀탭+오프캔버스 드로어. 산점도·히트맵·통계차트 모바일 대응, 표는 가로 스크롤 래퍼. 390px·1280px 스크린샷 눈검수 통과(`design-guide/반응형_검수_필수.md`).

### 2.6 품질·성능 기준 (정량 — v3 시리즈A 심화)
- 화면/뷰: **18종**(v2 12 + 발생확률·핫스팟·PTW·교육효과·협력사포털·연동센터 6). 다단계 워크플로 **4개 이상**(위험성평가 4·TBM 3·아차사고 3·PTW 3).

- 실 알고리즘: **6종**(v2 CSI·선형회귀 + v3 로지스틱·K-means·Welch t·Pearson) — 입력 변경 시 화면·표·KPI 일치.

- 상태 지속성: 새로고침 후 현장별 데이터·PTW·언어 100% 유지.
