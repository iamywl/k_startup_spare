## 1. 과업의 개요

### 과업의 목적

1) **직접 목적** — v1 PoC(단일 현장 안전기록)를 **본사↔다현장(멀티테넌트) 베타·시리즈A 데모 수준**으로 끌어올린다. 본사(HSE팀·경영책임자)가 전 현장의 안전 상태를 **중대재해 위험지수(CSI)**로 정량 비교하고, **5단계 RBAC**로 역할별 권한을 분기하며, **외부 시스템(카카오 알림톡·기상 API·CSV 입출력·전자결재)** 연동을 시연 가능한 형태로 구현한다.
2) **사업 연결** — [`1_제안서.md`](./1_제안서.md) §Scale-up(다현장 본사 통합·SaaS 확장)과 §차별화 기술의 구매동인 논증에서 약속한 "현장 단위 의무 이행 입증 → 본사 통합 위험관리"를 실증한다. 시리즈A 투자·실현장 베타 도입 심사 자리에서 단독 구동되어야 한다.

### 과업의 범위

1) **수행 업무(In Scope)** — v1 위에 다음을 추가/심화한다.
   - **멀티테넌트**: 본사 1 + 현장 3개(강북 주상복합·수원 물류센터·부산 교량) 데이터 분리, 사이드바 현장 전환 셀렉터, 현장별 독립 시드(근로자·교육·기상·추세).
   - **RBAC 5역할**: 작업자/반장 · 현장소장/안전관리책임자 · 본사 안전관리자(HSE팀) · 원청/발주처(열람전용) · 대표이사(경영책임자). 역할별 접근 뷰 잠금, 열람전용 권한 차단(기록 작성·결재 불가), 결재 단계 분기.
   - **실 알고리즘 2종**: ① 중대재해 위험지수 CSI(6요인 가중 합성 0~100·등급·요인 기여도 분해) ② 선형회귀(최소제곱) 위험 추세 2시점 예측.
   - **외부 연동 4종(mock)**: 카카오 알림톡 발송 로그·기상청 API(폭염·강풍·호우 경보)·안전교육 CSV 입출력·전자결재 게이트웨이. 각 이벤트가 시점·채널·상태로 로그 적재.
   - **신규 뷰**: 본사 통합 대시보드·중대재해 위험지수·기상 위험경보·알림톡/연동 로그(+ v1 8뷰 유지) = 12뷰.
   - 전 현장 위험지수 비교 차트·전사 KPI 집계·현장별 안전현황 기상 배너 연동.
   - 모바일/PC 반응형 유지(테넌트 셀렉터·로그 콘솔·게이지 모바일 대응), 실 구동 캡처(PC·모바일 분리) 및 검수.
2) **Out of Scope(범위 외)**
   - 실 백엔드 서버·DB·실 멀티테넌트 인프라(클라우드 분리) (정식판)
   - 카카오 알림톡 **실 발송**·기상청 **실 API 키**·전자문서 실 연동·실 OAuth/SSO (v2는 mock·결정적 시드로 대체, `CLAUDE.md` §3.1·§3.4)
   - 회원가입·로그인 정식 인증(역할 전환으로 우회)
   - 모바일 네이티브 앱 패키징·보안 인증(ISMS)·법적 자문

## 2. 과업의 세부 내용

### 2.1 시제품 사양
- **구동 형태**: 단일 HTML(`v2.html`), 빌드리스·오프라인 단독 구동. v1(`index.html`) 미수정 보존(`CLAUDE.md` §3.5).
- **상태 저장**: localStorage 별도 키 `safeon-v2`(v1 `safeon-v1`과 분리). 현장별 데이터·연동 로그·역할 지속, 새로고침 후 유지, 스키마 변경 시 자동 마이그레이션.
- **멀티테넌트**: `S.sites[]` 배열로 현장 분리, `S.curSite`로 현재 현장 지정, 모든 기록/서류/결재가 현재 현장 스코프로 동작.
- **RBAC**: `ROLES` 맵에 역할별 `views`(접근 가능 뷰)·`apprStep`(결재 단계)·`readonly` 정의. 권한 없는 뷰 진입 차단·사이드바 잠금 표시·열람전용 쓰기 차단.

### 2.2 알고리즘 사양 (정량)
- **중대재해 위험지수 CSI** = Σ(요인 정규화값 × 가중치) × 100. 6요인: 법정교육 미이수(0.20)·고위험(상) 요인(0.26)·아차사고 빈도·미결(0.22)·당일 TBM 미실시(0.12)·기상 위험(0.12)·서류·결재 이행(0.08). 가중치 합 1.0. 60↑ 위험·35↑ 주의·그 외 안정. **가중치는 자체 가정 `[추정]`으로 화면에 명시.** 입력 데이터 변경 시 즉시 재산정되어 본사 대시보드·현장현황·요인 분해표 간 일치.
- **위험 추세 예측** = 최근 7일 아차사고 시계열에 최소제곱 1차회귀 적용, 기울기로 D+1·D+2 예측(음수 0 클램프). 예측값은 차트에 빗금 막대로 구분 표시.

### 2.3 외부 연동 사양 (각 가시 산출물 필수)
- **카카오 알림톡(mock)**: 발송 시 수신자 수·템플릿명·상태코드(2000)·본문이 연동 로그에 적재. 위험성평가 고위험·아차사고 '상'·기상경보·교육 미이수에서 자동/수동 발송.
- **기상청 API(mock)**: 현장별 최고기온·풍속·강수 결정적 시드 조회, 임계(폭염33/35·강풍10/14·호우10/30) 초과 시 경보·작업가이드 발령.
- **CSV 입출력**: 교육 이수현황 내보내기(BOM·UTF-8) + **외부 CSV 가져오기 파싱→근로자·교육 매칭 갱신** + 위험지수 전현장 CSV 내보내기. 각 입출력이 로그 적재.
- **전자결재 게이트웨이(mock)**: 결재 상신·단계별 승인·완결이 로그 적재.

### 2.4 UI/UX 사양
- 디자인 톤: Apple×Toss(`design-guide/디자인_애플_토스.md`), 세이프티 오렌지 액센트, WCAG AA.
- 위험지수 게이지(conic-gradient 링)·요인 기여도 진행바·추세+예측 막대(빗금 예측)·연동 로그 콘솔(monospace 다크).
- 반응형: 좁은 화면(360~430px) 가로 overflow 0, 사이드바→바텀탭+오프캔버스 드로어(테넌트 셀렉터·RBAC 셀렉터 포함). 표는 가로 스크롤 래퍼. 390px·1280px 스크린샷 눈검수 통과.

### 2.5 품질·성능 기준 (정량 — v2 5억 기준)
- 화면/뷰: **12종**(본사·위험지수·현장현황·기상·위험성평가·TBM·아차사고·교육·서류·결재·연동로그·설정). 다단계 워크플로 **3개 이상**(위험성평가 4·TBM 3·아차사고 3).
- 멀티테넌트: 본사 1 + **현장 3** 데이터 분리·전환.
- RBAC: **5역할** 권한 분기(뷰 잠금·열람전용 차단·결재 단계 분기).
- 실 알고리즘: CSI 6요인 가중모델 + 선형회귀 예측 — 입력 변경 시 화면·표·KPI 일치.
- 외부 연동: **4종** 각 가시 로그/산출물 남김(알림톡·기상·CSV·전자결재).
- 구현 기능 수: **50개 이상**(`window.__FEATURES.length`).
- 상태 지속성: 새로고침 후 현장별 데이터 100% 유지.
- 한글 표시·PDF 깨짐 0건, 390/1280 가로 overflow 0·레이아웃 깨짐 0, pageerror 0.
