## 3. 환경
- OS/브라우저: macOS · Chromium(Playwright 자동화)
- 언어/구성: 순수 HTML/CSS/JavaScript 단일 파일(빌드리스). 외부 의존성 = jsPDF 2.5.1(CDN, 부재 시 텍스트 폴백). 외부 API(카카오·기상)는 mock(키 불필요, 오프라인 시연).
- 뷰포트: PC 1280×900 / 모바일 390×844(deviceScaleFactor 2, isMobile, hasTouch)
- 저장: 브라우저 localStorage (`safeon-v2`, v1 `safeon-v1`과 분리)

## 4. 실행/구동 방법
1. `projects/site-safety/v2.html` 을 브라우저로 연다(서버 불필요, `file://` 가능). v1은 `index.html`로 그대로 보존.
2. 최초 진입 시 본사+현장 3개 데모 데이터가 자동 시드된다.
3. 사이드바 상단 **현장 셀렉터**로 본사↔현장 전환, 하단 **역할 셀렉터**로 RBAC 5역할 전환.
4. 위험지수·기상·연동로그는 신규 뷰, 위험성평가·TBM·아차사고는 워크플로로 끝까지 진행 가능.
5. 재캡처: `cd projects/site-safety && node capture-v2.mjs` → `biz/captures/v2`·`biz/captures/mobile/v2` 갱신.
