# 과업지시서 v3 — 임대지기 시리즈A 임대운영 OS (외주 위탁분)

> 본 문서는 발주자(임대지기 창업팀)가 v2 검수 통과 후 추가된 과업을 외주 용역사에 위탁할 때의 지시서다.
> v1·v2 과업지시서([3_과업지시서_v1.md](./3_과업지시서_v1.md) · [4_과업지시서_v2.md](./4_과업지시서_v2.md))는 수정하지 않으며, 본 v3는 신규 파일이다.
> 위탁 범위는 [2_개발계획서.md](./2_개발계획서.md) WBS의 v3 외주 구분 행(다변수 회귀·현금흐름·세금·최적화·룰엔진·다국어·외부통합 심화)에서 옮겨왔다.

## 1. 과업의 개요

### 과업의 목적
1. **직접 목적:** v2(3역할·연체 스코어링·적정 임대료·Cap rate·알림톡/CSV/전자서명)를 **시리즈A 투자 데모·실 사용 베타 수준의 "임대운영 OS"**로 끌어올린다. 구체적으로 **(a) 다국어 i18n(KO/EN) 전면 토글, (b) 다변수 임대료 회귀(면적·층·연식·역세권 가중), (c) 12개월 현금흐름 시뮬레이션, (d) 임대소득 세금 자동계산(종합 vs 분리 + 간주임대료), (e) 포트폴리오 수익률-공실 최적화, (f) 연체 단계별 자동 독촉 룰엔진(D+1→D+7→D+30), (g) 외부통합 심화(국토부 실거래가 mock API·오픈뱅킹 입금대사 webhook·전자세금계산서 발행 mock·OAuth 로그인 게이트)** 를 제작·통합한다.
2. **사업 연계:** [1_제안서.md](./1_제안서.md)가 약속한 데이터 해자(연체 예측)와 의사결정 자동화(GTM·수익모델)를 한층 심화한다. v2가 "데이터로 의사결정"을 보였다면, v3는 **알고리즘이 임대인을 대신해 판단·실행(룰엔진 자동 발송·세금 절세 권고·공실 최적 충원 시뮬)** 하는 단계를 보여, 단순 장부 SaaS가 아닌 "운영 자동화 OS"로의 포지셔닝을 실증한다. 다국어는 글로벌(외국인 임대인·관리 위탁사) 확장 가능성을 시연한다.

### 과업의 범위 (In Scope)
- **다국어 i18n(KO/EN):** 전 화면 텍스트·통화·월 표기·역할/등급 라벨·건물명을 사전 기반으로 토글. 데스크톱 사이드바·모바일 앱바 양쪽에 토글 버튼. 선택 언어 localStorage 지속.
- **다변수 임대료 회귀(알고리즘 심화):** 면적단가 × 면적 × (층 가중 × 연식 가중 × 역세권 가중)을 peer 정규화 회귀와 시장가로 혼합. 세대 상세 모달에 추정가·회귀 모델가·단가·가중계수·권고 표시.
- **12개월 현금흐름 시뮬레이션(신규 알고리즘):** 월별 임대수입(연체위험 반영 회수율 가중)−관리비−대출이자−유지보수±일회성 → 누적 현금흐름·최저 월 순익·적자 월 수. 막대(월 순익)+선(누적) 복합 차트.
- **임대소득 세금 자동계산(신규 알고리즘):** 분리과세(14%, 필요경비 50%·기본공제) vs 종합과세(누진 8구간) 비교 + 간주임대료(보증금 환산). 권장 방식·절감액 산출, 비교 차트.
- **포트폴리오 수익률-공실 최적화(신규 알고리즘):** 건물별 공실을 (현 상태/시세 임대/회귀 추정가 임대) 3시나리오로 Cap·ROI·NOI 비교 + NOI 증분 산출.
- **연체 단계별 자동 독촉 룰엔진(신규):** 미납 경과일에 따라 D+1 사전안내 → D+7 독촉(알림톡) → D+30 내용증명(mock) 자동 트리거. 단계별 중복 방지, escalation 로그 누적.
- **외부통합 심화 4종:** ①국토부 실거래가 조회 mock API(건물 역세권·연식 기반 결정적 단가·실거래 5건), ②오픈뱅킹 입금대사 webhook mock(입금 이벤트 수신 → 세대·금액 매칭 자동 수납), ③전자세금계산서 발행 mock(수납 청구 → 국세청 e-세로 mock 공급가·VAT 발행), ④OAuth 로그인 게이트(카카오/네이버 mock 토큰, 건너뛰기 제공).
- 위 기능을 단일 HTML(`v3.html`) **신규 파일**에 통합, 오프라인 구동·localStorage 지속 보장. **v1.html·v2.html은 보존**.
- 반응형(데스크톱 사이드바 ↔ 모바일 바텀탭 + 더보기 오프캔버스 드로어), 역할 스위처·언어 토글 모바일 대응.

### 범위 외 (Out of Scope)
- 실 서버·DB·계정 인증(정식 버전 과업). OAuth는 키 없이 mock 토큰 통과(오프라인 시연).
- 실 국토부 OpenAPI·실 오픈뱅킹 API·실 국세청 e-세로 연동(모두 결정적 mock으로 대체).
- 실 세무 신고 전송. 세금 계산은 데모 가정(타소득 3천만·필요경비 50%·간주임대료 이자율 2.9%) 기반 추정 — 실 신고는 세무 전문가 확인 필요.
- 다국어는 KO/EN 2개 언어. 제3언어·RTL 미포함.

## 2. 과업의 세부 내용

### 2.1 다국어 i18n (KO/EN)
- 사전(`I18N`) 기반 정적 텍스트(`data-i18n`) + 동적 라벨(`T()`/`L(ko,en)`) 전 화면 적용.
- 통화 표기: `KRW`/`KRWk`가 LANG에 따라 `원`/`만원`/`억` ↔ `KRW`/`k KRW`/`M KRW`. 월 표기·역할(임대인/Landlord)·등급(A 안전/A Safe)·건물명(행복빌라/Happy Villa) 동시 전환.
- 토글: 사이드바 `#langBtn` + 모바일 앱바 `#mLangBtn`. 전환 시 현재 뷰 재렌더·토스트, `S._lang` localStorage 지속.

### 2.2 다변수 임대료 회귀
- `floorFactor`(층당 +2%)·`ageFactor`(연식 1년당 −0.6%, 하한 0.82)·`stationFactor`(역≤0.5km ×1.06 / ≤1.0km ×1.0 / 초과 ×0.95).
- peer 정규화: 동일 건물 입주 월세 세대 단가를 각자의 가중으로 나눠 기준단가(원/㎡) 중앙값 산출 → 대상 세대 면적·가중 적용 = 회귀 모델가. 최종 추정 = 회귀 60% + 시장가 40%, 천원 단위 반올림.
- 출력: 세대 상세 모달에 추정 적정가·단가/㎡·회귀 모델가·층/연식/역세권 가중계수·갭(%)·권고.

### 2.3 12개월 현금흐름 시뮬레이션
- 월별 income = Σ(월세 × 회수율), 회수율 = 연체위험 점수 가중(≥70 0.6 / ≥45 0.85 / else 0.98).
- 비용 = 대출이자(연 4.5%/12) + 유지보수 적립(관리비/12 + 월세합×8%/12). 일회성 = 4개월차 수선 −150만, 7개월차 공실 신규임대 보증금 +500만(가정).
- 출력: KPI(12개월 누적·최저 월 순익·적자 월 수·대상 건물) + 복합 차트(막대 월 순익·선 누적) + 월별 상세 표.

### 2.4 임대소득 세금 자동계산
- 간주임대료 = max(0, 총보증금 − 3억) × 2.9% × 60%.
- 과세 총수입 = 연 임대료 + 간주임대료. 분리과세 과표 = max(0, 총수입×50% − 200만), 세액 = 과표×14%.
- 종합과세 = 타소득(3천만 가정) + 임대 필요경비 50% 후 누진(8구간) 한계세액. 권장 = min(분리, 종합), 절감액·비교 차트 출력. 연 2천만 이하 분리 선택 가능 안내.

### 2.5 포트폴리오 수익률-공실 최적화
- 각 건물 공실을 (a)현 상태 (b)시세(market) 임대 (c)회귀 추정가(estimate) 임대로 충원 시 EGI·NOI·Cap·ROI 재계산.
- 출력: 시나리오별 Cap rate 막대 차트(3계열) + 건물별 카드(3시나리오 Cap/ROI/NOI) + NOI 증분 권고 문구.

### 2.6 연체 단계별 자동 독촉 룰엔진
- 규칙: `D+1 사전안내`(알림톡)·`D+7 독촉`(알림톡)·`D+30 내용증명`(mock). `dunningStage(days)`로 현재 단계 판정.
- `runDunningEngine()`: 가시 범위 미납 청구 순회 → 현재 단계 미발송 건만 escalation 적재 + 채널별 발송(알림톡은 `notices[]`, 내용증명은 별도 로그). 단계 중복 방지(`charge+stage` 키). 발동 건수 토스트.
- 출력: 연체·룰엔진 뷰에 D+1/D+7/D+30 단계 카드 + 미납별 현재 단계 표 + 실행 버튼. 발동 누계 배지.

### 2.7 외부통합 심화 사양
- **국토부 실거래가 mock API:** `molitFetch(building)` — 건물 역세권·연식 기반 결정적 단가(원/㎡) + 실거래 5건(면적·월세·보증금·일자). 평균 단가 표시.
- **오픈뱅킹 입금대사 webhook mock:** `bankWebhook()` — 미납 청구 최대 2건을 입금 이벤트(`deposit.matched`)로 자동 수납 + `webhookLog[]` 적재(은행·예금주·금액·세대·상태).
- **전자세금계산서 발행 mock:** `issueTaxInvoice(charge)` — 수납 청구 → 국세청 e-세로 mock으로 공급가(합계/1.1)·VAT·총액·발행번호(TXI-) + `taxInvoices[]` 적재.
- **OAuth 로그인 게이트:** `?login`으로 진입, 카카오/네이버 mock 토큰 발급(키 불요) 또는 "로그인 없이 둘러보기"(게스트). 세션 칩 표시.

### 2.8 UI/UX 사양
- 톤: design-guide 애플×토스. 브랜드 인디고 1포인트. `:focus-visible` 링. press 인터랙션.
- 반응형: 데스크톱(≥768px) 다크 사이드바 + 역할 스위처 3버튼 + 건물 셀렉트 + 언어 토글. 모바일 바텀탭 4개 + 더보기 오프캔버스 드로어(전체 메뉴 + 역할 스위처) + 앱바(역할/언어/＋). 차트 고정 높이 래퍼.

### 2.9 사용 라이브러리·의존성
- jsPDF 2.5.1, html2canvas 1.4.1, Chart.js 4.4.1, Tailwind(CDN), Pretendard(CDN). 전부 CDN, 키 불요, 오프라인 구동. CDN 라이브러리는 외부 통합 카운트에서 제외(자체 통합으로 국토부/오픈뱅킹/e-세로/OAuth mock을 구현).

### 2.10 품질·성능 기준(정량)
- KO/EN 전환 시 텍스트·통화·라벨·건물명 **누락 없이** 전환(미번역 잔존 0).
- 다변수 회귀·현금흐름·세금·최적화 산출 **결정적**(동일 입력 → 동일 결과).
- 룰엔진 실행 시 단계 중복 발송 0(charge+stage 키 보장), 발동 건수 정확.
- 역할 3종 전환 시 메뉴·데이터 가시성 분기 **100% 정확**(임차인 화면에 타 세대 데이터 노출 0).
- 새로고침 후 수납·서명·룰엔진 로그·언어·역할 **100% 유지**(localStorage).
- 모바일 390px 가로 overflow **0**, 사이드바 미노출(바텀탭/드로어), 차트 0px 붕괴 **0**, pageerror **0**.
- 신규 캡처 PC ≥15장 + 모바일 ≥15장.
