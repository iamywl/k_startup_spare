## v2 한계 및 v3 개선 매핑

v3는 v2의 부족함을 메우는 사이클이다. v2가 "데이터로 의사결정"을 보였다면, v3는 **알고리즘이 임대인을 대신해 판단·실행**하는 단계(룰엔진 자동 발송·절세 권고·공실 최적 충원·현금흐름 예측)와 **글로벌·외부 시스템 연동**으로 한 단계 올렸다.

| # | v2 한계(부족했던 점) | v3 해결 방식 | 증거 캡처 |

## v3 신규/심화 산출물

### 실 알고리즘 (입력→계산→판단·실행)
1. **다변수 임대료 회귀** — 면적단가 × 면적 × (층×연식×역세권 가중)을 peer 정규화 회귀(60%)와 시장가(40%)로 혼합. 추정가·모델가·가중계수·갭·권고 산출.
2. **12개월 현금흐름 시뮬레이션** — 월별 임대수입(연체위험 점수에 따른 회수율 0.6/0.85/0.98 가중)−이자−유지보수±일회성 → 누적 현금흐름·최저 월 순익·적자 월 수.
3. **임대소득 세금 자동계산** — 분리과세(14%·필요경비 50%·기본공제 200만) vs 종합과세(누진 8구간·타소득 합산) + 간주임대료(보증금 환산). 권장 방식·절감액.
4. **포트폴리오 수익률-공실 최적화** — 공실을 현상태/시세/회귀가로 충원 시 Cap·ROI·NOI 재계산·비교.
5. **연체 단계별 자동 독촉 룰엔진** — 경과일에 따라 D+1→D+7→D+30 단계 자동 판정·발송·중복 방지·로그.

### 외부 시스템 통합 (CDN 라이브러리 제외, v3 신규 4건)
1. **국토부 실거래가 조회 mock API** — 건물 역세권·연식 기반 결정적 단가 + 실거래 5건.
2. **오픈뱅킹 입금대사 webhook mock** — 입금 이벤트(`deposit.matched`) 수신 → 세대·금액 매칭 자동 수납·로그.
3. **전자세금계산서 발행 mock** — 수납 청구 → 국세청 e-세로 mock 공급가·VAT·발행번호.
4. **OAuth 로그인 게이트** — 카카오/네이버 mock 토큰 + 게스트 둘러보기.

### 다국어·다중 사용자
- KO/EN 전면 i18n. 3역할(임대인 12뷰·관리소장 6뷰·임차인 3뷰) 권한 분기 + 다중 건물 포트폴리오 + 역할/언어 스위처(데스크톱·모바일 양쪽).

## 2. 구현/제작 범위

- **뷰: 임대인 12 / 관리소장 6 / 임차인 3종.** 각 뷰는 표시가 아니라 인터랙션 가능.
  - 임대인: 포트폴리오·건물세대·청구수납·연체룰엔진·계약서명·하자보수·수익률·현금흐름·최적화·세금계산·외부연동·알림센터.
  - 관리소장: 운영현황(할 일 큐)·건물세대·청구수납·연체룰엔진·하자보수·알림센터(담당 건물 한정).
  - 임차인: 내 임대·청구납부·하자신고(본인 세대 한정).
- **워크플로 3개+:** ① 미납→수납→영수증/세금계산서 PDF, ② 연체 룰엔진 실행→단계별 자동 알림톡/내용증명, ③ 계약→전자서명→PDF 반영→알림톡, ④ 오픈뱅킹 webhook 수신→자동 입금대사, ⑤ 임차인 하자 신고→임대인 알림톡→관리소장 진행/완료, ⑥ 세대 상세→다변수 회귀 추정→갱신 인상 의사결정.
- **실 동작(토스트 mock 아님):** 5종 알고리즘 실 계산, 룰엔진 실 발송·중복방지, 캔버스 서명 실 드로잉·PDF 삽입, KO/EN 실 전환, OAuth mock 토큰 발급, localStorage 지속, jsPDF+html2canvas 실 PDF, Chart.js 실 차트 6종.
- 로그인 없이 3역할·KO/EN 전환·시연, 키 불요·오프라인 구동.
