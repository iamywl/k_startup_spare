# 과업지시서 v1 — 임대지기 임대관리 PoC (외주 위탁분)

> 본 문서는 발주자(임대지기 창업팀)가 외주 용역사에 시제품 일부를 위탁할 때의 지시서다.
> 위탁 범위는 [2_개발계획서.md](./2_개발계획서.md) WBS의 외주 구분 행(계약서 PDF·하자 증빙·영수증 PDF 모듈)에서 옮겨왔다.

---

## 1. 과업의 개요

### 과업의 목적
1. **직접 목적:** 소규모 임대인용 임대관리 웹 PoC 중 **(a) 표준 임대차계약서 PDF 생성 모듈, (b) 하자보수 워크플로 + 증빙 사진 업로드 모듈, (c) 임대료 영수증 PDF 생성 모듈**을 제작·통합한다.
2. **사업 연계:** 이 모듈들은 [1_제안서.md](./1_제안서.md) Solution §S-2의 핵심 기능 ④계약·⑤하자, 그리고 워크플로(수납→영수증)를 완성하는 부분이다. 한글 깨짐 없는 PDF와 증빙 보존은 제안서가 약속한 "분쟁·과세 리스크 해소"의 실증 근거가 된다.

### 과업의 범위 (In Scope)
- 표준 임대차계약서 PDF 생성: 호실·임차인·보증금·차임·기간 데이터를 표준 양식에 바인딩, A4 PDF 다운로드.
- 임대료 영수증 PDF 생성: 수납 완료 청구건 → 영수증 양식 PDF 다운로드.
- PDF 한글 처리: html2canvas 기반 DOM→이미지 렌더로 한글 깨짐 0.
- 하자보수 워크플로: 접수→진행→완료 상태 전이, 우선순위, 증빙 사진 **실 업로드 + 미리보기(FileReader, dataURL)**, localStorage 보존.
- 위 기능을 단일 HTML(v1.html)에 통합, 오프라인 구동 보장.

### 범위 외 (Out of Scope)
- 실 서버·DB·계정 인증(정식 버전 과업).
- 실 SMS/알림톡 발송(독촉은 mock 로그로 대체) · 오픈뱅킹 입금매칭.
- 전자서명·공인인증·세무 신고 연동.
- 대시보드·호실/임차인/수납 추적 코어(자체 개발분).

## 2. 과업의 세부 내용

### 2.1 계약서 PDF 모듈
- 입력: 임차인 1명(호실·보증금·차임·관리비·납부일·계약기간) 데이터.
- 양식: 제1~5조(목적물·보증금/차임·계약기간·수선/관리·특약), 임대인/임차인 서명란.
- 출력: A4 세로 PDF, 파일명 `임대차계약서_<호실>_<임차인>.pdf`.

### 2.2 영수증 PDF 모듈
- 입력: 수납 완료(`paid=true`) 청구건.
- 양식: 발행번호·발행일·호실·임차인·대상월·입금일·임대료·관리비·합계·영수 문구·임대인 서명란.
- 출력: A4 PDF, 파일명 `영수증_<호실>_<YYYY-MM>.pdf`.

### 2.3 하자보수 모듈
- 접수 폼: 호실 선택·증상·우선순위(낮음/높음/긴급)·메모·**증빙 사진 업로드**.
- 사진: `<input type=file accept=image/*>` → FileReader dataURL → 미리보기 + 저장.
- 상태 워크플로: 접수 → (진행 시작) → 진행 → (완료 처리) → 완료. 상태별 색 배지.
- 상세 모달: 사진·메모·상태·우선순위 표시.

### 2.4 UI/UX 사양
- 톤: design-guide 애플×토스(명료·여백 위계·의미 있는 1개 포인트 컬러).
- 반응형: 데스크톱(≥768px) 사이드바 + 본문 / 모바일 바텀탭 + 바텀시트. 모바일 모달은 바텀시트형. 주요 버튼 ≥44px(풀폭 52px).

### 2.5 사용 라이브러리·의존성
- jsPDF 2.5.1, html2canvas 1.4.1, Chart.js 4.4.1, Tailwind(CDN), Pretendard(CDN). 전부 CDN, 키 불요, 오프라인 구동.

### 2.6 품질·성능 기준(정량)
- 한글 PDF 깨짐 **0건**(계약서·영수증 모두).
- 사진 업로드 후 미리보기 표시 **1초 이내**.
- 새로고침 후 하자·청구 상태 **100% 유지**(localStorage).
- 모바일 390px 가로 overflow **0**, pageerror **0**.
