# 과업지시서 v2 — 임대지기 포트폴리오 임대관리 (외주 위탁분)

> 본 문서는 발주자(임대지기 창업팀)가 v1 검수 통과 후 추가된 과업을 외주 용역사에 위탁할 때의 지시서다.
> v1 과업지시서([3_과업지시서_v1.md](./3_과업지시서_v1.md))는 수정하지 않으며, 본 v2는 신규 파일이다.
> 위탁 범위는 [2_개발계획서.md](./2_개발계획서.md) WBS의 v2 외주 구분 행(다중 역할·연체 스코어링·수익률·외부 통합)에서 옮겨왔다.

## 1. 과업의 개요

### 과업의 목적
1. **직접 목적:** v1(뷰 5종: 대시보드·호실/임차인·수납·계약·하자)을 **베타 출시·시리즈A 데모 수준**으로 끌어올린다. 구체적으로 **(a) 다중 역할(임대인·관리소장·임차인) 권한 분기 + 역할 스위처, (b) 다중 건물(포트폴리오) 운영, (c) 3대 실 알고리즘(연체 위험 스코어링·적정 임대료 추정·수익률 분석), (d) 외부 시스템 통합(카카오 알림톡 mock 발송 로그·CSV 입출력·전자계약 서명 mock)** 을 제작·통합한다.
2. **사업 연계:** [1_제안서.md](./1_제안서.md)가 약속한 데이터 해자(§M-2 "수납·연체 패턴 축적 → 연체 위험 예측")와 Scale-up 로드맵 Phase 2~3(수익화·연체 위험 스코어링·다건물 포트폴리오)을 PoC로 선제 실증한다. 즉 v2는 "감으로 버티던 임대인"이 **데이터로 의사결정**하는 단계를 보여준다.

### 과업의 범위 (In Scope)
- **다중 역할·권한 분기:** 임대인(전체 권한)·관리소장(담당 건물 운영)·임차인(본인 세대) 3역할. 역할 전환 스위처(데스크톱 사이드바 + 모바일 드로어), 역할별 메뉴·데이터 가시성 분기.
- **다중 건물 포트폴리오:** 2개 이상 건물, 건물 선택 필터, 전체 포트폴리오 합산 뷰.
- **연체 위험 스코어링(알고리즘):** 임차인별 과거 지연일수·미납 횟수·빈도·잔여 계약을 가중합산해 0~100 점수 + A/B/C/D 등급 + 권고 산출. 산출 근거 모달.
- **적정 임대료 추정(알고리즘):** 동일 건물 입주 세대의 단가(원/㎡) 중앙값 + 면적 보정 + 시장가 가중으로 추정가·갭(%)·권고 산출.
- **수익률 분석(알고리즘):** 건물별 GPI·EGI·OpEx·NOI·Cap rate·자기자본수익률(ROI)·세전 현금흐름 계산 + 비교 차트.
- **외부 통합 ①카카오 알림톡 mock:** 독촉·사전안내·계약알림 발송 → 알림 센터에 채널·수신자·템플릿·시각·상태 로그 누적.
- **외부 통합 ②CSV 입출력:** 수납내역 CSV 내보내기(BOM 포함 한글 안전), 은행 입금내역 CSV 붙여넣기 → 세대명·대상월 매칭 자동 입금대사.
- **외부 통합 ③전자계약 서명 mock:** 동의 체크 → 캔버스 서명 입력 → 서명 이미지 계약서 PDF 반영 + 알림톡 발송.
- **세무·정산 리포트:** 연 임대료·관리비 집계, 분리과세(연 2천만원) 대상 판정, 월별 실 수납 내역 + CSV 내보내기.
- 위 기능을 단일 HTML(`v2.html`) **신규 파일**에 통합, 오프라인 구동·localStorage 지속 보장. **v1.html은 보존**.
- 반응형(데스크톱 사이드바 ↔ 모바일 바텀탭 4개 + 더보기 오프캔버스 드로어), 역할 스위처 모바일 대응.

### 범위 외 (Out of Scope)
- 실 서버·DB·계정 인증(정식 버전 과업). 역할 분기는 클라이언트 데모 시뮬레이션.
- 실 카카오 비즈니스 채널 발송·실 은행 오픈뱅킹 API 연동(모두 mock·CSV 파싱으로 대체).
- 공인전자서명(KISA)·실 전자계약 플랫폼 연동(서명은 캔버스 캡처 mock).
- 실 세무 신고 전송(국세청 홈택스 연동). 리포트는 집계 보조용.

## 2. 과업의 세부 내용

### 2.1 다중 역할·권한 분기
- 역할 3종: `owner`(임대인)·`manager`(관리소장)·`tenant`(임차인).
- 역할별 메뉴: owner 9개 / manager 5개 / tenant 3개. 역할 전환 시 기본 뷰·건물 자동 보정.
- 데이터 가시성: owner=선택 건물(전체 가능), manager=담당 건물만, tenant=본인 세대·본인 청구·본인 하자만.
- 역할 스위처: 데스크톱 사이드바 3버튼 그리드 + 모바일 오프캔버스 드로어. 전환 시 토스트·상태 저장.

### 2.2 연체 위험 스코어링 알고리즘
- 입력: 임차인별 6개월 지연일수 이력, 현재 미납 여부, 잔여 계약개월.
- 가중: 평균 지연(×2.2, 최대 35) + 지연 빈도(×22) + 최대 지연(×0.6, 최대 15) + 최근 3개월 가중(최대 12) + 현재 미납(+14) + 만료 임박(+2). 0~100 클램프.
- 등급: <20 A안전 / <45 B주의 / <70 C경고 / ≥70 D고위험. 등급별 권고 문구.
- 출력 UI: 등급 분포 카드 + 막대 차트(Chart.js) + 임차인별 점수 카드(진행바·근거 모달).

### 2.3 적정 임대료 추정 알고리즘
- peer = 동일 건물 입주 월세 세대. 단가 = (월세/면적) 중앙값.
- 추정가 = (면적기반 base ×0.5 + 시장가 ×0.5), 천원 단위 반올림.
- 갭 = (추정가−현재가)/현재가. ±5% 기준 인상 여지/적정/공실위험 권고.
- 출력: 세대 상세 모달 내 추정가·단가·시세·권고.

### 2.4 수익률 분석 알고리즘
- EGI(유효총소득)=월세 합 ×12, OpEx=관리비 + EGI×8%(유지보수 추정), NOI=EGI−OpEx.
- Cap rate=NOI/매입가, 자기자본=매입가−대출, 대출이자=대출×4.5%, 세전현금흐름=NOI−이자, ROI=현금흐름/자기자본.
- 출력: 건물별 카드(전 지표) + Cap rate·ROI 비교 차트.

### 2.5 외부 통합 사양
- **카카오 알림톡 mock:** `sendKakao(to,name,template,body)` → `notices[]`에 채널·수신자·템플릿·시각·상태(발송완료) 적재. 알림 센터 뷰에서 카드 리스트로 표시.
- **CSV 입출력:** 내보내기는 BOM(`﻿`) + CSV 이스케이프로 한글·콤마 안전. 가져오기는 `세대,임차인,대상월` 헤더 파서 → 세대명·대상월 매칭 → 미납 청구 자동 수납 처리, 매칭 건수 토스트.
- **전자서명 mock:** 캔버스 마우스/터치 드로잉 → `toDataURL` → 계약서 PDF 임차인 서명란에 이미지 삽입, `contracts[]` 보관, 알림톡 계약체결 안내 발송.

### 2.6 UI/UX 사양
- 톤: design-guide 애플×토스. 브랜드 인디고 1포인트. `:focus-visible` 링.
- 반응형: 데스크톱(≥768px) 사이드바 + 역할 스위처 + 건물 셀렉트. 모바일 바텀탭 4개 + 더보기 드로어(나머지 메뉴 + 역할 스위처). 차트 고정 높이 래퍼.

### 2.7 사용 라이브러리·의존성
- jsPDF 2.5.1, html2canvas 1.4.1, Chart.js 4.4.1, Tailwind(CDN), Pretendard(CDN). 전부 CDN, 키 불요, 오프라인 구동. CDN 라이브러리는 외부 통합 카운트에서 제외(자체 통합으로 알림톡 mock·CSV·전자서명을 구현).

### 2.8 품질·성능 기준(정량)
- 역할 3종 전환 시 메뉴·데이터 가시성 분기 **100% 정확**(임차인 화면에 타 세대 데이터 노출 0).
- 연체 스코어 산출 **결정적**(동일 입력 → 동일 점수·등급).
- CSV 내보내기 한글 깨짐 **0**, 입금대사 매칭 정확.
- 새로고침 후 수납·서명·독촉 로그 **100% 유지**(localStorage).
- 모바일 390px 가로 overflow **0**, 사이드바 미노출(바텀탭/드로어), 차트 0px 붕괴 **0**, pageerror **0**.
- 신규 캡처 PC ≥10장 + 모바일 ≥10장.
